### 二、底层逻辑代码组成

#### 1. **数据通信层**
- **SSE (Server-Sent Events) 实时数据流**
  - 位置：各 Store 文件中的 `connectSSE()` 和 `disconnectSSE()` 方法
  - 功能：接收设备实时数据推送
  - 特点：单向通信、服务器主动推送

- **HTTP API 请求**
  - 位置：`src/api/api.ts` 和各模块 API
  - 功能：设备状态查询、控制命令发送

#### 2. **状态管理层 (Pinia)**
- 核心文件：`src/stores/index.ts`
- 功能：统一管理所有业务数据状态
- 特点：响应式数据、跨组件共享

#### 3. **图表可视化层**
- **ECharts 图表库**
  - 位置：`src/plugins/echarts.ts`
  - 用途：能谱图、趋势图、地图等可视化
  - 组件：`vue-echarts` 封装

#### 4. **通用组件层** (`src/components/`)
| 组件 | 功能 |
|------|------|
| **panel-wrap** | 面板容器包装 |
| **metric-card** | 指标卡片显示 |
| **trend-line-chart** | 趋势线图表 |
| **scale-screen** | 大屏自适应缩放 |
| **border-box-13** | 数据可视化边框效果 |
| **device-management** | 设备管理组件 |

#### 5. **工具函数层** (`src/utils/`)
- `storage.ts` - 本地存储管理
- `query-param.ts` - URL 参数处理
- `index.ts` - 通用工具函数

#### 6. **配置层**
- **环境配置**：`.env.development`、`.env.production`
- **构建配置**：`vite.config.ts` - 代理设置、路径别名
- **路由配置**：`src/router/index.ts`
- **枚举配置**：`src/enums/` - 请求枚举、存储枚举

#### 7. **Mock 数据层** (`src/mock/`)
- 功能：开发阶段模拟后端数据
- 位置：`mock-index.ts`

### 四、核心业务流程

1. **设备连接流程**
   - 用户在设置页面点击开关
   - 发送 HTTP 请求到 `/api/devices/connect`
   - 建立 SSE 连接接收实时数据
   - Store 更新状态，UI 自动响应

2. **数据展示流程**
   - SSE 推送设备数据
   - Store 解析并存储数据
   - 组件监听 Store 变化
   - ECharts 图表渲染更新

这就是辐射环境自动监测系统的完整代码结构！


#### 预设场景题
1️⃣ 既然你用了 Axios 封装和 SSE，那如果你的后端同学突然想把所有 SSE 改成 WebSocket，你的前端代码改动大吗？

- 答：“改动会非常小。因为我采用了‘接口与实现分离’的策略。在 `deviceStore` 中，业务逻辑只关心 `updateData()` 这个方法，而不关心数据是怎么来的。我只需要写一个新的 `WebSocket` 服务类，替换掉原来的 `SSE` 服务类，然后把解析后的数据依然喂给同一个 Store 即可

理解：**接口与实现分离**体现为上层业务逻辑只关心“我要做什么”，而不关心“底层是怎么做到的”。即Vue 组件（界面）和 Pinia Store（大脑）不应该知道数据是通过 Axios 拿到的，还是通过 SSE 推送过来的。

2️⃣ 实时数据渲染方案：从“直接驱动”到“数据缓冲池”
