#### 1️⃣指针和引用的区别
理解：指针是一种数据类型（`char`），引用是 c++提供的语法糖，引用是一个变量的别名
	- **指针写法**：`void update(int* p) { *p = 100; }` -> 调用时：`update(&a);`
	- **引用写法**：`void update(int& r) { r = 100; }` -> 调用时：`update(a);`
**引用**不会开辟新的内存空间，而是**指向已有的一个内存空间**，所以当函数形参传入的是引用的时候其实传入的是 a 变量的地址，主要实现的功能是不通过指针和取地址符即可实现函数修改外部变量的功能
解引用即顺着地址去找真身
指针一定要记得初始化，否则随机指向一个地址成野指针了可能修改系统关键数据
常见的初始化方法：
1. 初始化为 `nullptr`（C++11 新特性）
```cpp
#include <iostream>
int a = 10;
int b = 20;
std::cout << "结果是: " << a + b << std::endl;
```

#### 2️⃣ Vue.js 数据双向绑定
Vue 中 template 是模版，采用 html 的语法，告诉 vue 引擎哪个地方需要显示哪个变量，实时更新
**Vue 框架源码**：那是尤雨溪和团队写的一堆 JS 代码（包含我们聊过的 Proxy 拦截、虚拟 DOM 算法等）
**一、数据从 Vue 变量挂到标签当中**
Vue 引擎会调用浏览器底层的 JavaScript 原生指令：
1. **创建元素**：`const el = document.createElement('div');`
2. **设置属性**：`el.setAttribute('id', 'app');`
3. **填入数据**：`el.textContent = '0.15';` （或者操作 `nodeValue`）
4. **插入页面**：`document.body.appendChild(el);`
**结论**：数据是通过 **浏览器原生的 DOM 操作 API** 被写进 HTML 标签的。
Vue 内部有一个特殊的属性（通常是隐藏的），它把**真实的 DOM 元素**和它的**虚拟节点（VNode）关联在一起。
- 在内存里，Vue 存着一个类似 **`Map<RealDOM, VNode>`** 的字典。
- 当 `doseRate` 变了，Vue 会通过这个字典瞬间找到对应的那个 `div` 元素，然后再次执行 `el.textContent = newValue`
二、数据从标签反向挂到 Vue 变量当中
