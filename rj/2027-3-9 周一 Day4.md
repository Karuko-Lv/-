#### 1️⃣vue中的computed和watch的核心区别是？分别适用于哪些场景
在 Vue 3 开发中，`computed`（计算属性）和 `watch`（侦听器）是你最常打交道的两个响应式工具。
简单来说：**`computed` 是为了“推导结果”，而 `watch` 是为了“执行动作”。**

|**特性**|**computed (计算属性)**|**watch (侦听器)**|
|---|---|---|
|**核心目的**|从现有数据派生出**新的值**|观察某个数据变化并**执行副作用**（如弹窗、调接口）|
|**缓存性**|**有缓存**。只有依赖项变了才会重新计算|**无缓存**。只要被侦听的数据变了，函数就会运行|
|**返回值**|**必须有返回值**，供模板或其他逻辑使用|**不需要返回值**|
|**异步支持**|**仅支持同步逻辑**|**支持异步逻辑**（如 `setTimeout` 或 `fetch`）|
|**执行时机**|只有在被读取时才会计算（惰性执行）|数据变化时立即触发（也可配置为立即执行）|
- 如果只是想对现有的数据进行**加工、转换、筛选**，并希望结果能自动更新，选 **`computed`**。
- 如果需要在某个值变化时**去干点别的事情**（比如发请求、打日志、弹窗、操作 DOM），选 **`watch`**。

#### 5️⃣JS 数据类型
8 种
基础数据类型（原始类型）和引用数据类型
原始类型：null、undefined、number、string、boolean、Symbol、BitInt
引用类型：对象

```JavaScript
// 基本类型
console.log(typeof 42);           // "number"
console.log(typeof 'Hello');       // "string"
console.log(typeof true);          // "boolean"
console.log(typeof undefined);     // "undefined"
console.log(typeof Symbol('id'));  // "symbol"
console.log(typeof 10n);           // "bigint"

// 引用类型 & 陷阱
console.log(typeof { a: 1 });      // "object"
console.log(typeof [1, 2]);        // "object" (无法区分数组和对象)
console.log(typeof null);          // "object" (这是 JS 历史悠久的 Bug)
console.log(typeof function(){});  // "function" (函数也是对象，但 typeof 做了特殊处理)
```

| **特性**   | **基本数据类型 (Primitive)** | **引用数据类型 (Reference)** |
| -------- | ---------------------- | ---------------------- |
| **存储位置** | 栈（Stack）               | 堆（Heap）                |
| **访问方式** | 按值访问                   | 按引用访问（通过地址）            |
| **不可变性** | 值不可变（只能替换整个值）          | 属性可变（可以修改对象内部）         |
| **赋值表现** | 复制值，两个变量互不影响           | 复制地址，修改其中一个会影响另一个      |
