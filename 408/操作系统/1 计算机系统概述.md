## 1 .0 一些概念
### 程序🆚作业🆚进程
> [!summary]
> - **菜谱**（固定不变） → **程序**
> - **你下的订单** → **作业**
> - **厨房正在炒这份菜** → **进程**


一个作业有多个进程，一个进程一般术语一个作业
高级调度（作业调度）
低级调度（进程调度）
#### 作业（Job）
作业是用户提交给计算机的一项任务，是操作系统进行高级调度的对象
#### 进程（Process）
进程是程序的一次执行过程，是系统进行资源分配和 **CPU 调度的基本单位**

### 系统命令🆚系统调用
> [!summary]
> **系统命令**是给**用户**使用的；**系统调用**是给**程序**使用的
> 很多**系统命令**最终都会**调用一个或多个系统调用**来完成真正的工作

![[ChatGPT Image 2026年7月7日 20_39_45.png|300]]
#### 系统命令
用户与操作系统交互的一种方式，本质上是一个用户程序，用于完成某项功能。例如用户在终端输入的一条命令
例如Linux：
```linux
ls
```

```linux
cp a.txt b.txt
```

```linux
mkdir test
```
Windows：
```
dir
```

```
copy
```

```
del
```

这些都是**系统命令** 它们本质上都是**一个普通用户程序**
例如Linux：
```
ls
```
实际上就是：
```
/bin/ls
```
`ls` 就是一个普通的可执行程序，它的作用是"把指定目录中的文件列表显示出来"
`ls` 必须调用：
```C
opendir()
readdir()
closedir()
```
这些系统调用（或库函数最终封装的系统调用），因为硬盘属于受保护资源，只有操作系统内核才能访问
#### 系统调用
系统调用是用户程序**进入操作系统内核**的一种接口
系统调用通常是由陷入指令（Trap Instruction，也叫软中断指令、System Call Instruction）来完成用户态到内核态切换的。
系统调用是**用户程序请求操作~={red}系统提供服务**=~的**接口**
例如C 语言

```C
read(fd, buf, 100);
```

这里 `read()`就是系统调用
再例如：
```
write()
```

```
fork()
```

```
execve()
```

```
open()
```
这些都是系统调用，它们**最终都要进入内核态**，由操作系统真正执行
### 系统服务🆚系统资源
> [!summary]
> 系统资源是"东西"，系统服务是"操作系统提供的功能"
>  **系统资源 = 操作对象**
> **系统服务 = 对资源进行管理和操作的方法**
> 
> 资源是"东西"，服务是"管理东西的方法"，系统调用是"请求这些方法的入口"。

![[截屏2026-07-07 20.49.33.png|200]] 
#### 系统资源（Resource）
系统资源就是**计算机中所有可以被程序使用和竞争的资源**
例如：CPU 、内存、磁盘、打印机、网卡、文件、信号量
资源就是**可以被分配、共享、回收的对象。**
#### 系统服务（Service）
系统服务就是**操作系统向用户程序提供的各种功能**
系统服务的目的，就是帮助程序安全、合理地使用系统资源。
例如程序想读文件，操作系统提供：
```C
read()
```
程序想创建进程，操作系统提供：
```C
fork()
```
程序想申请内存，操作系统提供：
```C
malloc()
```
（底层最终会调用相关系统调用。）
这些就是**系统服务**

### 操作系统特征‼️选择题爱考

#### 四大基本特征
![[截屏2026-07-07 21.47.13.png]]
1. 并发性: **宏观上同时微观上交替**
    - 并发：宏观（人类看到的）同时、微观（计算机看到的）交替（如 QQ 和微信交替运行）**同一时间间隔**发生多个事件
    - 并行：真正同时执行（需区分概念）**同一时刻**发生多个事件
2. 共享性: 
	1. 互斥共享：资源独占使用（如打印机）
		- 比如有一个进程 A 想要访问某个资源，则需要先提出请求，若资源是空闲的系统就可以把资源分配给进程 A 使用；之后有进程 B 也想要访问者个资源，但是 A 没用完，新进程就需要在旁边等待，只有 A 访问完这个资源将其释放之后，B 进程才能使用这个资源。
		- 两个进程不能同时访问，一个在访问另一个就得等待
    2. 同时共享：
	    1. 宏观同时微观交替访问
		    - 这类资源允许多个进程访问，但是访问要求是交替进行的，在用户看来是同时进行
		2. **可重入代码**编写的文件
			- 这类资源真的可以同时被访问，可以有多个进程同时访问可重入代码编写的文件里的内容，⚠️只是访问只是读取，不是往里面写
3. 虚拟性:
    1. 时分复用：提高时间利用率（CPU 时间片）
		- 在某个设备为一个用户提供服务时，若某个时刻用户进程空闲就可以转去为其他用户提供服务
    2. 空分复用：提高空间利用率（内存共享）
		- 利用存储器的空闲空间来存放其他进程，以提高内存的空间利用率。即有一个内存空间为某个进程 A 服务，但是进程 A 某个时刻空闲了，进程 A 没有执行这块空间，这块**空间是空闲的**里面没有东西，就**可以暂时借给其他进程使用、存储**
4. 异步性:
    - 进程执行"走走停停"，速度不可预知
    - 由于资源校址，进程是并发执行的，所以进程需要等待资源 A 使用资源 A，再等待资源 B 使用资源 B ... 不是一气呵成的
#### 特征间关系 
- 并发与共享是对 os 来说最基本的两个特征
- **并发与共享互为对方的存在条件**：
    - 资源共享是以进程的并发作为执行条件的（若系统不允许并发执行就不存在资源共享了）
	    - 教材里的"资源共享"指的是操作系统中的"并发共享"，不是一般意义上的"大家都能用同一个资源"。
	    - 并行叫资源分配，资源共享面临多个程序竞争资源
    - 有效资源共享管理保障并发执行（若系统不能**对资源共享实施有效管理**必然会影响各个进程的并发执行，设置根本没法并发执行）
- 记忆要点：这四个特征是选择题高频考点，需重点掌握区别与联系

需要重点理解的是各类型的操作系统主要想解决的是什么问题，是用什么样的方式来解决的。

### 人工操作方式
没有 OS，使用计算机是人先将已经打孔的纸带/卡片输入计算机，手动启动，计算机才能识别纸带上打的孔，以运行这个程序
![[Pasted image 20260707215040.png]]
- 操作流程：用户将穿孔纸带/卡片装入输入机，手动启动设备将程序和数据输入计算机后启动运行
- 主要特点：完全依赖人工操作，无操作系统参与
- 纸带原理：长方形纸条通过打孔表示二进制数据（0/1），每个孔代表一个字符
- 缺点：
	- 第一个程序员在自己的程序没有完全运行结束之前不允许第二个程序员的程序同时使用计算机
	- 人机速度矛盾，资源特别是计算机 CPU 利用率极低，计算机这种昂贵资源大量时间空闲
#### 脱机 IO 方式
与人工操作方式在同一时期，是对人工操作方式的改进，缓解人机速度矛盾
各个程序员可以将自己的程序依次放到纸带机上，通过外围机（外围控制机）专门控制将纸带机的程序、数据先放到磁带上，之后计算机可以直接从这个磁带里读取这些程序的数据，而磁带的读写速度要比纸带机快很多。![[截屏2026-07-07 21.58.24.png]]
改进人工操作方式的人机矛盾，先讲装有程序（代码和数据）的纸带先放到输入机里去，由**外围机控制**计算机来读取这个纸带上的数据，人不用在旁边守着了
[chatgpt]人工操作需要人工换纸带，CPU 得等人换纸带，而 CPU 资源很贵，故想办法 CPU 不需要读纸带
-->用外围机读纸带，专门负责输入输出
可以理解成今天电脑里的USB 控制器、硬盘控制器、 DMA 控制器，这些都是外围控制设备
纸带 ➡️  外围机读取  ➡️  先放到缓冲区  ➡️  再交给 CPU
### 批处理系统
> [!warning]
> 多道程序的运行环境比单道程序的运行环境更复杂

批处理阶段操作系统正式诞生
#### 单道批处理系统
计算机中运行着一个叫做~={red}**监督程序**=~的程序，由这个监督程序控制着自动的从这些磁带当中输入输出作业的数据。而从磁带中输入输出作业其实是速度是要更快的，引入了脱机 IO 技术之后，读取一个作业所需要花的时间比例就小了很多了
这个监督程序会控制着计算机自动的从磁带当中读入第一个作业（PS：这个读入的过程要比从纸袋机读入要快很多）
接下来会进行程序的运算
再接下来进行输出，输出到这个磁带上
等第一个作业完成之后就可以开始紧接着读入第二个作业，然后计算再输出...
以此类推，在引入了脱机 IO 技术之后，**作业的输入输出过程所需要占的时间比例降低了很多**，让 CPU 有更高比例的时间处于计算处于忙碌的状态
使 CPU 的资源利用率有一定的提升，缓解了一定程度上的人机速度矛盾。
但是这种方式很显然它的**资源利用率依然是不够高**的，内存中同一时刻只能有一道程序运行，只有这个程序运行结束并且输出完成之后才可以继续读入第二个程序，即**各个程序之间是串行执行**，所以虽然比起上一个阶段 CPU 利用率有所提升，但是这种方式依然会使 CPU 有大量的时间是在空闲等待的状态，资源利用率依然是很低的  
![[Pasted image 20260707221217.png]]
发明了**监督程序**，把所有需要完成的任务打到一个文件里，再将文件装到内存中，再由监督程序把文件所有程序一个一个督促计算机执行，执行一个访问下一个，再执行一个再访问下一个...
🤨单道批处理文件系统是由监督程序将作业送入 CPU 处理(送入内存 CPU 读内存)的
![[截屏2026-07-07 16.52.09.png]]
- 监督程序作用：
    - 将磁带上的作业顺序装入内存
    - 移交运行控制权给作业
    - 作业完成后收回控制权再加载下一个作业
- 典型缺点：
    1. 资源浪费：内存仅驻留一道程序，I/O 操作时 CPU 完全空闲（系统中的资源得不到到充分利用）
    2. 效率低下：I/O 操作计算时段占比小，大量时间处于等待状态
    3. I/O 瓶颈：低速设备显著降低 CPU 利用率

#### 多道批处理系统
同样用甘特图的方式来看一下和上一个阶段的区别。
![[截屏2026-07-07 22.10.36.png]]
首先第一个作业的数据会**从磁带输入到计算机**当中，等输入结束之后就可以开始对程序的计算工作。但是当 CPU 在计算第一个作业的时候，输入设备是处于空闲的状态的，**因此 CPU 计算第一个作业的时候可以让输入设备同时把第二个作业也读入内存**，紧接着第一个作业计算完成就可以使用输出设备，把这个结果输出到磁带上；而第一个作业在输出的时候只是输出设备在忙碌，在这个时间段内 CPU 已经空闲了，在这个时间段其实可以让 CPU 为第二个作业服务开始计算第二个作业；同样此时输入设备又空闲了，因此在这个时候又可以开始同时读入第三个作业...
总之在引入了多道程序技术之后，**多道程序可以~={red}并发=~执行共享计算机当中的资源**。所以资源利用率就得到了大幅的提升
从甘特图知引入了多道程序技术之后，CPU 是会有大量的时间处于忙碌的状态的，也就是说 CPU 的利用率得到了更大的提升。
🧚‍♀️流水线来了
准备很多作业放到外存上，形成一个后备队列，后备队列就是一个串，串上面放着很多个等待作业，根据一定调度算法（这个算法是在处理机管理中重点学习的内容）选择一个合理可以运行的作业进入内存然后上 CPU 运行
![[截屏2026-07-07 17.00.59.png]]
- 核心机制：
    - 后备队列：用户提交的作业在外存排队等待
    - 调度算法：**作业调动程序**按特定策略从后备队列选择若干作业调入内存，使得作业在管理程序的控制下并发执行，共享系统资源（处理机管理重点）
    - 并行处理：CPU 计算与 I/O 操作重叠（如处理 A 时加载 B）
    - 中断技术：当某道程序因 I/O 请求暂停运行时，CPU 立即切换到另一就绪程序继续执行，该机制依赖中断技术使得系统各部件尽可能保持忙碌，显著提高整体吞吐量和资源利用率
- 显著优势：
    1. 资源利用率：CPU 持续忙碌，内存和 I/O 设备利用率同步提升
		- 上图程序 ABCD 都是先请求再输入再上 CPU 运行；CPU 处理 A 的时候 B 在输入，CPU 处理 B 的时候 C 在输入... CPU 一直在干活，处理完 A 之后发现 B 准备好了再等待了，于是接着处理 B，接下来再处理 C，一直在干活没有空闲时间，CPU 利用率非常高
    2.  吞吐量大：减少作业切换开销（仅作业完成/出错时切换）
		- CPU 和其他资源（输入设备键盘输出设备打印机等）
		- 只有作业完成/出错时切换，所以系统开销小（切换就是上 CPU 和下 CPU 这个过程，切换上去和切换下来都会耗费大量的时间和空间，所以必须尽可能减小切换过程）
- 主要缺陷：
    1. 平均周转时间长：$周转时间=完成时间-输入到内存的时间$，作业需排队等待，通常耗时数小时至数天
    2. 交互缺失：交互即人和计算机进行交流，人输入一个指令计算机执行一个指令、人输入下一个指令计算机执行下一个指令... 多道批处理没有交互能力是因为将整个程序输入进去之后就不能再管了，CPU 一个一个执行程序 A、B、C、D...作业提交后无法中途干预（调试不便）

#### 分时系统
**To实现人机交互的功能，人们又发明了分时操作系统**
在分时操作系统当中计算机会**以时间片为单位轮流**的为各个用户或者说各个作业服务；用户可以通过像**键盘、鼠标等等这一类终端**和计算机进行交互。
比如说这个计算机先给 A用户服务 50 毫秒，再给 B用户 50 毫秒，C用户 50 毫秒，总之**在分时操作系统当中，各个用户可以轮流的得到服务**。
分时操作系统的主要优点就是用户的请求可以被及时的响应，分时操作系统也存在一个明显的缺点：就是不能优先处理一些紧急的任务。**这种操作系统对各个用户都是完全公平的**
![[截屏2026-07-07 22.17.56.png]]
有没有发现我们之前举的老查约会的例子，其实它就是一个分时操作系统。老查以时间片为单位，轮流的为各个用户来服务，这样的话，各个用户的请求就可以被及时的响应，因为对于他们俩来说。
把 CPU 运行时间分成几个很小的时间片（例如 2 s）第一个进程访问 2 s 第二个进程访问 2 s... 再轮换回去，进程是交替轮换使用 CPU 的  
##### 分时系统特征
1. 多路性：单主机服务多终端用户（如学校机房）
	- 允许多个用户同时使用一台主机

3. 及时性：响应时间符合人类感知（2-4 秒）在人所能接受的时间里得到计算机回复
4. 交互性：支持指令→响应→新指令的对话模式，即人可以输入一个指令计算机完成之后回复一下再输入一个新的指令

#### 特权指令🆚非特权指令 
1. 特权指令特点：可无限制访问内存空间（包括用户空间和系统空间），能直接管理系统硬件和软件资源。🧑‍🏫OS 执行的，可以管理整个程序、计算机，不仅能访问用户空间还能访问系统空间
2. 非特权指令限制：仅能完成一般性操作，不能直接访问硬件/软件，内存访问范围限于用户空间。🧑‍🏫在用户态下所能执行的指令，一般只能完成很简单的操作，**不能访问计算机的硬件/软件**，即不透明的球不能访问，只能调用它
- 核心记忆点：用户态对应非特权指令（用户程序使用），内核态对应特权指令（操作系统使用），这个对应关系必须牢记。

##### 内中断
1️⃣2️⃣都是异常陷入 Exception
1️⃣很典型的内中断的例子：发生了非法的事情之后OS重新夺回了CPU的控制权。
CPU在执行当前的一条指令的时候，由这条指令引发了一个中断。
2️⃣有的时候即使CPU执行的是非特权指令也有可能会引发内中断：CPU执行除法指令，除数是零
3️⃣陷入指令（软中断指令）：
🤨应用程序自发中断
运行在用户态的一个应用程序有时是想要请求操作系统内核为应用程序提供一些服务，此时应用程序会执行一条**特殊的指令——陷入指令**，这个指令会引发内部中断信号，接下来CPU会转向这个中断信号，相应的处理程序进行这个中断信号的处理。
当一个应用程序执行了陷入指令意味着该应用程序主动的把CPU使用权还给 OS内核。想让操作系统内核为它提供某些服务
**~={red}系统调用=~是用陷入指令来完成的**，陷入指令也叫访管指令
‼️**陷入指令**是一个比较特殊的指令，但是它**并不是特权指令。**
🧑‍🏫指令本身非法或指令参数非法会产生内中断信号
🤨非法信号触发中断信号，CPU要转换成内核态去处理中断信号相关的内核程序
🧚‍♀️throw new runtimeexception()；ZeroDivisionError（；上溢；catch，然后抛出这个异常；处理中断对应的中断程序，执行完毕后，在返回原来执行的程序；有没有一个程序：在触发异常的时候不触发中断; trap; system call; 自陷；缺页陷：我那么大块内存呢？内核你快给我啊；**陷入程序：应用程序主动发出中断信号**；陷入指令，完成系统调用；这个是不是计组里面的自陷啊；陷入指令是一种具体的指令。自陷是内部中断的一个种类。两者完全不一样；syscall 就是陷入指令；主动将 cpu 还给操作系统；计组 249 页说了自陷指令；晶振？；故障是被动的。自陷是主动的
##### 外中断
两个应用程序在中断机制的支持下实现并发运行，一直不断相互切换
🧚‍♀️同步定时方式；时钟中断；中断查询方式和总线事务的同步定时方式；因为 cpu 会自己判断是否继续给程序 1 分配时间片，符合条件就分配；也是时分复用，CPU 的多核是类似频分服用；dma 方式里面的周期窃取；CPU 通常在每条指令执行结束后，会检查是否有可响应的中断请求，如果有，就在下一条指令开始之前响应中断；
#### 中断与异常（陷阱）定义
当 CPU 无法继续执行时触发的事件，分为硬件中断（如设备信号）和软件陷阱/异常（如除零错误）
🧑‍🏫中断与异常事件：CPU 执行不下去了，不能完成后续的功能了，需要停下去处理一些东西：比如报错，错误分为终端和陷阱；或者源于用户的特定请求：访问计算机硬件中的某些功能（比如使用打印机或者使用内存中的某一块），因为**用户态不能访问硬件**，涉及到这种请求得切换到核心态，由 OS 完成
##### 典型异常案例 
- 运算异常：2÷0 等非法运算
- 内存异常：访问已删除文件
- 特权请求：用户程序申请使用打印机等硬件
#### 处理流程 
比如某时刻 CPU 执行不下去了，CPU停止了，之后就会发出一个中断请求（异常请求）给操作系统，CPU 从用户态切换到核心态，由操作系统处理异常，改正之后将改正完的信息返回回来，从核心态切换回用户态，继续再上 CPU 执行接下来没有完成的任务
- CPU 停止当前任务
- 用户态→内核态切换（关键考点）
- 操作系统处理异常
- 内核态→用户态切换（关键考点）
- 继续原任务

核心记忆点：必须记住"用户态→内核态→用户态"的双向切换过程，这是高频考点。

- 定义：从硬件启动到操作系统内核运行的完整过程，核心是引导程序将操作系统从存储介质（如硬盘）加载到内存并启动运行。
- 三大功能：
    - 硬件初始化：检测 CPU/内存/磁盘等硬件，配置中断向量和内存，故障时会发出报警
    - 内核加载：从硬盘/U 盘等存储介质读取操作系统核心代码到内存
    - 控制权转移：将计算机控制权交给操作系统内核完成后续初始化

    - 硬件检测（CPU/内存/磁盘等基础硬件状态检查）
    - 加载引导程序（从硬盘激活引导程序）
    - 加载操作系统内核（读取内核可执行文件到内存）
    - 启动操作系统（内核初始化进程、驱动，最终显示用户界面）

##### 6. 习题讲解﻿42:55﻿

###### 1）系统调用接口题﻿42:59﻿

- 核心考点：用户态到核心态的切换机制
- 关键答案：操作系统通过系统调用为应用程序提供硬件访问接口

###### 2）操作系统功能题﻿43:33﻿

- 四大功能体系：
    - 处理机管理（CPU 资源调度）
    - 存储器管理（内存分配与回收）
    - I/O 设备管理（输入输出设备控制）
    - 文件管理（外存数据组织）
