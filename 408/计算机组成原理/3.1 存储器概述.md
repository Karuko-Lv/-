## 主存的基本结构
主存基本硬件组成：存储体（有很多存储单元组成，每个存储单元由若干个存储元件组成，每个存储元件可以存储一个二进制数 0 或 1，每个存储单元可以存储一串二进制信息，这串二进制信息 called 存储字，这串二进制信息的位数 called 存储字长，可以是 8 位、16 位、32 位等）、MDR、MAR
MDR 用来存放从存储体某个存储单元取出的信息或者准备往某个存储单元存入的信息，因此 MDR 位数应该与存储字长相等
MAR 用来存放预防问的存储单元的地址，当 CPU 要访问主存中的某个存储单元时，首先要将存储单元的地址送到MAR 中，所以 MAR 的位数决定了存储单元的数量，具体为 $2^{MAR}$ 个
🧚‍♀️这是按字编址的时候吗？按字节编址是不是就不是了🤨这里不就是按照字节的吗
Q：为什么将带访问的存储单元的地址写入 MAR 中就可以找到存储体中具体的存储单元了呢
🧚‍♀️mdr 位数与数据线位数相等；4 k 对齐？；一套房（存储空间）里有 8 位空间，一共 8 套房，只用 0、1 给房编号需要三位（2^3=8）![[截屏2026-06-12 17.56.53.png]]
‼️地址译码器本例中是典型的3-8 译码器
地址译码器根据移码规则将 3 位输入信号译码成相应的 8 位输出信号，仅有一位不同，这一位信号 called 有效信号，用于选中相应的存储单元![[截屏2026-06-12 18.00.59.png]]
🧑‍🏫理解地址译码器作用即可，不用掌握利用逻辑门构建译码器
‼️存储器地址寄存器 MAR 内容由 CPU 通过**地址总线**送入；存储器数据寄存器 MDR 用于暂存 CPU 通过**数据总线**送来的待写入存储体中某个存储单元的信息，或者暂存从存储体中某个存储单元读出的信息
上述操作以及地址译码器的译码都需要在 CPU 控制下进行![[截屏2026-06-12 18.04.17.png]]
犹豫控制信号一般不止一个，所以将传输各种控制信号的传输线统称为**控制总线**（本节课不区分都有哪些控制信号线）

### 双译码结构 
大量存储单元排列成二维阵列的形式，使用行地址译码器的各条译码输出线连接相应行中的全部存储单元；用列地址译码器的各条译码输出线连接相应列中的全部存储单元 ![[截屏2026-06-12 18.26.53.png|350]]
若行地址译码器有 $\frac{n}{2}$ 条地址输入线，则行地址译码器就有 $2^{\frac{n}{2}}$ 条译码输出线，可以寻址 $2^{\frac{n}{2}}$ 行存储单元；若列地址译码器有 $\frac{n}{2}$ 条地址输入线，则列地址译码器就有 $2^{\frac{n}{2}}$ 条译码输出线，可以寻址 $2^{\frac{n}{2}}$ 列存储单元
==>存储器译码输出线的总数： $2^{\frac{n}{2}}+2^{\frac{n}{2}}=2 \times 2^{\frac{n}{2}}$ ;该存储器的存储容量 $2^{\frac{n}{2}} \times 2^{\frac{n}{2}}=2^{n}$
![[截屏2026-06-12 20.35.46.png]]
