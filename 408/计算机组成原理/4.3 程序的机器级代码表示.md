##### 算术运算
🧑‍🏫对 d 和 s 两个操作数进行相应处理之后**最终运算结果会被放回 d 所指的位置**，意味着 d 字母所代表的目的操作数只有可能来自于寄存器或者主存，不可能是一个常量，只有寄存器或者某个主存地址才可能存放结果
🧚‍♀️常量就是立即数
![[截屏2026-06-26 22.25.53.png]]
🧑‍🏫在进行**除法运算**之前需要将被除数进行位扩展，例如 32 位的被除数/32 位的除数，需要将被除数扩展为 64 位，用 64 位的被除数除以 32 位的除数。而存放 64 位的被除数需要两个寄存器，用两个寄存器连起来用，更高的 32 位在 edx 更低的 32 位存放在 eax，除法运算的结果得到的商会存放在 eax 寄存器中，除法的余数会存放在 edx 寄存器中，故 `div s` 中的唯一一个操作数是除数，被除数是被提前安排到 edx 和 eax 两个寄存器中
🧚‍♀️被除数位扩展原因：对于两个 n 位补码整数的除法，需要先对被除数进行符号扩展到 2 n 位，使得得到的商也是一个 n 位补码整数。；s 是除数，被除数采用了隐含寻址，已经被放在 edx 和 eax 两个寄存器；mq 最后存放运算结果商 acc 存余数
![[截屏2026-06-26 22.28.00.png]]
🧑‍🏫 `<reg>` 表示操作数是寄存器中的操作数；`<mem>` 表示操作数是内存中的操作数；`<con>` 表示是常数操作数。👆第一个语句代表两个寄存器中的数相加结果放回第一个寄存器当中；第二个语句表示操作数可以分别来自寄存器和主存；最后一句表示加法指令操作数可以分别来自内存和常数

#### Cmp 指令硬件实现
![[截屏2026-06-27 17.39.38.png]]


🧑‍🏫CPU 执行 `cmp a,b` 指令背后完成的是：让 alu 计算 a 和 b 两个数相减
减法运算会产生几个的标志位信息：
1️⃣OF 判断运算结果是否溢出 OF=1 则溢出
2️⃣SF 判断运算结果是正的还是负数 SF=1 则负数
3️⃣ZF 判断运算结果是否为 0 ZF=1 则为 0
4️⃣CF
标志位如何确定和产生在第二章，产生的标志位会被自动存放到 CPU 内部特殊的寄存器中，**psw 程序状态字 Program Status Word寄存器（intel called 标志寄存器）**
例如 intel 8086 的 CPU 中有一个 16 bit 的标志寄存器，ALU 每进行一次运算之后都会产生相应的标志位，无论是加法减法乘法除法运算，每次运算之后的标志信息都会被自动的存储到标志寄存器中，每进行一次运算会把上次产生的标志信息覆盖掉，每进行一条运算指令之后根据寄存器里面相应的标志位判断刚才的运算结果是什么
`cmp` 指令后面通常会跟着一个条件转移指令，CPU 根据刚才运算结果的标志位判断
🧚‍♀️我们看语句逻辑 cpu 看 psw 中的标志位；做减法但是不保存，只保存标志位；`SF==OF` 等同于同或即相同为 1
> [!warning] `CMP` 本质上就是"不保存结果的 `SUB`"，但 `SUB` 会修改第一个操作数，而 `CMP` 不会。
> CMP = SUB - 保存结果

### 循环语句机器级表达
使用条件转移指令可以实现循环![[截屏2026-06-27 17.52.07.png]]
🧚‍♀️理解成 `if break` 就可以了![[截屏2026-06-27 18.03.17.png]]
```
mov eax, 0      ; result = 0
mov edx, 1      ; i = 1

L1:
add eax, edx    ; result += i
inc edx         ; i++

cmp edx, 100
jle L1          ; 如果 i <= 100，继续循环
```
它先执行循环体，再判断是否继续，所以更像：

```C
do {
    result += i;
    i++;
} while (i <= 100);
```

对于本题因为 `i` 初始值是 1，且一定满足 `i<=100`，所以结果完全正确。

### 函数调用机器级表示
#### 高级语言的函数调用
##### Call & ret 指令
程序运行系统会分配一片内存区域，这一片区域 called 函数调用栈，专门用来实现函数调用的内存区域
每个函数栈帧主要包含 1️⃣局部变量 2️⃣函数调用相关信息
1. 每个程序都是从 main 函数出发先执行 main 函数，执行 main 函数的时候函数调用栈有两块数据：1️⃣存储 main 函数的栈帧 2️⃣栈底的位置保存一些和硬件及系统相关的其他信息
2. `main`  函数中调用 P函数, 故程序开始执行 P这些代码。当 P 函数执行时栈顶有和 P 函数相关的栈帧
3. P 函数调用 Q 函数，Q 函数相关的信息压入栈顶，Q 函数返回之后从栈顶删除 Q 得栈帧
4. 调用 caller 函数同样将 caller 函数栈帧压入栈顶，caller 函数在调用 add 函数的时候需要传递两个参数 `temp1` 、`temp2`, 这两个函数调用的参数同样放在 caller 栈帧中的
5. 执行 add 函数，栈帧在栈顶位置，当 add 函数执行完毕返回之后 add 函数栈帧会出栈 
![[截屏2026-06-27 18.29.59.png]]
🧚‍♀️这就是为什么写的无限递归 "stack overflow"；递归最好理解，就是时间复杂度高；没有递归，只是递归也是这个逻辑，或者说递归其实说的也就是函数调用的过程；也就是函数相互调用，没有递归啊。递归是自己调用自己
![[截屏2026-06-27 20.29.45.png]]
🧑‍🏫使用 `call` 和 `ret` 这两条指令实现了函数的调用与返回
通常在汇编语言用一个标号标记一个函数的开始，add 加一个冒号表示接下来的指令是add 函数；caller 加一个冒号表示接下来的指令是 caller 函数
 `call` 和 `ret` 指令都会使得程序的执行流发生改变，故这两条指令会影响 PC 寄存器的值
 函数调用发生时需要确保：
 1️⃣call 指令执行后得过得去，即 IP 寄存器得指向正确的位置
 2️⃣过去之后要能回来，执行 return 指令之后需要将 IP 寄存器回到 call 指令的后一个指令![[截屏2026-06-27 20.47.40.png]]
 🧚‍♀️对对对，保存作案现场，和恢复现场，很形象的比喻；总结：call 保存 pc 值后跳转，ret 从栈中取出 pc 值后，放回 pc ![[截屏2026-06-27 20.48.32.png]]

##### 标志寄存器 Flag Register

FLAGS / EFLAGS（x86）是程序状态字寄存器 PSW 的核心载体，每一位都是一个独立的状态触发器，按功能分为**状态标志**和**控制标志**两类。

> [!NOTE] 考试重点
> 重点关注 CF、ZF、SF、OF 这四个状态标志，以及它们在不同指令（ADD / SUB / CMP / SHL / SHR 等）下的变化规律。OF 的判断（双高位法 / 符号判断法）是高频考点。

###### 状态标志（Status Flags）

状态标志反映**最近一次算术/逻辑运算结果**的特征，是条件转移指令（Jcc）的判定依据。

| 标志 | 全称 | 位置 | 含义 | 置 1 典型场景 |
|:---:|:---|:---:|:---|:---|
| **CF** | Carry Flag | bit 0 | 进位 / 借位标志 | 无符号数加法最高位有进位，或减法有借位 |
| **PF** | Parity Flag | bit 2 | 奇偶标志 | 结果低 8 位中 1 的个数为偶数 |
| **AF** | Auxiliary Carry Flag | bit 4 | 辅助进位标志 | 低 4 位向高 4 位有进位/借位（BCD 码用） |
| **ZF** | Zero Flag | bit 6 | 零标志 | 运算结果为 0 |
| **SF** | Sign Flag | bit 7 | 符号标志 | 结果为负数（最高位 MSB = 1） |
| **OF** | Overflow Flag | bit 11 | 溢出标志 | **有符号数**运算结果超出表示范围 |

**OF 的两种判断方法（重要）：**

1. **双高位法**：$OF = C_n \oplus C_{n-1}$，即最高位进位与次高位进位异或。若只有其中一个产生进位，则溢出。
2. **符号判断法**：两个同号数相加得异号，或两个异号数相减得与减数同号 → 溢出。

> [!example] 例题
> 8 位补码运算：$120 + 100 = 220$（超出 [-128, 127]）
> - 0111 1000 + 0110 0100 = 1101 1100
> - $C_7 = 0$，$C_6 = 1$，$OF = 0 \oplus 1 = 1$ → **溢出**
> - 同时 SF = 1（结果为负），CF = 0

**CMP 指令与标志位的关系（无符号数）：**

| 比较关系 | 标志位状态 |
|:---:|:---|
| dest = src | ZF = 1 |
| dest < src | CF = 1（不够减，有借位） |
| dest > src | ZF = 0 且 CF = 0 |

**各指令对标志位的影响总结：**

| 指令 | OF | SF | ZF | CF |
|:---|:---:|:---:|:---:|:---:|
| ADD / ADC | ✓ | ✓ | ✓ | ✓ |
| SUB / CMP / SBB | ✓ | ✓ | ✓ | ✓ |
| INC / DEC | ✓ | ✓ | ✓ | ✗（不影响 CF） |
| MUL（无符号） | ✓ | ✗ | ✗ | ✓ |
| IMUL（有符号） | ✓ | ✗ | ✗ | ✓ |
| AND / OR / XOR / TEST | 0 | ✓ | ✓ | 0 |
| NOT / NEG | ✗ | ✓ | ✓ | ✓ |
| SHL / SAL（左移 1 位） | ✓ | ✓ | ✓ | ✓ |
| SHR（逻辑右移） | ✗ | ✓ | ✓ | ✓（移出的末位） |
| SAR（算术右移） | ✗ | ✓ | ✓ | ✓（移出的末位） |

> [!warning] 易错
> - **INC / DEC 不影响 CF**，但影响 OF / SF / ZF。这是常考点。
> - **逻辑运算（AND/OR/XOR）必定使 OF = 0、CF = 0**。
> - 移位指令的 OF 仅在移动 1 位时有效（反映符号位是否改变）；移多位时 OF 无定义。

###### 控制标志（Control Flags）

控制标志由**程序员显式设置**，用于控制 CPU 的某些行为模式。

| 标志 | 全称 | 位置 | 含义 | 说明 |
|:---:|:---|:---:|:---|:---|
| **DF** | Direction Flag | bit 10 | 方向标志 | 控制串操作指令中 SI/DI 的自增/自减方向。CLD 清 0（自增），STD 置 1（自减） |
| **IF** | Interrupt Flag | bit 9 | 中断允许标志 | IF = 1 允许响应可屏蔽中断（STI 置 1，CLI 清 0） |
| **TF** | Trap Flag | bit 8 | 陷阱标志 | TF = 1 使 CPU 进入单步执行模式（调试用），每执行一条指令后产生 1 号中断 |

> [!summary] 记忆口诀
> - **状态标志看结果**：CF 无符号进位，OF 有符号溢出，ZF 判零，SF 看符号
> - **控制标志看行为**：DF 管方向，IF 管中断，TF 管单步
