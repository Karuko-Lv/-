 **EAX、EBX、ECX……这些名字并不是随便起的，而是历史发展的结果。**

如果理解了它们的历史，以后学 x 86 汇编会轻松很多。

---

# 第一代：AX、BX、CX、DX（16 位时代）

最早的 Intel 8086（1978 年）是 **16 位 CPU**。

它只有几个主要寄存器：

|名称|全称|最初用途|
|---|---|---|
|AX|Accumulator|累加器（算术运算）|
|BX|Base|基址寄存器|
|CX|Count|计数寄存器|
|DX|Data|数据寄存器|

所以：
```
AX = Accumulator Register（累加器）
BX = Base Register（基址寄存器）
CX = Count Register（计数器）
DX = Data Register（数据寄存器）
```
例如：
```asm
MOV AX, 5
ADD AX, BX
```
这里：
```
AX
```
就是：
> **Accumulator（累加器）**

因为很多加减乘除指令默认都使用 AX。

---

# 为什么叫 BX？

BX 最初设计出来主要用于：
```
Base Address
```
例如：
```asm
MOV AX, [BX]
```
意思：
```
访问 BX 指向的内存
```
所以：
```
BX = Base Register
```

---

# 为什么叫 CX？

很多循环
```asm
LOOP
REP MOVSB
```
默认使用
```
CX
```
作为 Count
即：
```
循环次数
```
例如：
```asm
MOV CX,100
LOOP again
```

---

# 为什么叫 DX？
DX：
```
Data Register
```
最初用于：
```
乘法
除法

I/O端口
```
例如8086：

```
DX:AX
```
共同组成：
```
32位乘法结果
```

---

# 第二代：EAX、EBX……

后来到了：
```
80386
```
Intel从
```
16位
↓
32位
```
怎么办？

Intel 希望：
> **老程序还能运行。**

于是没有重新命名。
而是在前面加了 E
表示 Extended，即扩展版
于是：
```
AX
↓
EAX
```
意思
```
Extended Accumulator
```
同样：
```
BX
↓
EBX
```
就是
```
Extended Base
```

---

所以：

|16 位|32 位|
|---|---|
|AX|EAX|
|BX|EBX|
|CX|ECX|
|DX|EDX|

# 第三代：RAX、RBX……

后来：

AMD 发明 64 位。
Intel 跟进。
于是再扩展：
```
64位
```
名字变成：
```
RAX
```
其中
```
R
```
表示
```
Register
```
或者理解成
```
64-bit Register
```
于是：
```
AX（16）
↓
EAX（32）
↓
RAX（64）
```

# 它们其实是同一个寄存器！

例如：

```
RAX (64位)

┌────────────────────────────┐
│                            │
└────────────────────────────┘
               ↑
             EAX(32位)

          ↑
        AX(16位)

     ↑        ↑
    AH       AL
```

可以画成：

```
RAX
┌──────────────────────────────┐
│                              │
│      EAX                     │
│   ┌──────────────┐           │
│   │      AX      │           │
│   │ ┌──┬──┐      │           │
│   │ │AH│AL│      │           │
│   │ └──┴──┘      │           │
│   └──────────────┘           │
└──────────────────────────────┘
```

例如：

```
AL
```

只有：

```
8bit
```

而：

```
AX
```

就是：

```
16bit
```

# 为什么现代教材还喜欢叫 EAX？

因为很多教材：
```
组成原理
操作系统
```
都喜欢用：
```
IA-32
```
（32 位 x 86）
所以教材大量出现：
```
EAX
EBX
ECX
```
实际上现代电脑（包括你的 Mac，如果运行 x 86 程序）更多看到的是：
```
RAX
RBX
RCX
```
因为现在几乎都是 64 位了

## 一句话记忆
这些名字都有含义，不是随机起的：

|寄存器|含义|最初用途|
|---|---|---|
|**AX / EAX / RAX**|**Accumulator（累加器）**|算术运算默认寄存器|
|**BX / EBX / RBX**|**Base（基址）**|存放基地址、内存寻址|
|**CX / ECX / RCX**|**Count（计数）**|循环计数、字符串操作|
|**DX / EDX / RDX**|**Data（数据）**|乘除法、高位数据、I/O|

所以，它们本质上都是**通用寄存器**，只是名字保留了早期 CPU 为它们设计的"默认用途"。现代 x 86 中，这些寄存器大多都可以作为通用寄存器自由使用
