### double->float
![[截屏2026-06-06 20.44.41.png]]
`float`表示范围小，大数值转换可能发生溢出
`float`尾数有效位变少，高精度转换还会发生舍入误差
🧚‍♀️因为float的24到31位是用来表示阶码的，所以没法全部表示

### float/double->int
![[截屏2026-06-06 20.48.21.png]]
`int`没有小数，所以小数part直接丢弃
浮点数超出`int`表示范围会发生整数溢出
PS：**在同样的 32 位（4 字节）硬件格子里，`float` 能表示的最大值，远远、疯狂地大于 `int` 的最大值。它们之间差了整整 29 个数量级！**
- **`int` 的最大值**：$\approx 2.14 \times 10^9$（约 21 亿，也就是我们常说的 `0x7FFFFFFF`）
- **`float` 的最大值**：$\approx 3.40 \times 10^{38}$（这是一个 39 位的超级巨无霸）
同样的 32 个物理格子， 内部结构不同所表达数的范围不一样

###  int->float
不会溢出但是`float`尾数(含隐藏位1.M)是24位 int是32位 
当整数二进制有效位超过24位需要做舍入处理，存在精度损失
![[截屏2026-06-06 20.52.01.png]]
即整数二进制24到31位不是非0的时候无法精确转换成24位浮点数的尾数，会发生精度溢出
🧚‍♀️因为浮点数表示法只有尾数部分表示精度，阶码的8位只是把尾数扩大或者缩小，而int里这8位也可以用来表示精度；“向0方向截断” 是指：直接舍弃小数部分，保留整数部分，且无论数值是正还是负，都朝着接近0的方向取整。；3.99 → 3、-3.99 → -3，这就是~={red}**向 0 截断，说白了就是朝着数轴 0 的位置截断**=~；高位非零比如就是32位，那么转换为小数全都占满了23位也装不下，但是高位是有八个零，转为浮点数就刚好转下了
「PS」普通的 32 位 `int` 如果转成 64 位的 `double`，精度是完美的（因为 `double` 有 52 位尾数，装得下 `int` 的 31 位数值）
```cpp
#include <stdio.h>

int main() {
    // 1. 声明一个大整数（处于int合法范围内）
    int original_int = 2147483645;
    
    // 2. 隐式类型转换：int -> float
    float converted_float = (float)original_int;
    
    // 3. 打印对比结果
    // %d 打印整数，%.2f 打印浮点数（保留0位小数）
    printf("原 int 值:   %d\n", original_int);
    printf("转 float 后: %.0f\n", converted_float);
    
    // 4. 验证在计算机眼里它们还相不相等
    if (original_int == (int)converted_float) {
        printf("结果：惊天奇迹，它们居然相等！\n");
    } else {
        printf("结果：惨烈翻车，数字变了！相差了 %d\n", 
                (int)converted_float - original_int);
    }
    
    return 0;
}
```

#### 🔍 `int` 的真实长相

`2147483645` 的二进制写满 32 位格子是：
`0111 1111 1111 1111 1111 1111 1111 1101`
这个二进制数，**足足有 31 位的有效数字**。
#### 🔍 `float` 过来强行装箱
当这个 `int` 试图住进 `float` 的宿舍时，`float` 拿出科学计数法的尺子一量：
“很好，符号位填 `0`。接下来把小数点左移 30 位进行规格化，写成：
$1.1111111111111111111111111111101 \times 2^{30}$”
这时候，`float` 翻开自己的物理规格大吃一惊：**天哪，我的尾数格子（M）只有 23 个！可这里有 30 位小数需要存！**
没办法，硬件只能施展无情的截断手术：
- 它把前 23 位小数死死保住。
- 把最后剩下的 7 位废品小数 `1111101` **直接切掉**！
- 切掉时，因为被切掉的部分非常接近 1，硬件执行了**就近舍入（Round to nearest, ties to even）**，相当于给第 23 位尾数强行**向前进了一位**。

经过这一轮悲惨的“整容”，原本的尾数被进位硬生生顶成了全 `0`，公式变成了：
$$\text{最终 float 存的值} = 1.00000000000000000000000 \times 2^{30} = 2^{30} = \mathbf{2147483648}$$
PS：每个编译器执行的舍入不一样，有的就是直接$1.11111111111111111111111 \times 2^{30} = \mathbf{2147483647}$ 相差2
