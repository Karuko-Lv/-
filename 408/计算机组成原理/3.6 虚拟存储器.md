### 地址映射和页表
 ![[截屏2026-06-24 16.05.46.png]]
 For 主存中的某个信息，用该信息所在的**主存数据块地址**以及**块内偏移**两部分描述信息在主存中的位置  ![[截屏2026-06-24 16.07.26.png]]
 对于 cache 中某个 cache 行，其数据块副本缓冲区用于存放一个主存数据块的副本；标记 tag 用于存放相应的主存数据块地址；有效位用于指示该 cache 行中的数据是否有效
 🧚‍♀️块内偏移相当于块内地址，数据块地址相当于主存区号
 🧑‍🏫访问主存中某个信息
 **通过该信息的主存地址在 cache 中进行快速查找**：
 即用该信息主存地址中的主存数据块地址与各 cache 行中的标记 tag 并发比较
 比较结果相等的 cache 行如果其有效位值为 1 则**命中**，即待访问信息位于该 cache 行中的数据块副本缓冲区；
 待访问信息主存地址中的**块内偏移的值**用于得**到待访问信息在数据块副本缓冲区的位置**

🤨虚拟页和辅存的关系

> ❌ **虚拟页不是辅存（磁盘）**
> ✅ **虚拟页是虚拟地址空间中的页**
> ✅ **辅存中存放的是虚拟页的后备副本（Backing Store）**

虚拟页 ≠ 辅存
虚拟页是程序能看到的页，它可能在主存或者在辅存

🧑‍🏫本例中相关进程的页表，当某个**程序**被**加载到主存中运行**时called 进程，**进程是程序的执行实例**，是**操作系统进行资源分配和调度的基本单位**（进程是资源分配的基本单位，而线程是调度的基本单位）。
在采用虚拟存储器技术的系统中，操作系统会**为每个进程创建一个页表**以将虚拟地址空间映射到**物理地址空间，即主存地址空间** 
页表当中的**每一行即每个页表项**包含有效位、脏位、替换控制位、访问权限位、禁止缓存位等字段记录一个虚拟页的相关信息
🧚‍♀️打三角洲就一直提醒我说虚拟内存不足；引入线程后进程是不是只是资源分配的基本单位，调度的基本单位是线程
#### 有效位
![[截屏2026-06-24 21.12.45.png]]
页表采用虚拟页号作为索引进行访问，1️⃣当某个页表项**有效位的取值为 1** 时，表示该页表项对应的**虚拟页从辅存调入主存**；相应这些页表项各自存放位置字段中存放的是物理页号，用于表明所对应的虚拟页调入了主存中的哪个物理页
2️⃣当某个页表项**有效位的取值为 0** 时，表明该页表项所对应的**虚拟页没有被调入主存**；若其**存放位置字段为 `null`** 表明其所对应的虚拟页为没有内容的空页，通常意味着**该页还没有被分配物理内存**，例如本例中的 $VP_0$ 和 $VP_4$; 若其**存放位置字段不为 `null`** ，存放位置字段的值为该页表项所对应虚拟页在辅存，例如磁盘上的起始位置，本例中的 $VP_1$ 和 $VP_6$
🧚‍♀️个人理解为没有创建进程（🤨页表有效位取值为 0 没有创建进程）

#### 脏位
![[截屏2026-06-24 21.19.12.png]]🧑‍🏫用来标注页表项对应的虚拟页是否被修改过
进行页面替换时要根据脏位判断是否需要将主存中被替换的页面写回辅存==>引出辅存中的**交换分区，to 存放主存中替换出的动态修改数据页面**；**数据分区 to存放用户程序和数据**

#### 替换控制位
指明页表项所对应的**虚拟页的使用情况**，配合具体的替换策略设置![[截屏2026-06-24 21.20.42.png]]
使用 FIFO 算法，该位 calledFIFO 位；使用 LRU算法，该位 calledLRU 位
🧚‍♀️和 cache 行结构类似？不过把 cache 的 Tag 和 Data 换成了这里的访问权限位、禁止缓存位和存放位置？
#### 访问权限位
指明页表项所对应的**虚拟页的访问权限**，用于存储保护
1️⃣可读可写
2️⃣只读
3️⃣只可执行![[截屏2026-06-24 21.25.23.png]]
存储保护是 os 的一项重要功能，确保不同进程在主存中地址空间相互隔离，**防止进程间相互越界访问**从而保护进程和数据不被非法访问和修改
🧚‍♀️有点像数据库的锁那味儿了；为什么能确保互相隔离？；基址寄存器记录进程起始地址＋界地址寄存器限制进程的长度，从而定义了进程的分区；想到 docker；操作系统有，这不是计组的事，上下限寄存器，或者基质寄存器和限长寄存器；操作系统上讲的有，就是有的指令要求必须执行完才能中断
#### 禁止缓存位
指明页表项对应的虚拟页是否可以载入 cache，正确设置禁止缓存位可以确保辅存、主存和 cache 的数据一致性

❓**思考在页式虚拟存储器系统中如何描述主存中某个位置的信息**
🧑‍🏫采用全相联映射页式虚拟存储器的系统中，用物理页号描述信息位于主存中的哪个物理页，用页内偏移描述信息在物理页中的具体位置
故采用页式虚拟存储器系统信息的主存地址构成，**~={red}主存地址 called 物理地址 too=~**，由物理页号和物理页内偏移两个部分构成
采用**页式虚拟存储器的系统**中 **CPU 给出的以及程序员面对的都是**主存和辅存构成的页式虚拟存储器中的**虚拟地址**，其构成有物理地址构成延伸得到
由于划分页面时**物理页和虚拟页的大小相同**，而物理页内的偏移字段和虚拟页内的偏移字段各自的尾数决定了页面的大小因此**二者位数相同**，故两者都可简称为页内偏移字段，不用区分物理页还是虚拟页
