# 4.3 程序的机器级代码表示

> 第 4 章核心考点（408 占 **81 分**）。理解高级语言如何被翻译为机器指令，以及常见汇编指令序列的含义。

---

## 一、从高级语言到机器指令

```
C/Java 源码 → 编译器 → 汇编代码 → 汇编器 → 机器指令（二进制）
```

408 考试中通常以 **x86（IA-32）汇编**（AT&T 或 Intel 语法）考查，偶尔涉及 MIPS。

| 语法风格 | 源 → 目的 | 立即数前缀 | 寄存器前缀 | 408 常用 |
|----------|-----------|-----------|-----------|:---:|
| **AT&T** | op src, dst | `$` | `%` | ⭐ 常见 |
| **Intel** | op dst, src | 无 | 无 | 也有 |

> 📌 408 真题多用 AT&T 风格（`movl $5, %eax`），但也会有 Intel 风格（`mov eax, 5`）。

---

## 二、x86 常用寄存器（IA-32）

| 寄存器 | 名称 | 常见用途 |
|--------|------|----------|
| `%eax` | 累加器 | 函数返回值 |
| `%ebx` | 基址寄存器 | 基址寻址 |
| `%ecx` | 计数器 | 循环计数 |
| `%edx` | 数据寄存器 | I/O、乘除辅助 |
| `%esi` | 源变址 | 源字符串指针 |
| `%edi` | 目的变址 | 目的字符串指针 |
| `%esp` | **栈指针** | 指向栈顶 |
| `%ebp` | **帧指针** | 指向当前栈帧基址 |

> ⚠️ **caller-saved**：`%eax`、`%ecx`、`%edx`（调用者保存）
> ⚠️ **callee-saved**：`%ebx`、`%esi`、`%edi`、`%ebp`（被调用者保存）

## 三、常用指令速查

### 3.1 数据传送

| 指令 | 格式 | 操作 |
|------|------|------|
| `mov` | `mov src, dst` | dst ← src |
| `push` | `push src` | (SP) ← src; SP ← SP - 4 |
| `pop` | `pop dst` | dst ← (SP); SP ← SP + 4 |
| `lea` | `lea src, dst` | dst ← **地址**（不访存！） |

```asm
movl $5, %eax      # eax = 5（立即数传寄存器）
movl %eax, (%ebx)  # 把 eax 的值存入 ebx 指向的内存
leal 8(%esi), %edi # edi = esi + 8（只算地址，不访存！）
```

### 3.2 算术逻辑运算

| 指令 | 格式 | 操作 |
|------|------|------|
| `add` | `add src, dst` | dst ← dst + src |
| `sub` | `sub src, dst` | dst ← dst - src |
| `inc` / `dec` | `inc dst` | dst ← dst + 1 / -1 |
| `imul` | `imul src` | 有符号乘（edx:eax ← eax × src） |
| `and` / `or` / `xor` | `and src, dst` | dst ← dst 按位与/或/异或 src |
| `not` | `not dst` | 按位取反 |
| `shl` / `shr` | `shl k, dst` | 逻辑左移/右移 k 位 |
| `sal` / `sar` | `sal k, dst` | 算术左移/右移 k 位 |

```asm
addl %ebx, %eax    # eax = eax + ebx
subl $1, %ecx      # ecx = ecx - 1
shll $2, %eax      # eax = eax << 2  （左移 n 位 = × 2ⁿ）
```

### 3.3 条件码与比较

| 指令 | 操作 | 影响的标志位 |
|------|------|------------|
| `cmp a, b` | 计算 b - a，**只设标志位不存结果** | ZF、CF、SF、OF |
| `test a, b` | 计算 a & b，**只设标志位** | ZF、SF |

| 条件码 | 全称 | 含义 |
|:---:|------|------|
| **ZF** | Zero Flag | 结果为 0 |
| **CF** | Carry Flag | 无符号溢出/借位 |
| **SF** | Sign Flag | 结果为负 |
| **OF** | Overflow Flag | 有符号溢出 |

### 3.4 条件跳转

| 指令 | 含义 | 条件 |
|------|------|------|
| `je` / `jz` | 相等/为 0 跳转 | ZF = 1 |
| `jne` / `jnz` | 不等/不为 0 跳转 | ZF = 0 |
| `jg` | 有符号大于跳转 | SF = OF 且 ZF = 0 |
| `jge` | 有符号大于等于 | SF = OF |
| `jl` | 有符号小于 | SF ≠ OF |
| `ja` | 无符号大于 | CF = 0 且 ZF = 0 |
| `jb` | 无符号小于 | CF = 1 |
| `jmp` | 无条件跳转 | — |

### 3.5 函数调用

| 指令 | 操作 |
|------|------|
| `call addr` | ① push 返回地址；② jmp addr |
| `ret` | pop 返回地址 → PC |
| `leave` | mov %ebp, %esp; pop %ebp（恢复栈帧） |

## 四、C 语言 → 汇编对应关系

### 4.1 基本运算

```c
int x, y;
x = y + 5;
```
```asm
movl y, %eax       # eax = y
addl $5, %eax      # eax = eax + 5
movl %eax, x       # x = eax
```

### 4.2 if-else

```c
if (x > 0)
    y = 1;
else
    y = -1;
```
```asm
    cmpl $0, x      # 比较 x 和 0
    jle  .L2        # x ≤ 0 → 跳 L2
    movl $1, y      # y = 1
    jmp  .L3
.L2:
    movl $-1, y     # y = -1
.L3:
```

> 🔑 **408 出题套路**：给一段汇编 → 反推出 C 代码逻辑。

### 4.3 while 循环

```c
while (x > 0) {
    sum += x;
    x--;
}
```
```asm
    jmp  .L2        # 先判断
.L3:
    addl x, sum     # sum += x
    subl $1, x      # x--
.L2:
    cmpl $0, x      # 比较
    jg   .L3        # x > 0 继续
```

### 4.4 for 循环

```c
for (i = 0; i < n; i++)
    a[i] = 0;
```
```asm
    movl $0, %ecx      # i = 0（ecx = 计数器）
    jmp  .L2
.L3:
    movl %ecx, %eax    # eax = i
    sall $2, %eax      # eax = i * 4（sizeof(int)=4）
    addl a, %eax       # eax = &a[i]（A+IX → 变址寻址！）
    movl $0, (%eax)    # a[i] = 0
    addl $1, %ecx      # i++
.L2:
    cmpl n, %ecx
    jl   .L3
```

> 🔑 **这里用的是变址寻址**！`sall $2` = ×4（元素大小），`addl a` = 加基地址 = EA = A + (IX)。

### 4.5 数组访问

```c
a[i]  →  &a[0] + i × sizeof(element)
```

| 寻址方式 | 汇编写法 | 说明 |
|----------|----------|------|
| 变址 | `a(,%ecx,4)` | AT&T：基址(a) + 变址(ecx) × 比例(4) |
| 基址变址 | `a(%ebx,%ecx,4)` | 基址(a+ebx) + 变址(ecx) × 4 |

### 4.6 指针操作

```c
int *p = &x;
*p = 10;
```
```asm
    leal x, %eax       # eax = &x（LEA 取地址）
    movl %eax, p       # p = &x
    movl p, %eax       # eax = p（即 &x）
    movl $10, (%eax)   # *p = 10（寄存器间接寻址！）
```

### 4.7 结构体

```c
struct S { int a; int b; };
s.b = 5;
```
```asm
    leal s, %eax       # eax = &s
    movl $5, 4(%eax)   # *(eax+4) = 5（b 偏移 4 字节）
```

## 五、函数调用的栈帧结构（408 必考！）

### 5.1 栈帧布局

```
高地址
+------------------+
|  参数 n          |  ← 调用者压入
|  ...             |
|  参数 1          |
+------------------+
|  返回地址         |  ← call 自动压入
+------------------+
|  旧 %ebp         |  ← push %ebp（保存调用者的帧指针）
+------------------+  ← %ebp 指向这里
|  局部变量 1       |
|  ...             |
|  局部变量 k       |
+------------------+
|  callee-saved     |  ← %ebx, %esi, %edi
|  寄存器备份       |
+------------------+  ← %esp 指向这里
低地址
```

### 5.2 函数调用标准序（Prologue & Epilogue）

```asm
; === 被调用者序言（保存栈帧）===
pushl %ebp          ; 保存旧帧指针
movl  %esp, %ebp    ; 新帧指针 = 当前栈顶
subl  $N, %esp      ; 分配 N 字节局部变量空间

; === 函数体 ===
...

; === 被调用者尾声（恢复栈帧）===
movl  %ebp, %esp    ; 释放局部变量空间
popl  %ebp          ; 恢复旧帧指针
ret                 ; 返回
```

> 等价于 `leave; ret`

### 5.3 参数传递约定

| 方式 | 说明 |
|------|------|
| **寄存器传参** | 参数少时用 `%eax`、`%edx`、`%ecx` 传递（MIPS 用 `$a0`~`$a3`） |
| **栈传参** | 参数多时从右向左压入栈（x86 cdecl 约定） |
| **返回值** | **`%eax`**（32 位）/ `%rax`（64 位） |

## 六、408 真题常见套路

### 套路 1：给汇编 → 判断等价 C 代码
- 识别 `cmpl + jXX` 的组合 → 对应 if/while/for
- 注意 `jle`（有符号 ≤）vs `jbe`（无符号 ≤）

### 套路 2：给 C → 写关键汇编指令
- 数组访问 = 变址寻址 → `a(%ecx,4)` 
- 指针解引用 = 寄存器间接寻址 → `(%eax)`
- 取地址 = LEA → `leal x, %eax`

### 套路 3：栈帧分析
- `push %ebp; movl %esp, %ebp` → 函数入口
- 参数位置 = `8(%ebp)`、`12(%ebp)` …（跳过返回地址 + 旧 ebp）
- 局部变量位置 = `-4(%ebp)`、`-8(%ebp)` …

### 套路 4：递归函数的栈帧
- 每次递归调用创建**新栈帧** → 栈空间 O(n)
- 尾递归可优化为迭代（栈帧复用）

## 七、408 高频指令集锦

| 汇编指令 | 等价 C | 注意点 |
|----------|--------|--------|
| `movl $0, %eax` | `eax = 0` | |
| `leal (%eax,%eax,4), %eax` | `eax = eax * 5` | LEA 可做乘法！ |
| `xorl %eax, %eax` | `eax = 0` | 比 mov $0 更高效 |
| `testl %eax, %eax` | 判断 eax 是否为 0 | 只设 ZF |
| `sall $2, %eax` | `eax <<= 2` | 等价 `×4` |
| `sarl $1, %eax` | `eax >>= 1` | 算术右移：÷2（补符号位） |
| `imull %ecx` | `edx:eax = eax × ecx` | 有符号乘 |

## 🧠 记忆框架

```
C → 汇编三要素：
1. 数据在哪？（寄存器 / 内存 / 栈）
2. 地址怎么算？（寻址方式：直接/间接/变址/基址/相对）
3. 控制怎么流？（cmp + jmp / jXX + call / ret）

栈帧三板斧：
- push %ebp + movl %esp, %ebp → 进入函数
- 8(%ebp) → 第一个参数
- -4(%ebp) → 第一个局部变量
- movl %ebp, %esp + pop %ebp → 退出函数
```
