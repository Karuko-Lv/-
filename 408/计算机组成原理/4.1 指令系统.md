### 机器字长 vs 存储字长 vs 编址单位

三个概念在题目中极易混淆，先逐一厘清，再对比关系。

---

#### 1. 机器字长

> **CPU 一次能处理的二进制数据位数。**

$$
\text{机器字长} = \text{ALU 位数} = \text{通用寄存器宽度}
$$

- 它是 CPU **数据通路**的宽度，直接影响运算速度和硬件复杂度。
- 通常所说的”32 位 CPU””64 位 CPU”，指的就是机器字长。

**例**：64 位 CPU → 机器字长 = 64 bit，一次 ADD 可处理 64 bit 数据。

---

#### 2. 存储字长

> **一个存储单元中存放的二进制代码位数。**

- 存储字长由**存储芯片**的设计决定，和机器字长**没有必然相等关系**。
- 主存的每个存储单元对应一个地址，存储字长就是这个单元能存多少 bit。

**典型情况**：

| 场景 | 存储字长 |
|---|---|
| 按字节编址的现代 PC | 8 bit（1 B） |
| 某些嵌入式 / DSP | 16 bit / 32 bit |
| 按字编址的早期机器 | 等于机器字长 |

> ⚠️ 很多教材默认”按字节编址 → 存储字长 = 8 bit”，但两者本质不同：**编址单位决定地址粒度，存储字长决定单元容量**。

---

#### 3. 编址单位

> **一个地址对应的数据量**，即地址线每变化 1，数据空间跨越多少 bit。

**（1）字节编址**（现代计算机主流）

```
地址 0 → 1 B（8 bit）
地址 1 → 1 B
地址 2 → 1 B
```

$$
\text{编址单位} = 1\text{ B} = 8\text{ bit}
$$

**（2）按字编址**

设机器字长 = 32 bit：

```
地址 0 → 32 bit（1 字）
地址 1 → 32 bit
地址 2 → 32 bit
```

$$
\text{编址单位} = 1\text{ 字}
$$

---

#### 4. 三者关系

三者**没有必然相等关系**。现代 PC 的典型配置：

| 概念 | 含义 | 典型值（64 位 PC） |
|---|---|---|
| **机器字长** | CPU 一次处理多少位 | 64 bit |
| **存储字长** | 一个存储单元存多少位 | 8 bit（按字节编址时） |
| **编址单位** | 一个地址对应多少数据 | 8 bit（1 B） |
| **1 个字** | 机器字长对应的字节数 | 64 ÷ 8 = **8 B** |

**核心公式**：

$$
1\text{ 字} = \frac{\text{机器字长}}{\text{编址单位（bit）}}\ \text{个编址单位}
$$

当按字节编址时：

$$
\boxed{1\text{ 字} = \frac{\text{机器字长}}{8}\ \text{B}}
$$

**考试常见组合**：

| 机器字长 | 编址方式 | 1 字 |
|---|---|---|
| 64 bit | 按字节 | 8 B |
| 32 bit | 按字节 | 4 B |
| 16 bit | 按字节 | 2 B |
| 32 bit | 按字（32 bit） | 1 字 = 1 地址 |

---

#### 5. 为什么现代计算机按字节编址？

**灵活性**——不同数据类型需要不同字节数：

| C 类型 | 需要字节 |
|---|---|
| `char` | 1 B |
| `short` | 2 B |
| `int` | 4 B |
| `long` | 8 B |

如果按字编址（一个地址 = 8 B），访问 `char` 需要额外截断操作，极不方便。因此现代 CPU 统一**按字节编址**，兼顾各种数据类型的高效访问。

---

#### 6. 一句话记忆

> **机器字长决定”CPU 一次算多少”，存储字长决定”一个单元存多少”，编址单位决定”一个地址指多少”。现代 PC 标配：64 bit 机器字长 + 按字节编址（编址单位 = 存储字长 = 8 bit），故 1 字 = 8 B。**
