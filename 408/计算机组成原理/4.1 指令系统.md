### 不同层级的指令
计算机系统不同层次的用户会使用不同级别的指令![[截屏2026-06-24 22.00.57.png]]
**软件层面的指令需要编译、汇编或解释成机器指令后才能被计算机硬件执行**
一般来说一条高级语言指令会被编译或解释为多条机器指令；汇编指令通常会被汇编成一条机器指令
一条机器指令所要实现的功能往往依赖于多条微指令的执行
🧑‍🏫机器指令是计算机软件和硬件的接口，也是用户操作和使用计算机硬件的借口，故从机器指令级介绍指令系统，下一章从微指令集介绍中央处理器

## 指令的一般格式
指令一般由**操作码字段和地址码字段两部分**构成![[截屏2026-06-24 22.08.22.png]]
**地址码指明相应的操作数**，可以包含多个操作数
如何获取操作数由寻址方式决定，即寻址方式决定操作数的存放位置和访问方式，寻址方式可以显式包含在地址码字段中 or 隐式包含在操作码字段中

### 机器字长 vs 存储字长 vs 编址单位

#### 1. 编址单位是什么？

编址单位：

> **一个地址对应的数据量。**

例如：字节编址

```text
地址0 → 1B
地址1 → 1B
地址2 → 1B
```
此时：

```text
编址单位 = 1B = 8bit
```
现代计算机几乎都是这样。

按字编址

假设：
```text
机器字长 = 32bit
```
则：
```text
地址0 → 32bit
地址1 → 32bit
地址2 → 32bit
```
此时：
```text
编址单位 = 1字
```

---

#### 2. 机器字长是什么？

机器字长：

> CPU 一次能够自然处理的数据位数。

例如：

```text
32位CPU
```

则：

```text
机器字长 = 32bit
```

---

例如：

```text
ADD
```

一次加：

```text
32bit
```

数据。

---

#### 3. 两者有关系吗？
没有必然相等关系

例如：

现代电脑：

```text
机器字长 = 64bit
```

但：

```text
编址单位 = 8bit
```

即：

```text
按字节编址
```

---

因此：

```text
机器字长 ≠ 编址单位
```

很正常。

---

#### 举个实际例子

你的 Mac：

```text
Apple Silicon
64位CPU
```

即：

```text
机器字长 = 64bit
```

但内存地址：

```text
地址0 → 1B
地址1 → 1B
地址2 → 1B
```

所以：

```text
编址单位 = 1B
```

---

#### 它们之间最重要的关系

如果：

```text
机器字长 = 64bit
编址单位 = 8bit
```

则：

```text
一个字
=
64bit
=
8B
```

```text
1字 = 8个字节
```

#### 考试最常考公式

假设：

```text
机器字长 = 32bit
按字节编址
```

则：

```text
1字 = 4B
```

因为：

[  
32 bit ÷ 8 bit = 4  
]

---

假设：

```text
机器字长 = 16bit
按字节编址
```

则：

```text
1字 = 2B
```

因为：

[  
16 bit ÷ 8 bit = 2  
]

---

#### 为什么现代计算机都按字节编址？

因为灵活。

例如：

```c
char
short
int
long
```

分别需要：

```text
1B
2B
4B
8B
```

如果按字编址：

```text
一个地址 = 8B
```

那么：

```text
char
```

访问会很不方便。

所以现代 CPU 基本统一：

```text
按字节编址
```

---

#### 一张表总结

|概念|含义|
|---|---|
|编址单位|一个地址对应多少数据|
|机器字长|CPU 一次处理多少位|
|存储字长|一个存储单元存多少位|

例如现代 64 位计算机：

|参数|数值|
|---|---|
|编址单位|8 bit（1 B）|
|机器字长|64 bit|
|一个字|8 B|

---

#### 一句话记忆

> **编址单位决定“一个地址能表示多少数据”，机器字长决定“CPU 一次能处理多少数据”。现代计算机通常是“按字节编址（8 bit）+ 64 位机器字长”，因此一个机器字等于 8 个字节。**
