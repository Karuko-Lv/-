### Cache 系统的性能评价
#### Cache 系统
![[截屏2026-06-16 20.53.51.png]]
假设 CPU 要进行数据访问，为了简单起见假设数据访问为读操作 
1. CPU 首先在 cache 中查找待访问的数据（通过某种查找机制实现）
1️⃣找到了
2. 数据在 cache 中则 called 数据命中 
3. 将命中的数据读取到 CPU 即可
上述数据访问时间 called 命中访问时间,记为 $t_c$
包括 1️⃣cache 内查找时间 2️⃣cache 访问时间
![[截屏2026-06-16 20.58.26.png]]
2️⃣没找到
2. CPU 在 cache 中未查找到所要访问的数据 called 数据缺失
3. 将缺失的数据从主存调入到 cache 中
4. CPU 从 cache 中访问数据
上述数据访问时间 called 缺失补偿
包括 1️⃣cache 内的查找时间 2️⃣主存访问时间记为 $t_m$ 3️⃣cache 访问时间
由于 $t_m$ 较大所以缺失补偿粗略记为 $t_m$ 
🧚‍♀️这里白中英的书说的是，用主存读周期把该字从主存送到 cpu，与此同时把含有该字的块送到 Cache；在 cache 中没有命中，CPU 也不去主存，而是将数据从主存调入 cache，CPU 再在 cache 中访问；对，redis 未命中直接查数据库返回，同时再写入 redis；块存储；prefetch

Q：如何通过主存地址在 cache 中进行快速查找
A：🧑‍🏫通过信息的区内数据块号即 cache 行号的取值，可以索引到该信息所在主存数据块所对应的 cache 行；直接映射规则可知主存中会有多个数据块对应 cache 中的某个固定行，例如本例中主存分区 0 的数据块 1 对应 cache 行 1、主存分区 1 的数据块 1 也对应 cache 行 1... 主存分区 m-1 的数据块 1 也对应 cache 行 1，❓如何判断 cache 行 1 中当前保存的主存数据块的副本是否是待访问信息所在的那个主存数据块呢❓❓❓
Cache 行中包含有主存分区号的作用：通过比较 cache 行中主存分区号的取值与待访问信息的主存地址中的主存分区号部分的取值是否相等，从而判断出 cache 行中的数据块副本保存的是否是待访问信息所在的主存数据块，如果相等则数据命中；不等则数据缺失。若该 cache 行中的有效位为 1 则表示该 cache 行信息有效，通过待访问信息的主存地址中的数据块内偏移部分的取值就可以在该 cache 行中的数据块副本中找到待访问信息进而从快速的 cache 中读取该信息而不用再慢速主存中读取信息；cache 未命中即数据缺失或该 cache 行有效位为 0 则需要根据待访问信息的主存地址**将主存中该信息所在的数据块载入相应的 cache 行**，相应将该 cache 行的有效位置 1 以及将该 cache 行的主存分区号设置为该信息的主存地址中的主存分区号，同时将信息送入 CPU

#### 硬件实现
🧑‍🏫下面是一个包含有 4 个 cache 行的直接映射方式的硬件逻辑实现：![[截屏2026-06-18 17.45.23.png]]
1️⃣各 cache 行的有效位用英文 valid 表示
2️⃣各 cache 行的标记字段用英文 tag 表示
3️⃣各 cache 行用于保存数据块副本的缓冲区是 $W_0 \sim W_3$（假设个数据块由 4 个存储字构成）

## Chatgpt


```text
脏位（Dirty Bit）：
用于表示Cache行中的数据是否与主存对应数据一致。
```

具体含义：

#### Dirty = 1

表示：

```text
CPU修改了Cache中的数据
↓
主存中的对应数据还没修改
```

即：

```text
Cache ≠ 主存
```

数据是“脏”的。

例如：

```text
主存：100

↓

装入Cache

↓

CPU改成200

↓

Cache：200
主存：100
```

此时：

```text
Dirty = 1
```

---

#### Dirty = 0

表示两种情况：

##### 情况 1：从未修改

```text
主存：100

↓

装入Cache

↓

CPU没改
```

此时：

```text
Cache：100
主存：100
Dirty=0
```

---

##### 情况 2：修改后已经写回

```text
CPU改成200

↓

写回主存

↓

主存也变成200
```

此时：

```text
Cache：200
主存：200
Dirty=0
```

---

### 为什么需要脏位？

主要针对：

```text
写回法（Write Back）
```

因为写回法是：

```text
CPU改Cache
↓
先不改主存
```

等 Cache 行被替换时再决定：

```text
要不要写回主存
```

判断依据就是：

```text
Dirty Bit
```

---

替换时：

#### Dirty = 0

说明：

```text
主存已经是最新的
```

直接丢弃：

```text
Cache行
```

即可。

---

#### Dirty = 1

说明：

```text
主存还是旧数据
```

必须：

```text
Cache
↓
主存
```

先写回，再替换。

---

你可以记一句最经典的话：

> **脏位=1 表示 Cache 中的数据比主存更新；脏位=0 表示 Cache 与主存一致（或未被修改）。**

这句话在选择题和填空题里几乎不会错。

## 单元
### 为什么叫“单元”？
因为主存可以按不同粒度编址。
#### 按字节编址（现代计算机最常见）
```
地址0 → 1B
地址1 → 1B
地址2 → 1B...
```
此时：
```
存储单元 = 1字节
```

---

#### 按字编址

假设机器字长：
```
32bit = 4B
```
则：
```
地址0 → 一个字(4B)
地址1 → 一个字(4B)
地址2 → 一个字(4B)...
```
此时：
```
存储单元 = 一个字
```
不是一个字节。
### 存储单元🆚贮存块🆚字

|名称|本质|
|---|---|
|存储单元|最小编址单位|
|主存块 (Block)|Cache 一次调入的数据块|
|字 (Word)|CPU 一次处理的数据长度|

例如：

```
字长 = 32bit
主存块 = 32B
按字节编址
```

那么：

```
1个存储单元 = 1B
1个字 = 4B
1个主存块 = 32B = 8个字 = 32个单元
```

#### LRU 算法
将近期内最少被命中访问过的 cache 行淘汰
🧚‍♀️Least Recently Used Algorithm；最近这个条件很重要；这就是 os 的最近最久未使用算法；这里和王道的不一样，王道是按栈顺序，所以低的计数器加一其他不变；这个才是真正的淘汰计数器哈哈哈上一个是命中计数器；王道书上说的是比其计数低的计数器加 1 ；减一是不是比清零好；细节不太一样，但是思想是一样的；wd 的是说比它低的计数器才需要加 1 ![[截屏2026-06-22 22.54.46.png]] ![[截屏2026-06-22 22.54.58.png]]
🧚‍♀️值越大代表最近越没被访问，应该被替换出去；减一是不是比清零好![[截屏2026-06-22 22.55.40.png]]
🧚‍♀️根据袁春风老师的教材来说，n 路 cache 组相联，计数值位数为以 2 为底 n 的对数来说，wd 是符合标准的；老师参考唐朔飞老师的书更多一点；就是这，20 年 46 题。完全忘记 Q。Q ![[截屏2026-06-22 22.57.49.png]]

🧚‍♀️因为比它大的加一，排名还是不会变，比它小的加一可能超过它‼️；理解成排名次就好，每次访问到的去最下面，最上面的是准备被换的，或者你反过来也行；等于不会加啊，不然就有可能超过表示范围了；如果低于最高数值加一，三次之后大家就全相同了。此时用随机替换就体现不出计数器的优势了

`| 主存字块标记 (Tag) | 组索引 (Index) | 块内偏移 (Offset) |`

- **组索引 (Index) = 主存分区号（的低位）**
- **标记 (Tag) = 主存分区号的高位 + 区内块号**
- **块内偏移 (Offset) = 块内偏移**

🧚‍♀️写分配法的意思是，写一个新数据时，同时往主存和 cache 里写，这样可以利用好空间局部性；非写分配法就只往主存里写；未命中时写穿是先改主存，再把改后的主存调入 cache，这样两个里都是新的，写回是指先把主存中旧的调进 cache，直接改 cache，主存暂时不管，这样 cache 新主存旧；这里空间局部性是指无论是新数据还是旧数据，数据块内不需要被写的数据都可以调入 cache 后直接用；写回就是把要写的数据块先调入 cache，然后写 cache
