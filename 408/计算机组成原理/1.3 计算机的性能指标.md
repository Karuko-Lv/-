#### 机器字长
$机器字长=ALU位数=通用寄存器的宽度$
字长⬆️数的表示范围⬆️精度高⬆️计算精度⬆️
影响计算速度![[截屏2026-06-02 17.30.07.png]]
🧚‍♀️越大芯片的面积就越大，总线的数量也越大，所以有限制
#### 主存容量
![[截屏2026-06-02 17.46.18.png]]
主存容量⬆️程序运行访问辅存⬇️执行速度⬆️
##### 计算
知道MAR、MDR的位数就可以计算出主存储器的总容量了
![[截屏2026-06-02 17.46.52.png]]
🧚‍♀️MAR 的位数由 CPU 的地址总线宽度决定，它表示一次能寻址的存储单元个数（2^MAR位数）。插入内存条只是增加了物理存储容量，CPU 的地址总线宽度不变，因此 MAR 的位数也不会变；我有点不明白，买回来的电脑可以增加内存条，主存回增加，但这个时候地址总线的数量是不变的，这不就出问题了吗；你都把内存条插进去了，你说够不够？他都提前弄好了地址长度！；不会出问题，因为“地址总线位数决定的是最大可寻址空间”，而你增加的内存通常还远远没达到这个上限。
#### 吞吐量
计算机系统在单位时间内能够处理的信息量
##### 影响因素
1️⃣CPU处理能力
2️⃣内存（主存）的访问速度
3️⃣外村（辅存，例如硬盘）的访问速度
#### 响应时间
```
点击百度  
↓  
0.2秒后页面开始出现

```
那么响应时间 = 0.2秒
即使网页还没完全加载完：
又过了1秒页面才全部显示
响应时间仍然是：0.2秒
对于一个进程：
$响应时间=首次获得CPU时间−提交时间\text{响应时间} = \text{首次获得CPU时间} - \text{提交时间}响应时间=首次获得CPU时间−提交时间$
![[截屏2026-06-02 18.00.27.png]]
 VS周转时间
 周转时间关注的是：
> **从任务提交到任务全部完成所经历的总时间。**

公式：
$周转时间=完成时间−提交时间\text{周转时间} = \text{完成时间} - \text{提交时间}周转时间=完成时间−提交时间$
```
0ms   作业提交
50ms  开始运行
300ms 完成
```

则：

```
周转时间 = 300ms
```

### 与运算速度相关的性能指标
#### CPU时钟频率和时钟周期
✅ **在计组里，一般可以认为主频就是时钟频率。**

计算机执行指令每个步骤都需要一个特定的控制信号进行控制![[截屏2026-06-02 18.15.30.png]]
🧚‍♀️就是CPU中每个元件都需要步调一致
##### CPU时钟信号
![[截屏2026-06-02 18.19.44.png]]是固定频率的脉冲信号 该信号的频率是**CPU时钟频率** Clock Rate 单位是Hz
CPU时钟信号会周期性出现高电平即脉冲 
每经过时钟周期时间就会出现一次脉冲
![[截屏2026-06-02 18.20.06.png]]
同类型计算机同一指令执行所需CPU时钟周期的数量一样，CPU 时钟频率⬆️，指令的执行速度⬆️
🧚‍♀️非同类通过细分流水线提高频率但性能不升反降

e.g 2023年桌面级高端CPU的加速频率最高接近6GHz
![[截屏2026-06-02 18.27.09.png]]![[截屏2026-06-02 18.27.36.png]]
🧚‍♀️存储单位现在用的是过时的（IBM当初发明的不规范前缀）现在使用KiB，MiB，GiB，当然，考试不考；记住这个**口诀**，大2速10
#### CPI
Cycle per instruction：执行一条指令所需要的时钟周期（相邻两个时钟脉冲之间的时间间隔）数量
一般来说：

> **CPI 越低，CPU 性能越好。**


不同指令功能不同
功能相同的指令也可以有不同的寻址方式
==>每条指令执行所需要的时钟周期数量也可能不同
![[截屏2026-06-02 18.30.17.png]]
$CPI = 程序执行所需的时钟周期数量/程序所包含的指令数量$

#### IPC
Instructions per cycle 每个时钟周期能够执行的指令数量
IPC是CPI的倒数
与CPU架构、指令集、指令流水线技术密切相关
由于指令流水线技术以及多核技术发展 **IPC的值可以大于1了**![[截屏2026-06-02 20.04.54.png]]
#### MIPS
每秒执行多少百万条指令
million instructions per second
![[截屏2026-06-02 20.30.54.png]]
时钟频率是时钟周期的倒数
🧚‍♀️这不是难，这是概念多，容易混，只有反反复复重复才能记得牢；**其实就是IPC/时钟周期，即每周期的指令数/周期，就等于~={red}每秒的指令数=~**
![[截屏2026-06-02 20.32.48.png]]
🧚‍♀️第二个原因是不是有点多余了，如果指令数量相同那MIPS就体现时钟周期了
把除以CPI换成乘IPC $每秒包含的时钟周期数*每个时钟周期能执行的指令数量 =每秒能执行的总指令数$![[截屏2026-06-02 20.36.45.png]]
主频就是时间频率![[截屏2026-06-02 20.42.46.png]]
