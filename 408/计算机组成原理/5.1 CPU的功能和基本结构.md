4️⃣指令中的地址码会送入**地址生成逻辑部件**，通过该部件对操作数寻址方式进行译码，进而**生成操作数的相关地址**，根据寻址方式不同**操作数的相关地址**会送入**程序计数器 PC** 或**存储器地址寄存器 MAR** 中（对于立即寻址方式，该部件生成的就是操作数本身，会送入运算部件）![[截屏2026-06-28 21.41.33.png]]

6️⃣操作控制器接收来自指令译码器的译码信号，根据测试条件、状态信息等通过时序发生器产生各种具有严格时间先后顺序的操作控制信号，called 微操作控制信号序列![[截屏2026-06-28 21.43.07.png]]
7️⃣微操作控制信号序列会被传送到计算机各功能部件的控制端，例如运算器中的相关功能部件，用于控制各功能部件按指令要求依序进行动作，进而实现指令要完成的功能 
8️⃣CPU 取指令时会使用**程序计数器 PC 的值**作为**访问主存的地址**，送入主存地址寄存器 MAR 中，以便从主存中读取指令；有些没有 MAR 的 CPU 会将 PC 中的主存地址直接送到地址总线上
故~={red}**PC 的位宽=主存地址总线位宽**=~
![[截屏2026-06-28 22.59.38.png]]
9️⃣**取址完成之后**还需要**修改 PC 的值**以**形成下一条指令的地址**，当程序顺序执行时 PC 的值会被➕1（⚠️本质上是**加指令字长**，即指令包含字节的数量，根据主存是按字节或存储字编址以及指令字长是多少，指令按边界对齐等确定➕1 是加多少；对于变长指令系统的指令需要进行指令译码后才能确定 PC 的值应该增加多少）；当程序顺序执行到分支跳转处时，PC 的值需要基于分支指令所提供的分支地址进行修改形成跳转后的新指令地址

#### 运算
##### ALU
Arithmetic Logic Unit，算术逻辑单元，完成各种运算。
例如：1️⃣加法 2️⃣减法 3️⃣乘法（部分 CPU）4️⃣除法（部分 CPU）5️⃣与、或、非、异 6️⃣比较 (CMP) 7️⃣移位
例如：

```
ADD EAX, EBX
```
#### 存储
CPU 内部进行数据存储的部件，主要就是各种寄存器。
##### 通用寄存器
例如：EAX、EBX、ECX、EDX
> 存放操作数、中间结果、地址等。

例如：
```
MOV EAX,5
```
这里 EAX 就是保存数据。
##### 专用寄存器
例如：
```
PC
IR
MAR
MDR
PSW
```

|寄存器|保存什么|
|---|---|
|PC|下一条指令地址|
|IR|当前指令|
|MAR|主存地址|
|MDR|主存交换的数据|
|PSW|标志位|
也是CPU 内部存储信息的地方。
⚠️「PC 相关」1️⃣机器指令不能显式使用 PC，PC 的值时自增的，或者是由转移类指令设置的 2️⃣PC 是隐含使用的寄存器 3️⃣调用指令 `call` 执行后 PC 值一定是被调用过程的入口地址
#### 传送
CPU 内部的数据要移动
例如：
PC  ➡️  MAR  ➡️  MDR  ➡️  IR
##### CPU 内部总线
Internal Bus
例如：
1️⃣ EAX  ➡️ CPU 内部总线  ➡️  ALU
2️⃣ MDR  ➡️  CPU 内部总线  ➡️  IR
故CPU 内部的数据几乎都依靠**内部总线**进行传送
##### 控制器（CU）
控制器本身不存数据，也不运算
它负责发控制信号
例如：
告诉 MAR ➡️ 装载数据
或者ALU ➡️ 开始加法
故控制器负责**控制数据传送**

### 一些长度单位
> 408 常考各种"位宽/长度"之间的关系，尤其是相等和不相等的陷阱。

#### 机器字长 
CPU 内部用于整数运算的数据通路的宽度。数据通路指的是数据在指令执行过程中所经过的路径及路径上的部件，主要是 CPU 内部进行数据运算、存储和传送的部件，这些部件的宽度基本上要一致才能相互匹配

#### 存储字长
主存中一个存储单元（一个存储字）包含的二进制位数。通常 **~={red}存储字长 = 机器字长=~**，但不是绝对的（有的机器存储字长是机器字长的整数倍）

#### 指令字长
一条指令包含的二进制位数。指令字长可以是定长或变长：
- 定长指令字长通常等于机器字长（或存储字长）
- 变长指令字长可能小于、等于或大于机器字长

#### IR 的位宽
IR 必须能够装下一条完整的指令，因此 IR 的位宽至少要等于最长指令字长，⚠️不是机器字长，与机器字长没有关系

#### PC 的位宽
~={red}**PC 的位宽 = 主存地址总线位宽**=~。PC 中存放的是下一条指令的主存地址（按字节编址的字节地址），位宽取决于主存地址空间大小。
⚠️当指令按字边界对齐存放时，PC 实际需要的位数可以少于地址总线位宽（低几位恒为 0 可省略），但题目一般按地址总线位宽处理

#### MAR 的位宽
~={red}**MAR 的位宽 = 主存地址总线位宽**=~。MAR 保存 CPU 要访问的主存单元的地址，位宽决定了 CPU 能访问的最大主存地址空间（如 32 位 MAR → 最大 2³² 个地址单元）

#### MDR 的位宽
~={red}**MDR 的位宽 = 存储字长**=~（通常也是机器字长）。MDR 保存从主存读出或待写入主存的数据，一次访存能读写的数据量由 MDR 位宽决定。
⚠️但有的教材说 MDR 位宽 = 机器字长，二者在大多数机器中是一致的，不必纠结

#### ALU 位宽
ALU 一次能处理的二进制位数。**ALU 位宽 = 机器字长**，因为 ALU 是数据通路的核心部件

#### 通用寄存器位宽
通用寄存器的位宽 **= 机器字长**，用于存放操作数、地址、中间结果等

#### PSW 位宽
程序状态字寄存器的位宽等于各标志位的总数之和，与机器字长没有固定关系，通常远小于机器字长

#### 地址总线位宽
决定了 CPU 能寻址的最大主存空间：最大寻址空间 = 2^(地址总线位宽) 个地址单元（通常是字节单元）

#### 数据总线位宽
CPU 与主存或外设之间一次能传送的数据位数，通常 **= 存储字长 = MDR 位宽**

| MAR 位宽 | 主存地址总线位宽 | — |
| MDR 位宽 | 存储字长（≈ 机器字长） | 不是指令字长 |
| PC 位宽 | 主存地址总线位宽 | 指令边界对齐时位数可减少 |
| IR 位宽 | ≥ 最长指令字长 | **≠ 机器字长** |
| ALU 位宽 | 机器字长 | — |
| 通用寄存器位宽 | 机器字长 | — |
| 数据总线位宽 | 存储字长 | — |
