# 4.2 寻址方式（完整笔记）

> 寻址方式：指令中「如何根据地址码字段找到真正的操作数」的方法。
> 包含两大类：**指令寻址**（找下一条指令）和 **数据寻址**（找操作数）。

---

## 指令寻址 vs 数据寻址

| 类型 | 对象 | 方式 |
|------|------|------|
| **指令寻址** | 下一条指令 | ① 顺序：PC + 指令长度；② 转移：转移指令给出目标 |
| **数据寻址** | 操作数 | 形式地址 + 寻址方式 → 有效地址 EA |

---

## 1️⃣ 立即寻址（Immediate）

> 形式地址 **不是地址，是操作数本身**。

| | |
|---|---|
| 位置 | 指令字中 |
| EA | 无 |
| 访存 | **0 次** |
| 用途 | 常量赋值 |
| 例 | `MOV AX, #5` |

---

## 2️⃣ 直接寻址（Direct）

$$
EA = A
$$

| | |
|---|---|
| 位置 | 主存 |
| 访存 | **1 次** |
| 优点 | 简单直观 |
| 缺点 | 寻址范围受地址码位数限制 |

---

## 3️⃣ 间接寻址（Indirect）

> 形式地址 A 指向一个主存单元，该单元存的是 EA。

$$
EA = (A)
$$

| | |
|---|---|
| 访存 | **≥2 次** |
| 优点 | EA 位数不受形式地址位数限制（扩大寻址范围） |
| 缺点 | 多次访存，速度慢 |

> ⚠️ 多次间址：若间址单元最高位 = 1，继续间址。

---

## 4️⃣ 寄存器寻址（Register）

> 形式地址是寄存器号，操作数在寄存器中。

$$
\text{操作数} = (Ri)
$$

| | |
|---|---|
| 访存 | **0 次** |
| 优点 | 速度极快；地址码短 |
| 缺点 | 寄存器数量有限 |
| 例 | `ADD R1, R2` |

---

## 5️⃣ 寄存器间接寻址（Register Indirect）

> 寄存器内容 = EA。

$$
EA = (Ri)
$$

| | |
|---|---|
| 访存 | **1 次** |
| 优点 | 比间接寻址快（无额外的访存取 EA） |
| 例 | `LOAD R1, (R2)` |

---

## 6️⃣ 变址寻址（Indexed）

> **EA = A + (IX)**。IX 由程序员修改，A 不变。

$$
\boxed{EA = A + (IX)}
$$

| | |
|---|---|
| 访存 | **1 次** |
| 核心特征 | IX（变址寄存器）存**偏移量**，A 存**基地址** |
| 应用 | **数组/字符串遍历** |
| 例 | `LOAD R1, 1000H(IX)` |

> 🎯 `a[i]` 的机器实现：A = &a[0]，(IX) = i × sizeof(element)。

---

## 7️⃣ 基址寻址（Base）

> **EA = A + (BR)**。BR 由 OS 设定，程序不可改。

$$
\boxed{EA = A + (BR)}
$$

| | |
|---|---|
| 访存 | **1 次** |
| 核心特征 | BR（基址寄存器）存**程序起始地址**，A 存**偏移量** |
| 应用 | **程序重定位 / 多道程序** |
| 例 | x86: `[BX + offset]` |

---

### ⚔️ 变址寻址 vs 基址寻址（408 必考对比！）

| 维度 | 变址寻址 | 基址寻址 |
|------|----------|----------|
| **寄存器内容** | 偏移量 | 基准地址 |
| **谁改寄存器** | 程序员 | OS |
| **形式地址 A** | 基准地址（数组首址） | 偏移量 |
| **应用** | 数组遍历 | 程序重定位 |
| **EA** | A + (IX) | A + (BR) |

> 📌 **记忆口诀**：程序员看「变」化（IX 可变），OS 定「基」座（BR 基准）。

---

## 8️⃣ 相对寻址（Relative）

> **EA = (PC) + A**。A 是补码位移量（可正可负）。

$$
\boxed{EA = (PC) + A}
$$

| | |
|---|---|
| 访存 | **1 次** |
| 特点 | 代码与位置无关（**可浮动**） |
| 应用 | **转移指令**（JMP、JZ 等） |
| 例 | `JMP +8` / `JZ -4` |

> ⚠️ **关键**：(PC) 是**取指后已更新**的值（已指向下一条指令），不是取指前的值。

---

## 9️⃣ 堆栈寻址（Stack）

> 操作数隐含在栈顶，SP 隐含寻址（零地址指令）。

| | |
|---|---|
| 位置 | 栈顶 (SP) |
| 访存 | **1 次** |
| 优点 | 指令字最短 |
| 应用 | PUSH、POP、CALL、RET |
| 例 | `PUSH AX`、`POP BX` |

---

## 📊 九种寻址方式总表

| 寻址方式 | EA 公式 | 访存次数 | 操作数在 | 主要用途 |
|----------|---------|:---:|------|----------|
| **立即** | 无 EA | 0 | 指令中 | 常量赋值 |
| **直接** | EA = A | 1 | 主存 | 固定地址变量 |
| **间接** | EA = (A) | ≥2 | 主存 | 指针、扩大寻址 |
| **寄存器** | 无 EA | 0 | 寄存器 | 频繁使用的变量 |
| **寄存器间接** | EA = (Ri) | 1 | 主存 | 寄存器指针访存 |
| **变址** | EA = A+(IX) | 1 | 主存 | **数组遍历** |
| **基址** | EA = A+(BR) | 1 | 主存 | **程序重定位** |
| **相对** | EA = (PC)+A | 1 | 主存 | **转移指令** |
| **堆栈** | EA = (SP) | 1 | 栈 | 子程序调用 |

---

## 🔥 408 高频易错

### 1. 变址 ≠ 基址
- **变址**：IX 是偏移、A 是基址 → 数组
- **基址**：BR 是基址、A 是偏移 → 程序重定位

### 2. 立即 ≠ 直接
- **立即**：形式地址**是操作数本身**
- **直接**：形式地址**是操作数的地址**

### 3. 间接 ≠ 寄存器间接
- **间接**：地址在**主存**中
- **寄存器间接**：地址在**寄存器**中

### 4. 相对寻址的 PC 值
- **基于取指后的 PC**，不是取指前的

### 5. 复合寻址
- x86：`[BX + SI + 1000H]` = 基址 + 变址 + 偏移

---

## 🧠 记忆口诀

```
立即在指令，直接给地址。
间址多访存，寄存最快速。
变址走数组，基址管重定。
相对便跳转，堆栈零地址。
```
