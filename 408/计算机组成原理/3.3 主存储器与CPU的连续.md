### 数据总线位宽 🆚 机器字长
1️⃣机器字长（Word Length）：

> CPU 一次能够自然处理的数据位数。

32 位 CPU <==> 机器字长 = 32 位
2️⃣数据总线位宽：

> CPU 与存储器之间一次能传输多少位数据。

数据总线宽度 = 64 位<==>一次可传输 64 bit

例如经典 32 位处理器：
```
机器字长 = 32位数据总线 = 32位
```
此时：
```
CPU一次处理32位CPU一次传输32位
```

两者一样。

**但它们本质上不是同一个概念**

1. 例如 Intel 8088：
```
机器字长 = 16位
```
但：
```
数据总线 = 8位
```
意味着：
```
CPU内部能处理16位
```
但是：
```
从内存取16位数据需要分两次传输
```

2. 例如很多 64 位 CPU：

```
机器字长 = 64位
```
但：
```
内存总线可能64位128位256位
```
（通过多通道实现）
此时：
```
数据总线位宽≠机器字长
```
> 机器字长与数据总线宽度没有必然相等关系，但很多计算机为了提高效率会设计成相等。

### 三个容易混淆的概念

|概念|含义|
|---|---|
|机器字长|CPU 一次处理的数据位数|
|数据总线宽度|一次传输的数据位数|
|地址总线宽度|可寻址空间大小|

例如：
```
32位CPU
```
可能：
```
机器字长 = 32位；数据总线 = 64位；地址总线 = 36位
```
完全有可能。

### 字扩展中有关寻址的问题
