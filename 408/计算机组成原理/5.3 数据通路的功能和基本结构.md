### 总线结构的数据通路
![[截屏2026-06-30 17.19.24.png]]
#### CPU 的内部总线
连接运算器和各类寄存器，构成 CPU 内部的单总线结构的数据通路
#### CPU 的外部总线
连接 CPU 主存以及 I/O 设备等部件，called 系统总线或外部总线
#### CPU 内部的算术逻辑单元 ALU
![[截屏2026-06-30 17.31.36.png]]
与其配套的有暂存寄存器 X 和 Z；其中 X 用于暂存 ALU 的其中一个操作数，Z 用于暂存 ALU 得运算结果；ALU 的另一个操作数来自 CPU 的内部总线；还有一个 CPU 内部程序状态字寄存器 PSW，用于暂存 ALU 的运算状态标志
🧚‍♀️Z 是 ACC 吗？；Z 是暂存器；操作元件和状态元件这里有讲吗；z 是暂存器的一种。X 也是一个暂存器；ALU 没有存储功能所以给它加两个存储的；ACC 可以算是 x 的一种
[chatgpt]ALU 的作用只有：**对输入的数据进行运算，并立即输出结果**。ALU 是组合逻辑电路（Combinational Logic），没有长期存储能力。现代X 86没有真正意义上的ACC,因为有很多通用寄存器
#### CPU 内部的通用寄存器组/寄存器堆
![[截屏2026-06-30 17.32.40.png]]
包含一组通用寄存器，可通过通用寄存器的编号对相应的通用寄存器进行读写操作
`R#` 端口为**输出寄存器**编号
`W#` 端口为**写入寄存器**编号
🧚‍♀️CPU 内部总线结构链接 CPU 中哥哥寄存器等等系统总线连接 CPU 和主存和 IO 等；数据传输过程中，CU 按时序发出的控制信号实际上是控制输出使能（例如对应寄存器三态门开关）从而达到控制数据流方向，否则所有数据同时涌向总线会造成混乱和冲突[chatgpt]CU 并不是直接"搬运数据"，而是通过时序控制信号控制寄存器、ALU、MUX、三态门等部件的工作，使数据按照预定的数据通路流动；在单总线结构中，同一时刻只允许一个数据源驱动总线，以避免总线冲突。

### 不同类型指令基于 CPU 内部单总线结构的执行流程及其所涉及到的数据通路

🧑‍🏫5 条 MIPS 32 指令以及指令周期的各个节拍所执行的操作如上图所示：
1️⃣每条指令的取指周期完全一样
2️⃣ `addi` 和 `add` 指令没有计算周期
3️⃣每条指令的执行周期及其所包含的各节拍所要执行的操作
**在单总线结构中最慢的数据通路是访存通路**，例如每条指令的取指周期的第 2 个节拍包含访存通路 `M[MAR]`; `lw` 指令的执行周期的第 2 个节拍也有访存通路 `M[MAR]`; `sw` 指令的执行周期的第 3 个节拍也有访存通路 `M[MAR]`
**运算通路也是最慢的数据通路**，例如每条指令的取指周期的第 2 个节拍包含运算通路 `X+4->Z`; `lw` 指令 `sw` 指令 `beq` 指令各自的计算周期的第 2 个节拍包含有运算通路； `beq` 指令 `add` 指令 `addi` 指令各自的执行周期的第 2 个节拍包含有运算通路
⚠️每条指令执行完毕后还会进行一些公共操作，通常用于处理外部设备的 IO 请求，若在公共操作阶段发现**有外部设备中断请求**则**进入中断响应阶段**进行**保存断点和载入中断服务程序入口地址到 PC 的操作**，否则直接进入到下一条指令的取指令阶段
**取指令也属于公共操作**，每条指令都会进行取指令操作![[截屏2026-07-01 22.36.17.png]]
🧚‍♀️微程序流程图；addi 和 add 的操作数都是从寄存器获取的，没有访存；如果一个时钟周期内没有完成执行操作下一个周期控制信号改变那咋办；**建议大家都跟着画画，画完就感觉豁然开朗了**；做两道题就都会了
#### `lw` 指令格式+取指周期
是 load word 英文缩写，功能是从主存读取一个 32 位存储字到通用寄存器 `rt` ![[截屏2026-06-30 20.27.44.png]]
1️⃣ `SignExt(imm)` 表示将 **16 位有符号立即数** `imm` 进行符号扩展为 **32 位有符号数**
2️⃣ `R(rs)` 表示编号为 rs 的通用寄存器的内容
3️⃣相加得到操作数的主存地址
Q：该指令的操作数寻址方式？
`R[rt] <- M[R[rs]+SignExt(imm)]`
A：🤨基址寻址❌
🧑‍🏫属于变址寻址![[截屏2026-06-30 20.52.26.png]]
🧑‍🏫编号为 rs 的通用寄存器作为源寄存器，用于提供操作数，是输出寄存器；编号为 rt 的通用寄存器作为目标寄存器，用于存放从主存中取到的操作数。
在 `lw` 指令的取值周期**使用了两条~={red}数据通路=~**，1️⃣一条数据通路用于从主存取出指令到指令寄存器 IR `PC->MAR->MEM->MDR->IR` 2️⃣另一条数据通路用于计算并更新程序计数器 PC 的值以便 PC 指向下一条指令 ` PC—>X->ALU->Z->PC ` 
![[截屏2026-06-30 22.45.47.png]]
🧚‍♀️基址是形式地址在变；立即数不变吧，寄存器可变，那就是变址；lw 和 sw 是 MIPS 架构下的汇编操作, 采用基址寻址模式；X 86 架构的变址寻址跟基址寻址是明确不同的，但 MIPS 仅采用基址寻址（基址加偏移量），寄存器也是基址寄存器；基址是面向系统的；基址的话那个偏移量要比较方便改变吧，这里偏移量用的是立即数；这里寄存器 rs 的值是用户指定，应该是变址; 读取存储字接近于读取数组，所以更适合变址寻址，这样是不是好理解一点![[截屏2026-06-30 20.54.16.png]]
1️⃣该指令的指令周期 `R[rt] <- M[R[rs]+SignExt(imm)]`![[截屏2026-06-30 21.08.35.png]]
🧑‍🏫如图所示，该指令前 4 个 CPU 时钟周期构成取指周期，用于从主存取出一条指令；之后 2 个 CPU 时钟周期构成计算周期，用于计算操作数的有效地址；之后 3 个 CPU 时钟周期构成执行周期，用于实现指令功能
🧚‍♀️这个计算周期是哪来的，之前将指令周期的时候好像没讲过计算周期吧？；不是间址周期吗；刚刚指令都给你说了变址寻址，哪来的间址
[chatgpt]凡是有效地址（EA）能在 CPU 内部通过寄存器和 ALU 计算出来，就不需要间址周期；只有需要先访问一次主存才能得到 EA，才需要间址周期。

为了后续描述方便，将每个 CPU 时钟周期比喻成一个节拍。
![[截屏2026-06-30 21.09.44.png]]
对于 `lw` 指令，其指令周期各个节拍所要执行的操作：
1️⃣取指周期的第 1个节拍所要执行的操作：`PC->MAR` `PC->X` 即将程序计数器 PC 的值送入存储器地址寄存器 MAR，同时送入暂存寄存器 X
2️⃣取指周期的第 2 个节拍所要执行的操作：`M[MAR]->MDR`  `X+4->Z` 以 MAR 内容为地址从主存取出指令并送入存储器数据寄存器 MDR，同时将暂存器 X 的内容与 4 相加并将运算结果送入暂存寄存器 Z ![[截屏2026-06-30 21.22.13.png]]

##### 取值周期的节拍 T 4
图中红色 1 为该节拍执行操作所需的相关控制信号，是由 **CPU 内部的操作控制器产生**的
由于控制信号 $MDR_{out}$ 为有效信号，因此其所控制的三态门打开，这样**存储器数据寄存器 MDR**的内容就会通过该三态门传送到 **CPU 内部单总线**上，经过单总线的传送 MDR 的内容会传送到 CPU 内部**与单总线相连的所有寄存器以及算术逻辑单元 ALU 的其中一个操作数输入端**
由于控制信号 $IR_{in}$ 为有效信号, 因此在该节拍中当 CPU 时钟信号的上升沿到来时，CPU 内部单总线上传送来的内容会被写入指令寄存器 IR 中，**此时 IR 中内容为之前从主存取出的那条指令**，至此就完成了取值周期 4 个节拍各自所执行的全部操作最终取到了指令![[截屏2026-06-30 22.32.55.png]]
指令经过指令译码器的译码送入操作控制器，以便为后续操作产生相应的控制信号序列，这部分内容在控制器进行具体介绍（🧚‍♀️也就是说只是把主存主存中的指令取出来）
🧚‍♀️取址阶段有多个步骤，每一个步骤的相关部件都要进行数据传输。单总线只能一步步来，因为会产生数据冲突。0，1 控制信号就是控制一步步操作不发生冲突的（控制信号真正作用是告诉各个部件在当前节拍应该做什么）；这感觉就是将 MDR 中的指令放入 IR；把这个过程想象成开闸放水和接受就行了，out 是放，in 是接收，数据流就是水流的方向
##### `rs` 、`rt`、`rd` 字段
另外指令中 rs 字段、rt 字段、rd 字段各自的值会传送到寄存器组的相关多路选择器的相应输入端（MUX = Multiplexer = 多路选择器）![[截屏2026-06-30 22.35.44.png]]
需要说明的是在 MIPS 32 指令中 rs 通常作为第一个源寄存器，用于提供指令操作的第一个操作数；rt 既可以作为第二个源寄存器提供第二个操作数，也可以作为目标寄存器来存放某些指令执行后的结果
在大多数 MIPS 指令中 rd 通常作为目标寄存器用于存放执行执行后的结果
对于 `lw` 指令寄存器组的两个多路选择器各自的选择输出控制端的控制信号为 0
![[截屏2026-06-30 22.42.19.png]]
因此他们各自的 0 号端口输入被选择输出，于是 rs 字段的值被送入寄存器组的 `R#` 端口，作为**输出寄存器的编号**；rt 字段的值被送入寄存器组 `W#` 端口，作为**写入寄存器的编号**
🧚‍♀️目标寄存器是指令结果的唯一存放位置；就是说 rs 的值是一个寄存器的编号，然后多路选择器选了 rs，就表示选中了编号为 rs 内容的寄存器作为源寄存器

#### `beq` 指令格式 
![[截屏2026-07-01 21.25.24.png]]
`beq` 指令中立即数 imm 表示**分支目标地址相对下一条指令的指令条数**
经过之前对 `lw` 指令执行流程的分析知，在当前指令的取指阶段程序计数器 PC 的值会被更新为 `PC+4 ` 将 imm 符号扩展为 32 位有符号数；再左移两位（**~={red}对于无符号整数，或者不会发生溢出的有符号整数，二进制左移两位相当于乘以 $2^2=4$ =~**）以计算出字节偏移量，再加上 PC+4 的结果即为分支目标地址 
⚠️上述操作仅简单利用 $IR_{out}$ 信号控制立即数输出到 CPU 内部总线
