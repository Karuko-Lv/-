## 六层计算机系统
第一层：数电中学习的逻辑门、寄存器等逻辑电路
第二层：微程序的基本思想是将**一条机器指令**编写成**一个微程序**；每个微程序包含若干条**微指令**-->一条机器指令包含多条微指令；每条微指令包含一条或多条**微操作**
🧑‍🏫在有微指令的系统中CPU内部有一个控制存储器，用于存放各种机器指令对应的微程序==>CPU执行机器指令会在**控制存储器**里寻找与**该机器指令对应的微程序**，找到后取出微程序中的微指令控制执行各个微操作从而完成机器指令的功能
第三层：指令集体系结构，定义一台具体的计算机可以执行的全部的机器指令的集合，定义具体的机器指令计算机应该执行什么操作以及所处理的操作数存放的位置以及操作数的类型等等...
🧑‍🏫一个机器语言程序是一个符合ISA固定的多条机器指令的序列；计算机**硬件执行**机器语言**程序**的过程即**执行**程序所包含每一条**机器指令**的过程
第四层：操作系统对计算机硬软件资源进行统一的管理和调度，方便用户使用计算机
操作系统设计人员要对机器的指令集和汇编语言很熟以及适合用于编写操作系统软件的高级语言e.gC
第五层：汇编语言层，设计人员一般是汇编语言程序设计人员，汇编语言是面向机器的语言，**每一条语句都与机器语言的某一条语句一一对应**，要求程序员对实际机器的内部组成和指令系统要熟悉
第六层：高级语言层，设计人员无需掌握机器底层技术细节![[截屏2026-06-01 21.47.15.png]]

## 机器指令、微指令、控制信号、MQ寄存器
### 第一层：CPU执行的是机器指令
例如程序里有：

```
ADD R1,R2
MUL X,Y
LOAD A
```

这些叫**机器指令（Instruction）**，存放在主存里

CPU取指后：IR ← 指令CU分析IR，控制器（CU）发现：这是一条乘法指令
### 第二层：CU不会直接完成乘法

比如：

```
MUL X,Y
```

对于硬件来说太复杂了。CPU内部要拆成很多小动作：

```
取乘数
取被乘数
判断MQ最低位
ACC加X右移
判断结束
```

这些小动作叫**微操作（Micro-operation）**

```
ACC ← ACC + X
```

就是一个微操作
### 第三层：微指令

如果是微程序控制器：
控制存储器里会存：

```
微指令1
微指令2
微指令3
...
```

每条微指令本质上是在说：

```
本拍哪些控制信号有效
```

例如：

```
ACC_out = 1
X_out = 1
ALU_add = 1
ACC_in = 1
```

意思就是：

```
ACC ← ACC + X
```
### 第四层：为什么老看到MQ？
因为乘除法离不开MQ。
教材里的经典运算器：

```
ACC
MQ
X
ALU
```

#### 乘法
例如：$13 × 11$
初始化：

```
ACC = 0000
MQ  = 1011
X   = 1101
```

执行过程：检查MQ最低位，如果：$MQ_0 = 1$
执行：

```
ACC ← ACC + X
```

然后：

```
(ACC,MQ)整体右移
```

所以微指令里经常出现：

```
MQ右移
MQ送总线
MQ最低位送控制逻辑
```
