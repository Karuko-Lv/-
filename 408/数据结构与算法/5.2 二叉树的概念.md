## 二叉树基本概念
![[截屏2026-05-04 18.02.04.png]] 
二叉树左右不可以颠倒，是有序的
⚠️和度为 2 的有序树的区分：度为 2 是至少有一个结点的度为 2，即有两个子树
‼️🆚度为 2 的有序数是二叉树吗？
A：**二叉树**是严格区分左、右的；有序树虽然孩子有顺序（老大、老二），但只有在有两个孩子的时候才体现出顺序。如果一个结点只有一个孩子则**没有左和右的概念**。
🌲是递归定义的数据结构一棵树由根结点和若干个不相交的子树组成
二叉树也是一样的
🧚‍♀️红黑树涉及到位置交换，刚学你会很迷；度为 m 的树也不一定就是 m 叉树，三叉树里面可以有节点的最大度数为 2；听起来怪怪的，但是写过树和二叉树的代码那就很明白是为什么

### 几种特殊的二叉树
#### 满二叉树
除了最底层的叶子结点其他的结点都长满了两个分支
![[截屏2026-05-04 18.07.32.png]]
只有度为 2 或 0 的结点；父结点向下取整
🧚‍♀️叶子结点是指度为 0，满二叉树肯定在最后一层
③中的特性有助于用顺序存储的方式存储结点
🌟性质：具有 $n$ 个结点的满二叉树高度 $h$ 为 $\log_2 (n+1)$
高度为 $h$ 的满二叉树的结点数 $n$ 为 $2^h-1$
#### 完全二叉树（选择题高频考点‼️）
![[截屏2026-05-04 18.14.17.png]]
满二叉树每一层结点都满了，不可能有更多的结点
完全二叉树可以在满二叉树的基础上将某一些编号大的结点去掉
~={orange}满二叉树是一种特殊的完全二叉树，完全二叉树未必是满二叉树=~
②在做选择题的时候很有用‼️
 🧚‍♀️前 n-1 层满了，最后一层可能没满，但是从左到右；可以跟栈的先进后出的特性联系起来，完全二叉树就是把满二叉树后进来的序号先去掉
![[截屏2026-05-04 18.15.10.png]]
即②中度为 1 的结点只会有左孩子!！

### 二叉树的顺序存储
```cpp
#define MaxSize 100
typedef struct{
	ElemType value;//结点中的数据元素
	bool isEmpty;//结点是否为空
}TreeNode;
TreeNode t[MaxSize];
```
数组中每个元素 TreeNode 对应二叉树中的一个结点，每个结点包含一个实际要存的数据元素（ElemType）以及一个布尔变量用于表示结点是否为空结点
🧚‍♀️ 顺序栈和顺序表不都是存数组的吗；这是先定义结点，然后再定义由这类结点组成的数组；因为 treenode 就是结构体类型的
#### 初始化数组
将所有元素的 `isEmpty` 设置为 `true`
```cpp
for(int i = 0; i<MaxSize;i++){
	t[i].isEmpty = true;
}
```
![[截屏2026-05-04 21.53.35.png]]
`t[0]` 位置空缺给完全二叉树结点编号时是从 1 开始的，从 `t[1]` 开始存储数组的下标刚好可以反映出来出完全二叉树里面的结点编号
由于数组长度有限（静态数组）==>二叉树里包含的结点数量有上限
#### 重要常考基本操作
顺序存储反应结点之间父子关系，给一个结点 i（i 是结点的编号）
1️⃣找到 i 的左孩子
2️⃣找到 i 的右孩子
3️⃣找到 i 的父结点
4️⃣知道 i 所在的层次
##### 完全二叉树
1️⃣找到 i 的左孩子 ——$2 i$
2️⃣找到 i 的右孩子 —— $2 i+1$
3️⃣找到 i 的父结点 —— $\lfloor \frac{i}{2} \rfloor$
4️⃣知道 i 所在的层次 ——  $\lceil \log_2(n+1) \rceil$ 或 $\lfloor \log_2 n \rfloor + 1$
‼️总结：将完全二叉树的结点一层层放入到数组当中，可以利用数组下标（或结点的编号）反应结点之间的逻辑关系
![[截屏2026-05-04 22.40.03.png]]
‼️可以通过比较 $i$ 和 $\lfloor \frac{n}{2} \rfloor$ 判断 $i$ 结点是否是叶子结点
##### 非完全二叉树
结点的编号无法反映结点之间的逻辑关系，所以普通二叉树不可以通过这种方案存储

### 二叉树的链式存储
```cpp
typedef struct BiTNode{
	ElemType data;//结点元素
	struct BiTNode *lchild,*rchild;//两个指针 左右孩子
}BiTNode, *BiTree;
```
没有左右孩子只需要将对应指针设置为 null
![[截屏2026-05-04 22.56.58.png]]
一共 n 个结点，则有 2 n 个指针域，其中 n-1（除根结点）结点头上有一个指针，一共 n-1 个指针==>有 n+1 个指向 null 即空链域
⚠️这些空链域可以利用构造所谓的线索二叉树
#### 初始化方式
![[截屏2026-05-05 15.24.35.png]]1️⃣定义一个空树
2️⃣插入根结点
3️⃣让根结点的左右指针指向 NULL
4️⃣插入新结点
```cpp
typedef struct{
	int value;
}ElemType;
typedef struct BiTNode{
	Elemtype data;
	struct BiTNode *lchild,*rchild;
}BiTNode,*BiTree;
BiTree root = NULL;//‼️定义一个空树
//❌root = new BiTree[1];
root = new BiTNode();
root->data = {1};
root->lchild = NULL;
root->rchild = NULL;
BiTNode * p = new BiTNode();
p->data = 2;
p->lchild = NULL;
p->rchild = NULL;
root->lchild=p;//作为根结点的左子树
```
🧚‍♀️因为 elmenType 是结构体，结构体赋值要用{}进行赋值
#### 初始化方式二——逆向找到父结点 (三叉链表)
经常需要逆向找到某一个结点的父结点则可以在结构体里定义一个指针考研中不常考
如果没有父结点想要找父结点只能依次遍历
![[截屏2026-05-05 15.29.07.png]]
```cpp
typedef struct BiTNode{
	ElemType data;
	struct BiTNode *lchild,*rchild;
	struct BiTNode *parent;//父结点指针
}BiTNode,*BiTree;
```
🧚‍♀️线索二叉树？；类似双向链表；好浪费内存啊；中文翻译真的挺过时的；那你三叉树的链表叫什么呢；用空间换时间

#### 删除二叉链表的结点
删除二叉链表并释放存储空间，由于删除一个结点前要先递归删除一个结点的左右孩子所以**用后序遍历**的访问顺序最合理
