### 算法思想
![[截屏2026-05-29 22.08.22.png]]
最简单的就是顺序依次找，并且线性表分为顺序存储和链式存储 都是可以用顺序查找的
### 实现
#### 从头到尾实现示例
例如查找表用顺序表的方式实现
```cpp
typedef struct{//查找表的数据结构：顺序存储
	ElemType *elem;//‼️动态数组的基址
	int TableLen;//表的长度
}SSTable;
//顺序查找
int Search_Seq(SSTable ST,ElemType key){
	int i;
	for(i=0;i<ST.TableLen && ST.elem[i]!=key;++i);
	//查找成功返回元素下标；失败返回-1
	return i==ST.TableLen?-1;i;
}

```
🧚‍♀️书上的更绝，上来还从后往前，就先好好写点可读性高的最有代表性的不行吗？；（~={orange}Gemini=~）在这段代码里，把 `++i` 换成 `i++` 没有任何逻辑上或运行结果上的区别；所以这种形式的for还是用while好一点
🧑‍🏫这里的示例代码用的是顺序表实现方式，实际查找表可以是链式（单链表双链表都可以），只是for循环和return语句的写法稍微有些改变
「总结」顺序表是从头到尾或者从尾到头依次对比
#### 从尾到头实现示例（哨兵）
💂哨兵是应该停止的位置
将0号位置空出来，即实际数据从下标1开始放，当查找某个关键字的时候会将关键字放到0号位置，再让指针从最后一个位置往前扫描
如果当前指向的元素的值和目标不一样`i`会一直往左移动
![[截屏2026-05-29 22.41.05.png]]
🧚‍♀️用哨兵从前往后查，你的哨兵要放在最后吧
🧑‍🏫哨兵实现方式的优点是每一轮`for`循环只需要判断当前指向的元素和要找的关键字是否相等，因为无论查找表中原来是否有想要找的关键字只要`i`值一直-，减到0的时候可以正常跳出for循环
优点：无需判断`i`值是否越界，执行效率更高（事实上高不了多少，整个算法的时间复杂度没有得到质的提升）

### 算法思想
只适用于有序的顺序表 即查找表当中各个元素关键字必须有序
![[截屏2026-05-31 18.20.20.png]]
当前检查的元素mid将目前有可能出现目标元素的整个区间一分为二
根据要找的关键字和当前mid所指的元素的大小关系
🧚‍♀️有序地顺序存储的顺序表；有点像递归啊
![[截屏2026-05-31 18.23.45.png]]
🧚‍♀️如果low=high，判断low和high指向的数是否和查找目标相等不就可以了；如果low等于high，还不存在，不可以直接跳出吗；退出条件low > high
### 实现（升序）
```cpp
typedef struct{
	int *elem;
	int TableLen;
}SSTable;//顺序表（动态数组） SS = Sequential Search（顺序查找）
int Binary_Search(SSTable L,int key){
	int low=0,high=L.TableLen-1,mid;//每个变量中间是“,”
	while(low<=high){
		mid = (low+high)/2;
		if(L.elem[mid]==key)
			return mid;
		else if(L.elem[mid]>key)
			high = mid-1;
		else
			low = mid+1;
	}
	return -1; 
}
```
「PS」上面代码**基于升序排列** 如果是降序排列判断条件得更改
🧚‍♀️类似于双指针法；为啥不递归
🧑‍🏫只有顺序表或者说数组这种形式可以根据下标立即得到中间元素在哪里，链表的中间元素只能从头依次往后找，链表没有随机存取的特性

5️⃣特性五：成功结点为$n$的判定树失败结点为$n+1$，失败结点是连到成功结点的空链域上的
