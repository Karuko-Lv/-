# 考研数据结构：DFS 与 BFS 核心应用清单

在考研中，DFS（深度优先搜索）和 BFS（广度优先搜索）如同图论算法中的“少林长拳”，虽然基础，但却能衍生出解决各种复杂问题的招式。你需要将它们不仅理解为“遍历方法”，更要理解为“问题解决工具”。

## 核心应用对比速查表

|应用场景|首选方法|备选方法|核心考点与原因|
|---|---|---|---|
|**1. 判断图的连通性**|**BFS / DFS**|并查集|一次遍历能覆盖所有节点即为连通。通常配合计数器使用。|
|**2. 求连通分量的个数**|**BFS / DFS**|并查集|外层循环调用了几次完整的遍历函数（没有被 visited 拦截），就有几个连通分量。|
|**3. 寻找无权图的最短路径**|**BFS (绝对主力)**|无|BFS 的“层级扩展”特性，保证了第一次访问到目标节点时的路径就是边数最少的路径。DFS 无法保证最短。|
|**4. 拓扑排序**|**BFS (入度法)**|DFS (出度法/回溯)|BFS 配合入度数组（Kahn 算法）最常考，最直观。DFS 依靠回溯时入栈求逆拓扑序列。|
|**5. 寻找/判断环的存在**|**DFS (首选)**|BFS (拓扑法)|DFS 利用“三色标记法”（正在访问状态 `1`）判环，并能利用调用栈打印出具体的环路。BFS 仅能判断有无环，难打印。|
|**6. 寻找两点间的所有路径**|**DFS (必选)**|无|必须使用 DFS 配合**回溯（撤销标记 `visited[v]=0`）**。这是考研大题非常喜欢出的回溯法编程题。|
|**7. 判断是否为一棵树**|**DFS / BFS**|无|连通无向图是一棵树 $\iff$ 边数 $E = V - 1$ 且全连通。遍历一次，看覆盖的边数和顶点数。|

## 详细考点解析与避坑指南

### 1. 连通性与连通分量 (最常考的选择题/填空题)

- **考试套路：** 给你一个邻接矩阵或邻接表，问你调用 `DFS()` 或 `BFS()` 几次才能遍历完。
    
- **核心逻辑：**
    
    ```
    int count = 0; // 连通分量计数器
    for (int i = 0; i < V; ++i) {
        if (!visited[i]) {
            count++;    // 发现一个新的未探索的孤岛
            DFS(i);     // 或者 BFS(i)，把这个孤岛全部染黑
        }
    }
    ```
    
- **坑点：** 如果是强连通图（有向图），任意顶点出发 1 次就能遍历完；如果是普通的弱连通有向图，从某些顶点出发可能需要多次。
    

### 2. 无权图的单源最短路径 (大题核心工具)

- **适用场景：** 题目明确说是“无权图”（或所有边权相等的图），让你求 A 到 B 的最短路径长度。
    
- **核心逻辑：** 必须用 **BFS**。BFS 的搜索像水波纹一样一层层荡开。
    
    - 增加一个数组 `dist[v]` 记录到起点的距离，初始化为无穷大，`dist[start] = 0`。
        
    - 在出队遍历邻居 `w` 时：`if(!visited[w]) { dist[w] = dist[v] + 1; enqueue(w); }`
        
- **坑点：** 千万不要用 DFS 求无权最短路径！DFS 是一条路走到黑，找到的往往是很绕的路。虽然用 Dijkstra 也能解无权最短路径，但杀鸡用牛刀，时间复杂度不如直接 BFS 优秀（BFS 是 $O(V+E)$）。
    

### 3. 判断图是否有环 (高频代码题)

- **无向图判环：** 用 DFS。在遍历时传入当前节点的“父节点 (parent)”。如果遇到了一个 `visited` 为 true 的邻居，**并且这个邻居不是它的父节点**，那就说明有环。
    
- **有向图判环：** 必须用 DFS + **三色标记法**（状态 0, 1, 2）。遇到状态为 1（正在调用栈中）的节点即为有环。（参考之前的讨论）。或者使用 BFS 拓扑排序看最后输出的顶点数是否等于总顶点数。
    

### 4. 寻找两点间的“所有”简单路径 (压轴大题)

- **考试套路：** 给定图和起点终点，要求打印出所有可能的路线。
    
- **核心逻辑：** **DFS + 回溯**。这是区分中等生和拔尖生的分水岭。
    
    ```
    void FindAllPaths(int u, int dest, int path[], int pathLen) {
        visited[u] = true;         // 1. 踩下去
        path[pathLen++] = u;       //    记录路径
    
        if (u == dest) {           // 2. 找到终点，打印路径
            Print(path, pathLen);
        } else {
            for (int v = FirstNeighbor(u); v >= 0; v = NextNeighbor(u, v)) {
                if (!visited[v]) {
                    FindAllPaths(v, dest, path, pathLen); // 3. 递归深入
                }
            }
        }
    
        visited[u] = false;        // 4. 【核心灵魂】回溯：擦掉脚印，准备走别的岔路
        // pathLen 会在返回上一层调用栈时自动恢复，不需要手动减
    }
    ```
    

### 5. 判断二分图 (Bipartite Graph)

- **虽然大纲不常显式考，但在自主命题中偶尔作为难题出现。**
    
- **核心逻辑：** BFS 或 DFS 都可以。使用“染色法”。
    
- 给起点染红色，它的所有直接邻居染蓝色，邻居的邻居再染红色。
    
- 如果在染色的过程中，发现要染的节点已经有颜色了，**且颜色跟我要给它染的颜色冲突**（比如两个相邻的节点都是红色），那这就不是二分图。
    

## 记忆口诀

- **找最短，用 BFS。** (像水波纹，最先到达的就是最短的)
    
- **找所有路径，用 DFS 加回溯。** (像走迷宫探死胡同，必须退出来换路)
    
- **无向连通分量，数大循环。**
    
- **有向图找环，必用三色状态。**
