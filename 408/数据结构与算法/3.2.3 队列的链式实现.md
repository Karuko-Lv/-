1️⃣关注是否要求带头结点；
2️⃣第一个元素入队和最后一个元素出队需要一些特殊的处理，需要特别注意修改 `rear` 指针；
3️⃣其他和单链表没有太大区别可以和单链表内容比对学习
队列是单链表的阉割版
队列中由于每次插入元素都只能在表尾的位置插入，因此可以用专门的尾指针指向最后一个结点
出队：从头结点找到第一个数据结点，将其删除即可

## 初始化（带头结点）
![[截屏2026-04-14 22.18.55.png]]
```cpp
typedef struct LinkNode{
	int data;
	struct LinkNode *next;
}LinkNode;

typedef struct{
	LinkNode *front,*rear;
}LinkQueue;

void InitQueue(LinkQueue &Q){
	//初始化 front、rear指向头结点
	Q.front = Q.rear = new LinkNode();
	Q.front->next = nullptr;
}

void testLinkQueue(){
	LinkQueue Q;//声明一个队列 
	InitQueue(Q);//初始化队列
}
```
### C 语言基础
Q：为什么一会用Q.front 一会儿用->

A：
- **普通对象或引用** 访问其成员时，用点操作符 **`.`**
- **指针** 访问其所指向对象的成员时，用箭头操作符 **`->`**
1️⃣引用相当于原变量的一个别名，它的用法和普通的结构体变量一模一样，**它不是指针**。
因为 `Q` 是一个 `LinkQueue` 类型的引用（你可以把它当成一个普通对象看待），所以你要访问这个结构体里面的 `front` 和 `rear` 成员时，必须使用点操作符：
- `Q.front`
- `Q.rear`
2️⃣ `front` 和 `rear` 的类型都是 `LinkNode *`。那个 `*` 符号明确告诉你：**`front` 是一个指针**，它指向内存中的某个 `LinkNode` 节点。
3️⃣LinkQueue &Q是一个结构体类型的变量

🧚‍♀️尾指针的循环链表比较好
「PS」有自引用所以需要 `typedef struct [结构体真名] { 成员变量... } [类型别名];`；没有自引用可以偷懒，不写“结构体真名”，直接让编译器创建一个匿名的结构体，然后立刻用 `typedef` 给它贴上一个叫 `LinkQueue` 的别名标签

## 入队（带头结点）
新的数据元素入队，该数据元素包含在一个结点当中，所以申请一个新的结点
```cpp
void EnQueue(LinkQueue &Q,int x){
	LinkNode *s = new LinkNode();
	s->data = x;
	s->next = nullptr;
	Q.rear->next = s;//新结点插入到rear之后
	Q.rear = s;//修改表尾指针
}
```
新插入的结点一定是队列当中的最后一个结点，所以新结点的 `next` 指针域要设置为 `nullptr`
由于 `rear`  指针指向的是当前的表尾结点而新插入的结点需要连接到当前的表尾结点之后是新的表尾结点

## 入队（不带头结点）
插入第一个数据元素（第一个数据元素入队）的时候需要特殊处理，因为刚开始 rear 和 front 都是指向 `null` 的，插入第一个元素的时候这两个指针都要进行修改, 如果不是第一个只需要修改 rear 就可以了
```cpp
void EnQueue(LinkQueue &Q,int x){
	LinkNode *s = new LinkNode();//将新的结点s插入链表当中
	s->data = x;
	s->next = nullptr;
	if(Q.front == nullptr){//不带头结点判断是否为空
		Q.front = s;
		Q.rear = s;
	}else{
		Q.rear->next = s;//先入队后该尾结点
		Q.rear = s;//rear指针指向新结点
	}
}
```

## 出队（带头结点）
```cpp
//队头元素出队（带头结点）
bool DeQueue(LinkQueue &Q,int &x){
	if(Q.front == Q.rear)//防下溢 空队
		return false;
	LinkNode *p = Q.front->next;
	x = p->data;
	Q.front->next = p->next;
	return true;
}
```
❌错误 1️⃣：忘记 free 变量了
错误 2️⃣：忘了边界情况，当出队之后队为空了尾指针和头指针一并指向头结点了，需要修改尾指针了，因为尾指针原来指向的地方被删除了
Re
```cpp
bool DeQueue(LinkQueue &Q,int &x){
	if(Q.front==Q.rear)
		return false;
	LinkNode *p = Q.front->next;
	x = p->data;
	Q.front->next = p->next;
	if(Q.rear==p)//‼️表示是最后一个结点出队
		Q.rear = Q.front;
	delete p;
	return true;
}
```

## 出队（不带头结点）
```cpp
bool DeQueue(LinkQueue &Q,int &x){
	if(Q.front == nullptr)
		return false;
	LinkNode *p = Q.front;
	x = p->data;
	Q.front = p->next;
	if(Q.rear == p){
		Q.rear = nullptr;
		Q.front = nullptr;
	}
	delete p;
	return false;
}
```

## 队列满的条件

1️⃣**顺序存储**方式实现循环队列-->取余数
循环队列分配的空间是提前预分配-->用静态数组的方式分配-->存储空间有限、不可扩展

2️⃣**链式存储**的容量扩展很方便，只要内存资源还足够就可以继续给队列扩容==>链式存储不用关心列满问题一般不会
