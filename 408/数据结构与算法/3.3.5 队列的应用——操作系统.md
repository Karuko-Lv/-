## Os 先来先服务
操作系统需要管理系统中一些硬件资源——CPU 、某些 IO 设备
系统资源往往是有限的，系统当中同时会有多个进程并发运行，这些进程会争抢来使用系统资源
===>os 如何分配系统资源？
哪个进程先申请系统资源就先把资源分配给这个进程
![[截屏2026-04-17 20.13.35.png]]
CPU 资源有限，但是系统中可能同时启动多个进程，os 会让这些进程排成一个所谓的就绪队列，每次选择队头元素上 cpu 执行一个比较短的时间片然后迅速下 CPU，接下来让下一个进程上 cpu 运行... 循环往复
之后所有的进程可以轮流得到 CPU 的服务
-->一个电脑只有一个 cpu 却同时可以有很多程序运行的原因——离不开 os 的管理策略支持

## 打印数据缓冲区
打印机还在工作就可以按下打印按钮，因为系统开辟了一片缓冲区用于存放暂时还不能处理的数据，用队列将暂时不能被处理的任务组织起来，一定程度上缓解主机和打印机速度不匹配的问题，打印机是一个慢速设备电脑主机是一个很快速的设备，可以在短时间内发出大量的打印任务但是打印机处理不了就将需要打印的数据放到缓冲区里，比较合理的方式——用队列将数据组织起来（就绪队列）
