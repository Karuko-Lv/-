### 最小生成树的概念
生成树是什么？==>连通图的生成树是**包含**图中**全部顶点**的一个**极小连通子图**（n个顶点保持连通同时边的数量=n-1），一个连通图可能会有多个生成树
通过不同的遍历（DFS 或 BFS）可以确定不同形态的生成树

### 最小生成树的算法
考研初试这两个算法考查代码的概率不高，着重掌握如何手动的方式模拟这两个算法的执行过程
#### Prim 算法（普里姆）
1️⃣挑选图中的某一个顶点（任何一个都可以）
2️⃣将代价最小的新的（和生成树相邻）顶点纳入生成树中‼️
3️⃣所有顶点都纳入即完成算法

![[截屏2026-05-16 11.31.00.png]]
「PS」同一个树可能会有多个最小生成树，虽然形态不同但是最小生成树的所有边的权值之和肯定一样并且最小
🧚‍♀️贪心算法，局部最优；广度优先搜索？；动态规划？
#### Kruskal 算法（克鲁斯卡尔）
1️⃣每次挑选一条权值最小的边
2️⃣使边的两头相连通，但如果原本已经连通就不选

#### Prim 算法🆚Kruskal 算法
Prim 算法每次选一个顶点，Kruskal 算法每次选一个边
##### 时间复杂度
Prim 算法时间复杂度只和顶点的个数有关，是顶点个数的二次方， $O(|V|^2)$，比较适合**边稠密**即 $E$ 的值比较大的场景可以得到更好的效率
🧚‍♀️可以用堆优化成 O（VlogV） 
Kruskal 算法时间复杂度只和边的数量有关系, $O(|E|log_2|E|)$，比较适合**边稀疏**即 $E$ 的值比较小的场景

###### 代码
```cpp
#include <iostream>
#include <vector>

const int INF = 0x3f3f3f3f; // 用一个大常数表示无穷大 (即两个顶点不直接相连)

// graph 传入的是邻接矩阵，start 是起始顶点的下标 (如图片中的 V0 就是 0)
void Prim(const std::vector<std::vector<int>>& graph, int start) {
    int n = graph.size(); // 顶点总数
    
    // 初始化两个核心数组，与图片完全对应
    std::vector<bool> isJoin(n, false);
    std::vector<int> lowCost(n, INF);

    // ==========================================
    // 第一步：初始化
    // ==========================================
    isJoin[start] = true; // 将起始点(V0)标记为已加入
    
    // 初始化 V0 到其他各个顶点的初始代价
    // 此时对应图片右下角表格的第一轮状态：lowCost[6] = {0, 6, 5, 1, ∞, ∞}
    for (int i = 0; i < n; ++i) {
        if (i != start) {
            lowCost[i] = graph[start][i]; 
        } else {
            lowCost[i] = 0; // 自己到自己的代价是 0
        }
    }

    int totalWeight = 0; // 记录最小生成树的总权值

    // ==========================================
    // 第二步：开始循环，总共需要处理 n-1 轮（因为要加入剩下 n-1 个点）
    // ==========================================
    for (int i = 1; i < n; ++i) {
        int minCost = INF;
        int minIndex = -1;

        // 【操作1】：循环遍历所有节点，找到 lowCost 最低的且还没加入树的顶点
        // 这一步的时间复杂度是 O(n)
        for (int j = 0; j < n; ++j) {
            if (!isJoin[j] && lowCost[j] < minCost) {
                minCost = lowCost[j];
                minIndex = j;
            }
        }

        // 如果找不到，说明图不连通，无法生成完整的树
        if (minIndex == -1) {
            std::cout << "无法生成最小生成树，图不连通！" << std::endl;
            return;
        }

        // 找到后，将该顶点加入树
        isJoin[minIndex] = true;
        totalWeight += minCost;
        std::cout << "第 " << i << " 轮: 加入顶点 V" << minIndex 
                  << "，当前相连边权值为 " << minCost << std::endl;

        // 【操作2】：再次循环遍历，更新还没加入的各个顶点的 lowCost 值
        // 这一步的时间复杂度也是 O(n)
        for (int j = 0; j < n; ++j) {
            // 条件：顶点 j 还没有加入树，并且新加入的 minIndex 顶点到 j 的权值，比原来记录的 lowCost 更小
            if (!isJoin[j] && graph[minIndex][j] < lowCost[j]) {
                lowCost[j] = graph[minIndex][j]; // 更新最低代价
            }
        }
    }

    std::cout << "最小生成树构建完成，总权值为: " << totalWeight << std::endl;
}

int main() {
    // 构造图片中的图数据 (6个顶点 V0~V5)
    int n = 6;
    std::vector<std::vector<int>> graph(n, std::vector<int>(n, INF));
    
    // 对角线为 0
    for(int i=0; i<n; i++) graph[i][i] = 0;

    // 录入图片中的边信息 (无向图需对称赋值)
    graph[0][1] = 6; graph[1][0] = 6;
    graph[0][2] = 5; graph[2][0] = 5;
    graph[0][3] = 1; graph[3][0] = 1;
    graph[1][3] = 5; graph[3][1] = 5;
    graph[1][4] = 3; graph[4][1] = 3;
    graph[2][3] = 4; graph[3][2] = 4;
    graph[2][5] = 2; graph[5][2] = 2;
    graph[3][4] = 6; graph[4][3] = 6;
    graph[3][5] = 4; graph[5][3] = 4;
    graph[4][5] = 6; graph[5][4] = 6;

    // 从 V0(下标0) 开始执行 Prim 算法
    Prim(graph, 0);

    return 0;
}
```
外层循环执行了 $n-1$ 次。内层有两个并列的 `for` 循环，每个循环执行 $n$ 次。所以总时间是 $(n-1) \times (n + n) \approx 2n^2$，去掉常数后就是 $O(|V|^2)$。
###### 堆优化的 Prim 算法
**把“傻傻遍历”换成“优先队列（小顶堆）”，并把存储结构换成“邻接表”。** 是著名的 **堆优化的 Prim 算法**。
优先队列 (Min-Heap) + 邻接表
- **不用邻接矩阵，改用邻接表**：因为优化版更适合处理**稀疏图**，用邻接表遍历邻居更快，不用面对矩阵里大量的无穷大（无边）数据。
- **用优先队列（堆）自动排序**：我们不再维护一个静态的 `lowCost` 数组去每次循环查找最小值。而是把 `(边权值, 目标顶点)` 作为一个包裹，扔进 C++ 的 `std::priority_queue`（底层是小顶堆）里。
- **自动获取最小边**：堆结构有一个极其强大的特性——~={red}无论你往里面扔多少东西，**它堆顶永远是权值最小的那个**。=~每次取最小边的时间复杂度从 $O(|V|)$ 骤降到了 **$O(\log |E|)$**。
```cpp
#include <iostream>
#include <vector>
#include <queue>

using namespace std;

const int INF = 0x3f3f3f3f;

// 定义存储在优先队列里的结构：包含权值和到达的顶点
struct Node {
    int cost;
    int v;
    // 优先队列默认是大顶堆，为了变成小顶堆（让 cost 小的排前面），需要重载大于号
    bool operator>(const Node& other) const {
        return cost > other.cost;
    }
};

// 堆优化的 Prim 算法
void PrimOptimized(int n, const vector<vector<pair<int, int>>>& adj, int start) {
    // 记录顶点是否已经加入 MST
    vector<bool> isJoin(n, false);
    
    // 定义一个小顶堆
    priority_queue<Node, vector<Node>, greater<Node>> pq;
    
    // 1. 初始化：把起点先放进堆里，距离自己代价为 0
    pq.push({0, start});
    
    int mst_weight = 0;
    int edge_count = 0;

    // 2. 开始不断从堆中取最小的边
    while (!pq.empty()) {
        // 取出当前距离 MST 最近的节点
        Node current = pq.top();
        pq.pop();
        
        int u = current.v;
        int cost = current.cost;
        
        // 【核心判断】：如果这个点已经被拉入伙了，直接跳过（这是堆优化独有的废边处理逻辑）
        if (isJoin[u]) continue;
        
        // 否则，正式把它拉入 MST
        isJoin[u] = true;
        mst_weight += cost;
        edge_count++;
        
        // 如果已经加入了 n 个点，生成树构建完毕，提前退出
        if (edge_count == n) break;
        
        // 3. 扫描刚加入的节点 u 的所有邻居
        for (auto& edge : adj[u]) {
            int neighbor = edge.first;
            int weight = edge.second;
            
            // 如果邻居还没加入 MST，就把这条到达邻居的候选边扔进堆里
            // 堆会自动帮我们把权值最小的边浮到最上面
            if (!isJoin[neighbor]) {
                pq.push({weight, neighbor});
            }
        }
    }
    
    if (edge_count == n) {
        cout << "堆优化Prim构建完成，总权值为: " << mst_weight << endl;
    } else {
        cout << "图不连通，无法构建完整的最小生成树！" << endl;
    }
}
```
优化后，总的时间复杂度变成了 **$O(|E| \log |V|)$** （因为最多把 $E$ 条边扔进堆里，堆调整代价是 $\log V$）。
##### Kruskal 算法
1️⃣初始：将各条边按照权值递增的排序
需要将所有的边全部检查一遍
2️⃣检查第一条边的两个顶点是否连通（**并查集**：刚开始将所有结点分别看成不同的集合，对于第一条边的两个顶点刚开始是从属于不同集合的，即原本不连通，不连通就可选上，之后这两个顶点就属于同一个集合了），不连通就连起来
![[截屏2026-05-16 11.58.28.png]]
🧚‍♀️就跟之前的并查集找祖宗根节点一样，找到的两点的根节点一样就属于同一集合
「总结」一共有 $e$ 个边，所以需要执行 $e$ 轮，每轮用并查集判断两个顶点是否属于同一个集合需要 $O (log_2 e)$ ，总时间复杂度 $O (elog_2 e)$
1. **排序阶段**：对 $E$ 条边排序，耗时 **$O(E \log_2 E)$**。
2. **并查集阶段**：遍历这 $E$ 条边，每次用并查集判断边的两个顶点是否连通。因为优化后的并查集单次操作是常数 $O(1)$，所以处理 $E$ 条边的并查集总耗时大约是 **$O(E \times 1) = O(E)$**。
把这两部分加起来，总时间是：$O(E \log_2 E) + O(E)$。
🧚‍♀️优化后的并查集时间复杂度可以视为常数，log 2 E 是排序的时间复杂度；王道书上讲，每次选择最小权值的边需要 log 2 e，查和并的操作由于优化之后时间复杂度其实可视为常数；因为并查集树高不超过 log 2 e
在 Kruskal 算法的整个执行过程中，时间主要花在两件事上：**给边排序** 和 **使用并查集判断连通性**。
###### 代码
```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

// 1. 定义边的结构体
struct Edge {
    int u;      // 顶点1 (Vertex1)
    int v;      // 顶点2 (Vertex2)
    int weight; // 权值
    
    // 重载小于号，用于后续的快速排序 (按权值升序)
    bool operator<(const Edge& other) const {
        return weight < other.weight;
    }
};

// 2. 并查集 (Disjoint Set) 的实现
const int MAXV = 100; // 最大顶点数
int parent_node[MAXV];

// 并查集：初始化，一开始每个顶点各自为一个集合
void init(int n) {
    for (int i = 0; i < n; i++) {
        parent_node[i] = i; 
    }
}

// 并查集：查找根节点 (包含路径压缩优化，即前面提到的 O(1) 优化)
int find(int x) {
    if (x == parent_node[x]) {
        return x;
    }
    // 路径压缩：直接将当前节点连到根节点上
    return parent_node[x] = find(parent_node[x]);
}

// 并查集：合并两个集合
void join(int x, int y) {
    int rootX = find(x);
    int rootY = find(y);
    if (rootX != rootY) {
        parent_node[rootX] = rootY;
    }
}

// 3. Kruskal 算法主体
void Kruskal(int n, vector<Edge>& edges) {
    // 【步骤 1】：将各条边按照权值递增的排序 (时间复杂度 O(E log E))
    sort(edges.begin(), edges.end());
    
    // 初始化并查集
    init(n);
    
    int mst_weight = 0; // 记录最小生成树的总权值
    int edge_count = 0; // 记录加入最小生成树的边数

    cout << "开始检查边..." << endl;
    
    // 【步骤 2】：检查第一条边到最后一条边
    for (int i = 0; i < edges.size(); i++) {
        int u = edges[i].u;
        int v = edges[i].v;
        int w = edges[i].weight;
        
        // 检查边的两个顶点是否连通（是否属于同一个集合）
        if (find(u) != find(v)) {
            // 不连通就选上，并连起来
            cout << "选中边: V" << u << " - V" << v << " (权值: " << w << ")" << endl;
            join(u, v);
            mst_weight += w;
            edge_count++;
            
            // 如果已经选了 n-1 条边，说明最小生成树已经构建完毕，可以提前退出
            if (edge_count == n - 1) {
                break;
            }
        } else {
            // 连通则跳过 (对应图片中绿色的爪印和红字 "已连通，跳过！")
            cout << "跳过边: V" << u << " - V" << v << " (权值: " << w << ") -> 已连通，跳过！" << endl;
        }
    }
    
    cout << "最小生成树构建完成，总权值为: " << mst_weight << endl;
}

int main() {
    int n = 6; // 顶点数 (V0 到 V5)
    
    // 录入图片表格中的边数据
    vector<Edge> edges = {
        {0, 3, 1}, {2, 5, 2}, {1, 4, 3}, {2, 3, 4}, 
        {3, 5, 4}, {0, 2, 5}, {1, 3, 5}, {0, 1, 6}, 
        {3, 4, 6}, {4, 5, 6}
    };
    
    Kruskal(n, edges);
    
    return 0;
}
```

##### 代码实现
从广度优先算法进行改造
```cpp
bool visited[MAX_VERTEX_NUM];//访问标记数组
//广度优先遍历
void  BFS(Graph G,int v){
	visit(v);
	visited[v]=TRUE;
	Enqueue(Q,v);
	while(!Empty(Q)){
		DeQueue(Q,v);
		for(w=FirstNeighbor(G,v);w>=0;w=NextNeighbor(G,v,w)){
			if(!visited[w]){
				visit(w);
				visited[w]=TRUE;
				EnQueue(Q,w);
			}
		}
	}
}
```
👆代码用 `visit` 函数抽象表示“对某一个顶点的访问”，改造成记录结点顺序

```cpp
	while(!isEmpty(Q)){
		DeQueue(Q,a);
		for(w=FirstNeighbor(G,a);w>=0;w=NextNeighbor(G,a,w)){
			if(!visited[w]){
				d[w]=d[a]+1;
				path[w]=a;
				visited[w]=TRUE;
				EnQueue(Q,w);
			}
		}
	}
}
```

🤨根据上面的代码，进行手算推理，首先从 1 开始出发，有三个数组，一个用来记录 1 到各个结点的最短路径；另一个用来记录 1 到各个结点的最短路径是从那个顶点过来的（这里的第二个数组我有点不理解）；还有一个数组又来记录结点是否被访问过
那么一开始从 1开始走（a=1），这个结点的第一个数组值为 0 即 1 到 1 自身的最短路径是 0；第二个数组值不用记，第三个数组值改为 true；之后 1 进队列，找到 1 的第一个相邻节点 w=2，2 没有被访问过则 1 到 2 的距离为 1，第二个数组值为 1 从 1 来的，第三个数组值改为 true；之后 w=5...；之后因为 1 顶点的所有相邻结点被访问完了之后，Q 队列中是 2，5 再从 2 开始遍历，再记录...
