## 结构体
```cpp
typedef struct [结构体标签名] { // 成员变量 } [类型别名];
```
`typedef` 的语法结构：`typedef [原类型] [新别名];`。
如果在 `struct` 后面没有写名字，这被称为**匿名结构体**。这种做法通常用于**结构体内部不需要引用自己**的情况。
```cpp
typedef struct {
    int x;
    int y;
} Point; 

// 之后可以直接用 Point 声明变量： Point p1;
```
### 数组
```cpp
typedef struct VNode{
	VertexType data;
	ArcNode *first;
}VNode,AdjList[MaxVertexNum]; 
```
1️⃣第一个别名
```cpp
VNode v1;  // 完全等价于 struct VNode v1;
```
2️⃣第二个别名
```cpp
AdjList graph; 
// 完全等价于 struct VNode graph[MaxVertexNum];
// 也等价于 VNode graph[MaxVertexNum];
```

这么写纯粹是为了**提高代码的可读性和抽象性**，让你在写后续代码时“见名知意”：
- 当想表达“这里只有一个单一的顶点”时，你就用： `VNode node;`
- 当想表达“这是一个包含了所有顶点的图（邻接表）”时，你就用： `AdjList graph;`
## `malloc` 和 `new`
‼️ `malloc` 和 `new` 都是用来向操作系统的**堆区（Heap）**“借”内存的，通常用于不知道要申请多少内存的情况下，会在**堆区（Heap）** 动态申请内存
以二叉树数据结构最标准定义为例子
```cpp
typedef struct BiTNode {
    int data;
    struct BiTNode *lchild, *rchild;
} BiTNode, *BiTree;
```
「PS」 `BiTNode` 是一个结构体（树的节点），而 `BiTree` 是一个指针（指向节点的指针，等价于 `BiTNode*`）
初始化需要在空树基础上插入根结点

| 比较维度       | `malloc` / `free` (C语言) | `new` / `delete` (C++语言)      |
| ---------- | ----------------------- | ----------------------------- |
| **本质属性**   | 是**标准库函数**              | 是 C++ 的**运算符** (关键字)          |
| **大小计算**   | 必须手动用 `sizeof` 计算字节数    | 编译器自动计算，只需提供数据类型              |
| **返回类型**   | 返回 `void*`，必须手动强转       | 返回具体类型的指针，绝对安全                |
| **⭐对象初始化** | **不调用构造函数！** 只分配生硬的内存   | **分配内存后，会自动调用类的构造函数！**        |
| **⭐对象销毁**  | 用 `free` 时，**不调用析构函数！** | 用 `delete` 时，**会自动调用类的析构函数！** |
| **失败处理**   | 申请失败返回 `NULL`           | 申请失败抛出 `std::bad_alloc` 异常    |
| **内存扩容**   | 有 `realloc` 函数可以扩容原来的内存 | 没有直接对应的扩容语法，需手动重新 new         |

`malloc` 家族里有一个 `realloc` 函数，可以**就地扩大**一块连续内存的大小（常用于动态数组的底层实现），而 `new` 做不到这一点
### `malloc` 写法
`malloc` 是 "memory allocate" 的缩写，它是一个标准库函数（需要引入 `<stdlib.h>`）
```cpp
//1️⃣定义空树
BiTree root = null;
//2️⃣定义根结点
root = (BiTree)malloc(sizeof(BiTNode));
```
✅标准语法
```cpp
指针变量 = (类型强转 *)malloc(需要申请的字节数);
```
`malloc` 返回的是一个毫无感情的 `void*`（无类型指针）
**判空与释放。** 内存可能会申请失败（虽然很少见），用完后**必须**用 `free()` 释放，否则会内存泄漏！
```cpp
#include <stdio.h>
#include <stdlib.h> // malloc 和 free 在这个头文件里

int main() {
    // 1. 申请一个 int 的空间
    int* p1 = (int*)malloc(sizeof(int));
    *p1 = 100;

    // 2. 申请一个长度为 10 的 int 数组空间‼️🌟
    int* arr = (int*)malloc(sizeof(int) * 10);
    arr[0] = 1; 

    // 3. 申请失败检查（严谨的写法）
    if (arr == NULL) {
        printf("内存申请失败！\n");
        return -1;
    }

    // 4. 用完必须释放！
    free(p1);
    free(arr);

    return 0;
}
```
### `new` 写法
`new` 是 C++ 内置的**运算符**（就像 `+`、`-` 一样），它能自动计算大小并转换类型
```cpp
root = new BiTree();
```
意思是：实例化一个 `BiTNode` 结构体，并把它的指针赋给 `root`
❌错误写法
```cpp
root = new BiTree[1];
```
意思是：分配一个数组，这个数组里装了 1 个指针。返回的是二级指针 `BiTNode`，在内存中只分配了 4 个或 8 个字节（一个指针的大小），而**不是**一整个树节点的大小
`new` 后面应该跟着想创建的实体类型

```cpp
// 1. 申请一个 int 的空间
int* p1 = new int;      

// 2. 申请一个 int 并初始化为 100
int* p2 = new int(100); 

// 3. 申请一个长度为 10 的 int 数组
int* arr = new int[10]; 

// 4. 释放空间
delete p1;
delete p2;
delete[] arr; // ‼️‼️注意：释放数组必须加 []
```
## 指针大小
指针的大小与操作系统的位数有着非常直接的对应关系，但严格来说，它最终取决于**编译器将程序编译为多少位的目标平台**

在绝大多数情况下，我们看到的对应关系如下：

- **32 位操作系统：** 指针大小为 **4 个字节**（32 bit）。
- **64 位操作系统：** 指针大小为 **8 个字节**（64 bit）。
