## 广度优先遍历
树的广度优先遍历就是层序遍历，查找结点尽可能往横向找
### 树 vs 图
**相似**：无论是树还是图进行广度优先遍历都需要实现**通过某一个结点找到与之相邻的其他结点**，只有实现这个操作才能够一层一层的往下找到所有的结点
**不同**：
1️⃣对于树来说要**找到与一个结点相连的其他结点**即找他的孩子即可；对于图来说可以用 `FirstNeighbor` 和 `NextNeighbor` 这两个基本操作来完成
2️⃣对于树各个结点之间的路径不存在回路所以通过树中的某一个结点搜索其他结点的时候搜到的结点一定是之前没有访问过的结点；对于图可能出现环路，且无向图每一条边可以正向和逆向走，所以**可能会访问到之前已经访问过的结点**，这种情况需要特殊处理，**处理思路简单**即给各个结点进行一个标记即可

回忆：实现树的广度优先遍历也即层序遍历的时候需要一个**辅助队列**的帮助，**因为计算机一次只能处理一个结点，暂时没有被处理的结点需要用队列保存起来**

🧚‍♀️层次是广度，先根后根是深度；设立标志位保证每个结点只被访问过一次
### 代码实现
通过 `FirstNeighbor` 和 `NextNeighbor` 这两个基本操作可以实现找到与一个顶点相连接的所有的顶点（第一个实现找到指定顶点的第一个临界点，找不到返回-1；第二个基本操作可以找到后一个与之相连接的顶点）
PS：这两个基本操作用不同的存储结构实现起来会有所不同
这两个操作（`FirstNeighbor` 和 `NextNeighbor`）是图论中最经典的接口设计。它们就像是图遍历算法（如 BFS 和 DFS）的“眼睛”，决定了算法如何去探索周围的顶点。

不同的存储结构在这两个操作上的逻辑截然不同，直接决定了图遍历算法的整体时间复杂度。我们来做一个全方位的硬核对比：
#### `FirstNeighbor(G, x)`：寻找第一个邻居

**操作目的：** 站在顶点 $x$，环顾四周，找到与之相连的第一个顶点。

| **维度**    | **邻接矩阵 (Adjacency Matrix)**                              | **邻接表 (Adjacency List)**                                   |
| --------- | -------------------------------------------------------- | ---------------------------------------------------------- |
| **执行逻辑**  | 定位到第 $x$ 行，从第 $0$ 列开始向右**逐个扫描**。遇到的第一个非 0 元素的列号，就是第一个邻居。 | 直接定位到顶点 $x$ 的结构体，**读取头指针** `*first`。如果不为空，该指针指向的结点就是第一个邻居。 |
| **代码意象**  | `for(i=0 to V-1) { if(edge[x][i]) return i; }`           | `if(p=first) return p->adjvex;`                            |
| **时间复杂度** | $O (1)$                                                  | $O (V）$                                                    |
| **优势**    | 逻辑简单，基于数组连续内存，现代 CPU 缓存命中率高。                             | 极致的查询速度。                                                   |
