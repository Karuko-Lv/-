## 广度优先遍历
树的广度优先遍历就是层序遍历，查找结点尽可能往横向找
### 树 vs 图
**相似**：无论是树还是图进行广度优先遍历都需要实现**通过某一个结点找到与之相邻的其他结点**，只有实现这个操作才能够一层一层的往下找到所有的结点
**不同**：
1️⃣对于树来说要**找到与一个结点相连的其他结点**即找他的孩子即可；对于图来说可以用 `FirstNeighbor` 和 `NextNeighbor` 这两个基本操作来完成
2️⃣对于树各个结点之间的路径不存在回路所以通过树中的某一个结点搜索其他结点的时候搜到的结点一定是之前没有访问过的结点；对于图可能出现环路，且无向图每一条边可以正向和逆向走，所以**可能会访问到之前已经访问过的结点**，这种情况需要特殊处理，**处理思路简单**即给各个结点进行一个标记即可

回忆：实现树的广度优先遍历也即层序遍历的时候需要一个**辅助队列**的帮助，**因为计算机一次只能处理一个结点，暂时没有被处理的结点需要用队列保存起来**

🧚‍♀️层次是广度，先根后根是深度；设立标志位保证每个结点只被访问过一次
### 代码实现
通过 `FirstNeighbor` 和 `NextNeighbor` 这两个基本操作可以实现找到与一个顶点相连接的所有的顶点（第一个实现找到指定顶点的第一个临界点，找不到返回-1；第二个基本操作可以找到后一个与之相连接的顶点）
PS：这两个基本操作用不同的存储结构实现起来会有所不同
这两个操作（`FirstNeighbor` 和 `NextNeighbor`）是图论中最经典的接口设计。它们就像是图遍历算法（如 BFS 和 DFS）的“眼睛”，决定了算法如何去探索周围的顶点。
![[截屏2026-05-14 21.52.39.png]]
```cpp
bool visited[MAX_VERTEX_NUM];//访问标记数组
void BFS(Graph G,int v){//‼️从顶点v出发广度优先遍历图G
	visit(v);
	visited[v]=TRUE;//1️⃣首先访问v结点并设置2号结点对应的visit数组为true
	EnQueue(Q,v);//2️⃣让v结点入队Q
	while(!isEmpty(Q)){
		DeQueue(Q,v);//3️⃣while循环，如果队列不空让队头元素出队
		for(w=FirstNeighbor(G,v);w>=0;w=NextNeighbor(G,v,w)){//4️⃣通过v结点找到与之相邻的所有的顶点（利用两个基本操作‼️）
			if(!visited[w]){
				visit(w);
				visited[w]=TRUE;
				EnQueue(Q,w);//5️⃣让没有被访问过的与v相邻的结点入队，且设置结点为访问过的再入队
			}//if
		}
	}//while
}
```

#### 缺点
如果是非连通图无法遍历完全部的结点
Q：怎么处理
A：用 `visited` 数组记录所有的顶点是否被访问的信息，在第一次调用 `BFS` 函数之后可以检查一下 visited 数组中能否找到值为 `false` 的顶点了，如果找到了从找到的值为 `false` 的顶点出发在调用一次 ` BFS ` 函数即可

#### Final 版本
```cpp
bool visited[MAX_VERTEX_NUM];//访问标记数组
void BFSTraverse(Graph G){//对图G进行广度优先遍历
	for(i=0;i<G.vernum;++i)
		visit[i]=FALSE;//1️⃣将visited数组全部设为false（初始化visited数组）
	InitQueue(Q);//2️⃣初始化辅助队列
	for(i=0;i<G.vernum;++i)//3️⃣找到数组中第一个为false的顶点从他出发调用dfs
		if(!visited[i])
			BFS(G,i);
}
void BFS(Graph G,int v){//‼️从顶点v出发广度优先遍历图G
	visit(v);
	visited[v]=TRUE;//1️⃣首先访问v结点并设置2号结点对应的visit数组为true
	EnQueue(Q,v);//2️⃣让v结点入队Q
	while(!isEmpty(Q)){
		DeQueue(Q,v);//3️⃣while循环，如果队列不空让队头元素出队
		for(w=FirstNeighbor(G,v);w>=0;w=NextNeighbor(G,v,w)){//4️⃣通过v结点找到与之相邻的所有的顶点（利用两个基本操作‼️）
			if(!visited[w]){
				visit(w);
				visited[w]=TRUE;
				EnQueue(Q,w);//5️⃣让没有被访问过的与v相邻的结点入队，且设置结点为访问过的再入队
			}//if
		}
	}//while
}
```

【结论‼️】对于~={red}无向图=~来说**调用 dfs 的次数=无向图中的连通分量的个数**（极大连通子图数量）
##### 分析效率
1️⃣**空间复杂度**
主要来自于定义的辅助队列，最坏的情况是   $O(|V|)$
![[截屏2026-05-14 22.18.19.png|300]]
2️⃣**时间复杂度**
对于 `BFS` 来说需要 visited 每一个顶点，同时在 for 循环里还需要探索所有的边
所以这个算法的时间开销主要来自于访问各个顶点以及探索各条边
For 邻接矩阵要找到和顶点相连的所有的边只能把顶点所对应的一整行都一次遍历一遍，一整行总共有  $O(|V|)$ 这么多元素则找每个顶点的邻接点都需要  $O(|V|)$ 时间，总共又有  $|V|$ 个顶点所以时间复杂度  $=O(|V|^2)$ ,再加上访问个个顶点的

####  基本操作对比
邻接矩阵🆚邻接表   
设 n 为顶点数，d (x) 为顶点 x 的邻接点个数。

| 操作                       | 邻接矩阵实现                                                            | 时间复杂度 | 邻接表实现                                                                                                       | 时间复杂度     |
| ------------------------ | ----------------------------------------------------------------- | ----- | ----------------------------------------------------------------------------------------------------------- | --------- |
| `FirstNeighbor (G, x)`   | ```for i=0..n-1<br>if G[x][i] == 1 return i<br>return -1```       | O (n) | ```if G[x]. First != NULL<br>return G[x]. First. Adjvex<br>return -1```                                     | O (1)     |
| `NextNeighbor (G, x, y)` | ```for i=y+1..n-1<br>if G[x][i] == 1 return i<br>return -1<br>``` | O (n) | ```p = G[x].first<br>while p != NULL:<br> if p.adjvex == y:<br> return p.next.adjvex 或 -1<br> p = p.next``` | O (d (x)) |
| 判断 x 和 y 是否邻接            | return G[x][y] == 1                                               | O (1) | 遍历 x 的边表，查找 y                                                                                               | O (d (x)) |
| 遍历 x 的所有邻接点              | 扫描矩阵第 x 行                                                         | O (n) | 遍历 x 的边表                                                                                                    | O (d (x)) |

| --------------------- | ----- | --------------------- |
