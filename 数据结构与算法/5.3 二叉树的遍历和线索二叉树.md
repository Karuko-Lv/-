### 手算二叉树三种遍历（分支结点逐层展开法）
重点练习容易错
![[截屏2026-05-05 18.02.16.png]] 🧚‍♀️后序遍历就是把树往右压平；这个讲解顺序就是广度优先遍历；中序遍历就相当于把树压平
如果一个结点是分支而不是叶子就需要嵌套递归的按照特定的遍历序列展开到下一级
####  中序遍历的“垂直投影法”（最快）

在阳光正好的正午，太阳从树的正上方往下照。 **把这棵树所有的结点直直地垂直掉落到地上**，掉落后它们在地面上从左到右排成的序列，就是**中序遍历序列**。

- _应用：_ 如果这是一棵二叉搜索树（BST），投影下来的序列绝对是一个**从小到大的递增序列**！

#### 后序遍历的“剪叶子法”

像秋风扫落叶一样，**从左到右**寻找树上的叶子结点，把它“剪下来”写进序列。剪完之后，原本叶子的父结点就变成了新的叶子，继续从左到右剪，直到最后剪掉根结点。这就是后序遍历序列。

### 代码写三种遍历
#### 先序遍历
递归函数
![[截屏2026-05-05 21.17.10.png]]
```cpp
typedef struct BiTNode{
	ElemType data;
	struct BiTNode *lchild,*rchild;
}BiTNode,*BiTree;
void visit(BiTNode *p)
{
	if(p == nullptr)
		return;
	cout<<p->value<<" ";
}
void PreOrder(BiTree T){
	if(T!=nullptr){
		visit(T);
		PreOrder(T->lchild);
		PreOrder(T->rchild);
	}
}
```
🤨两个递归的 PreOrder 写在 if 条件语句里面是同时确保了如果 T 为 nullptr 就不能访问其左右孩子了
#### 中序遍历

typedef struct BiTNode{

}BiTNode,*BiTree;
void visit(BiTNode *p)
{
	if(p == nullptr)
		return;
	cout<<p->value<<" ";

void InOrder(BiTree T){
	if(T!=nullptr){
		InOrder(T->lchild);
		visit(T);

### 三种遍历的扩展应用
‼️考察树这一章的算法题大多数是从这三种算法进行一种变换得到的
例如
1. 求一个树的深度
A：由于二叉树有递归特性根结点、左子树和右子树，求一棵树的深度或者高度应该先递归的求出左子树的高度，之后右子树的高度，再取左右子树更高的一边即可
```cpp
int treeDepth(BiTree T){
	//int h = 0;
	if(T!=null){
		//h+=1;
		int l = treeDepth(T->lchild);
		int r = treeDepth(T->rchild);
		return l>r?l+1;r+1;
	}else{
		return 0;
	}
}
```
🧚‍♀️如 C 结点返回 0+1；d 结点返回 0+1；-结点得到两个一的数据, 返回上一层为 1+1=2；这样的话是不是高度是从底向上加的呢？
![[截屏2026-05-05 22.07.58.png]]
`int l = treeDepth(T->lchild);` 是访问左，`int r = treeDepth(T->rchild);` 是访问右；`return l>r?l+1;r+1;` 相当于访问根==>左右根是后序遍历算法的变种
🧚‍♀️else 里面最后一行改成 ++l和 ++r就好理解了。;同理由叶节点 d 能够得到上一个根节点-的 R=1

### 二叉树的层序遍历
![[截屏2026-05-07 17.48.59.png]]
#### 逻辑
一层一层遍历二叉树的各个结点
![[截屏2026-05-05 22.19.28.png]]
1️⃣需要设置辅助队列
2️⃣让根结点入队
3️⃣判断队列是否为空 
4️⃣如果队列不是空的则让**队头结点**出队再访问出队的结点，将其左右孩子插入队尾（如果有）
🧚‍♀️广度优先搜索；这就是 bfs 算法；和中序不一样

#### 代码
用代码实现算法思想如下
```cpp
//层序遍历
void LevelOrder(BiTree T){
	//1️⃣需要设置辅助队列
	LinkQueue Q;
	//「PS」使用链队列而不是顺序队列 因为难以估计要访问的这棵树到底多少结点==>方便队列结点的拓展‼️
	InitQueue(Q);
	BiTree p;//记录出来的结点
	EnQueue(Q,T);//2️⃣让根结点入队
	while(!isEmpty(Q)){//3️⃣判断队列是否为空 
		//4️⃣如果队列不是空的则让**队头结点**出队再访问出队的结点
		DeQueue(Q,p)；
		visit(p);
		if(p->lchild!=nullptr)
			EnQueue(Q,p->lchild);//4️⃣左孩子入队
		if(p->rchild!=nullptr)
			EnQueue(Q,p->rchild);//4️⃣右孩子入队	
	}
}
```
🧚‍♀️这里的 P 是用来存储已经从队头弹出的树节点；bitree p 说明 p 是指针类型；有个 if 函数，是队空才会 free 掉；`BiTree p` 等价于 `BiTNode *p`，链式队列的 `data` 也是这个类型，这不就一致了嘛也可以理解为 p 一直指的都是队头元素，队头出去指向下一个对头，如果从图里看的话你也可以理解不停的动; p 是被返回的出队结点; 弹出队列的头节点会赋值给 p 再根据 p 找儿子
‼️辅助链队列数据结构定义
```cpp
typedef struct BiTNode{
	char data;
	struct BiTNode *lchild,*rchild;
}BiTNode,*BiTree;
//链队列数据结构定义如下
typedef struct LinkNode{
	BiTNode *data;
	struct LinkNode *next;
}LinkNode;

typedef struct{
	LinkNode *front,*rear;
}LinkQueue;
```
不需要在队列中保存一整个结点真实数据，只需要保存结点的指针即可‼️
保存结点指针比保存结点本身需要的空间少很多，所以让结点入队的时候 `EnQueue(Q,T)` 真正保存的是对这个结点的指针

#### 二叉树遍历🆚线性表遍历
![[截屏2026-05-07 20.57.04.png]]
普通的二叉树有四种遍历方式，经过任意一种遍历方式之后本来呈现非线性数据关系的数据得到了一个线性（每一个元素都有一个与之对应的前驱和与之对应的后继，除了第一个和最后一个）的遍历序列，但是树本身的逻辑一个结点有一个前驱和多个后继
对二叉树进行中序遍历都得从根结点出发，只有知道根结点是哪一个才可能对整颗二叉树进行中序遍历
**Q 1**：但是如果在知道除了根结点之外的某一个结点的指针的场景下（不知道根结点是哪一个），能不能从这个结点出发对这棵树进行中序遍历
即能否从一个置顶结点开始中序遍历
![[截屏2026-05-07 20.59.41.png|300]]
例如从 G 结点出发能否遍历 GBEAFC 这些结点呢
A：显然这样的操作是完成不了的，因为 G 结点只有指向其孩子的指针，不可能从 G 结点往回找到他的双亲 D或者 G 的后继结点 B 

如果 DGBEAFC 是一个线性表，给其中任意一个元素的指针肯定能找到这个元素后面的所有元素
==>「**总结‼️**」对线性表的遍历并不是每一次都必须从头开始遍历的，可以从任何一个元素开始往后遍历，但是二叉树必须每次从头开始遍历
🧚‍♀️就是说树无法像顺序表那样从头开始遍历；那个三叉链表指针指向的是父节点，不是基于遍历序列的前驱结点
**Q 2**: 给定树中一个指定的结点并且给其指针，有无可能找到这个结点的前驱（中序遍历序列当中的前驱）？
![[截屏2026-05-07 21.07.30.png]]
A：F 这个结点是第六个被访问到的，他的前驱是 A，知道 F 肯定没法找到前驱 A，因为每个结点只拥有向下一层的指针，所以不能往上找

 Q 3: 怎么找到任意一个结点的前驱??？
 A: 对二叉树从根结点出发进行一次中序遍历，在进行中序遍历的过程当中用一个指针 `q` 来记录当前被访问的结点，并且用一个指针 `pre` 来记录当前结点的前驱
![[截屏2026-05-07 21.17.40.png]]
刚开始 q 指向 D，pre 指向 NULL（因为 D 这个结点没有前驱 ）; 在下一个结点被 visit 之前需要将 pre 指针指向当前q 所指向的结点，再将 q 指向下一个被 visit 的结点（🧚‍♀️pre = q; q = q->next;），此时 pre 指针所指向的结点是 q 指针所指向的结点的中序遍历序列的前驱
当 q 和 p 指向同一个结点 pre 就是 p 指向的结点的前驱；当 pre 和 p 指向同一结点 q 指向的就是 p 指向结点的后继
==>一颗普通二叉树如果要找到一个指定结点的前驱只能从头进行一次完整的中序遍历
「**总结‼️**」普通二叉树如果要从某一个结点开始往后遍历是不方便的，且如果制定一个结点要找到其前驱和后继必须从头开始进行一次完整的中序遍历才可以找到
==>如果某些应用场景中找前驱和后继以及遍历操作十分频繁，这种普通二叉树的设计有一些缺点

对二叉树的某一个结点进行线索化的时候-->将这个结点的左孩子指针连上其遍历序列的前驱或右孩子的指针连上后继
#### 中序线索化
Q：当给定一个结点的指针 p 的时候怎么找到其在中序遍历序列当中的前驱
A：思考用代码实现它
##### 土方法
对整颗二叉树再重新进行一次中序遍历
当访问到一个结点 q的时候用另外一个指针 pre 指向当前访问结点的前驱结点，所以要找中序的前驱后继实现代码核心逻辑**和中序遍历一模一样**
```cpp
//中序遍历的代码
void InOrder(BiTree T){
	if(T!=nullptr){
		InOrder(T->lchild);
		visit(T);
		InOrder(T->rchild);
	}
}
```
‼️在 visit 函数当中实现对 q 和 pre 的处理 e.g 可以在 visit 函数中判断当前访问的结点是否与最终要的结点是一样的, 如果一样则当前 pre 指针指向的结点即要找的 p 的中序前驱结点!！
```cpp
//定义辅助❗️全局变量❗️，用于查找结点p的前驱
BiTNode *p;//指向目标结点
BiTNode * pre=nullptr;//指向当前访问结点的前驱
BiTNode * final=nullptr;//记录最终结果
//‼️访问结点q的操作
void visit(BiTNode *q){
	if(q==p)
		final = pre;
	else
		pre = q;
}
```
🧚‍♀️相当于每次遍历节点如果没结束就把 pre 指向当前遍历的节点，~={orange}如果一开始就结束那就说明没有前驱节点=~

###### 线索二叉树的结点定义
🆚普通二叉树结点定义区别增加了两个标识位
```cpp
typedef struct ThreadNode{
	ElemType data;
	struct ThreadNode *lchild,*rchild;
	int ltag,rtag;
}ThreadNode,*ThreadTree;
//‼️线索二叉树的结点定义与普通二叉树有区别
```
初步建成的线索二叉树 ltag 和 rtag 都会设置为 0，即指针都是指向左右孩子的即并没有被线索化
![[截屏2026-05-07 22.52.20.png|300]]
用**全局变量**pre 指向当前访问结点的前驱，中序遍历规则第一个被访问的是 D 这个结点，由于 D 没有前驱让其 pre 指向 nullptr，由图可知当前访问结点没有左孩子，所以左孩子指针线索化成 null 即可
```cpp
void visit(ThreadNode *q){
	if(q->lchild=nullptr){//1️⃣建立前驱线索（左子树为空）
		q->lchild=pre;
		q->ltag=1;//将标识位进行修改 只有tag值为1的时候与之对应的孩子指针才是线索
	} 
	if(pre!=nullptr&&pre->rchild==null){//1️⃣建立前驱结点的后继线索（右孩子为空，右孩子指针线索化）
	//🧚‍♀️这里用rtag == 0 也可以当判断条件应该
		pre->rchild=q;
		pre->rtag=1;		
	}
	//🧚‍♀️一句话，pre是q的前驱，q是pre的后继；第一种判断q的左孩子是为了建立前驱线索，第二种判断pre的右孩子是为了建立后驱线索，一共就这两种情况
	pre = q;//2️⃣让pre指针指向当前结点，以方便后续访问下一个结点
}
```

🧚‍♀️其实我觉得都用全局变量了，用 while 去实现遍历会更好，就像链表一样，相反，不用全局变量，用 return 的话，递归会更好；我觉得 while 是可以实现的，但是对中序遍历就没有体现了，但是这个题求的是前驱，是事先定义好的，中序遍历有没有也无关紧要；这里是 p 指针是进行了递归操作，而 pre 是没有的进行递归操作的，pre 指针只能通过每一层递归的 p 指针去修改其指向的结点，而 p 指针在当前的结点的右子树结束线索化后会跳到父亲结点那一层的递归，pre 不变
