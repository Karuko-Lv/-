「PS」C 语言中的字符串必须以 `\0`（空字符，Null character）结尾。C 语言中，其实并没有专门的“字符串”数据类型（不像 Java 的 `String` 或 Python 的 `str`）。C 语言是通过**字符数组**来实现字符串的，👇方式一和二等价
```cpp
// 方式一：使用字符串字面量（最常见，编译器会自动在末尾补 '\0'）
char str1[] = "Hello"; 

// 方式二：手动逐个赋值字符（必须手动加上 '\0' 才能算作标准字符串）
char str2[6] = {'H', 'e', 'l', 'l', 'o', '\0'};
```

**如果没有 `\0` 会发生什么？**
如果定义了一个字符数组，但没有以 `\0` 结尾，比如：
```c
char bad_str[5] = {'H', 'e', 'l', 'l', 'o'};
```
这时如果尝试用 `printf("%s", bad_str);` 打印它，程序打印完 "Hello" 后**不会停下来**。它会继续读取紧挨着这个数组后面的内存，把里面的垃圾数据当作字符打印出来（你会看到一堆乱码），直到它碰巧在内存的某个地方遇到了一个值为 `0` 的字节（即 `\0`）才会停止。
这种情况非常危险，被称为**内存越界读取**，严重时会导致程序崩溃。

### 静态数组/定长顺序存储
用静态数组存储字符串，length 表示当前字符串的长度
```cpp
#define MAXLEN 255
typedef struct{
	char ch[MAXLEN];
	int length;
}SString;
```
采用这种方式声明静态字符串系统会在内存开辟一片连续的空间以存放各个字符和 length 变量
每个字符占一个字节
定长的顺序存储和静态数组是一样的
缺点：长度不可变，255 长度存不下扩展是不可能的
🧚‍♀️弹幕这个人分不清字符串和字符数组，字符数组不需要末尾 0

### 动态数组（可变长）/堆分配存储
在结构体中定义一个基地址指针，用 malloc 申请一整片的连续存储空间，让基地址指针指向这一片空间的起始地址
```cpp
#define MAXLEN 255
typedef struct{
	char *ch;
	int length;
}HString;

HString S;
S.ch = new char[MAXLEN];
S.length = 0;
```
`malloc` 函数申请的这一片内存空间是属于内存里面的堆区，这种实现方式也可以称之为“堆分配存储”，堆区的内存空间**需要手动 free**，调用 `free()` 函数将这一片空间释放回收
🆚静态数组实现系统分配的存储空间是**系统自动回收**的, 函数执行结束之后系统就会自动回收这一片空间

#### C++和 C new和malloc
**💡 关键区别总结：
1. **申请方式：** `new` $\rightarrow$ `malloc`
2. **头文件：** C 语言必须引入 `<stdlib.h>`
3. **释放方式：** `delete[]` $\rightarrow$ `free()`
4. **返回值：** `malloc` 返回的是 `void *`（无类型指针），所以在 C 语言中通常会强转一下 `(char *)` 匹配变量类型。
