## 定义（和单链表对比）
单链表各个结点离散分布在内存的各个角落，每个结点会存放一个数据元素以及指向下一个结点的指针（下一个结点在内存当中的存放地址）
![[截屏2026-04-10 20.45.52.png]]
静态链表分配一整片内存空间，各个数据元素存放在这一整片空间中的某些位置，每个结点包含数据元素还有下一个结点的数组下标
静态链表中数组下标为 0 的结点充当头结点的角色，头结点不存放实际数据元素
静态链表的数组下一个结点的下标也称之为**游标**，和单链表中的指针差不多，只不过指针指明了具体的内存地址，游标只是指明了下一个元素的数组下标
单链表的表尾元素的 next 指针指向 null，静态链表用游标 -1 值来表示指针是最后一个结点，表示在这个结点后没有其他结点
🧚‍♀️带跳转的数组？；类似于邻接表；可以联想 FAT 表；fat 就是静态链表；操作系统文件章节的 FAT；结构体形数组；有点像操作系统的页表；那指针有啥用？；《微机原理与接口技术》；os 里的索引表；算法比赛中会用到；有点像查并集；对 cpu 缓存友好，都在一个缓存行长，不需要再查找内存；用在算法竞赛中建图；这不是就是结构体数组吗；acm 里面常用

基本原理：由于静态链表当中存放各个结点的空间是连续的==>静态链表当中的游标（数组下标）可以映射成所对应结点的实际存放地址

### 代码
#### 方式一
```cpp
#define MaxSize 10
struct Node{//静态链表结构类型的定义
	ElemType data;//存放数据元素
	int next;//下一个结点的游标
}
```
用数组的方式声明
```cpp
void testSLinklist(){
	struct Node a[MaxSize];
	//后续代码...
}
```
🧚‍♀️这 struct Node 到底是链表还是结点？上面的说是链表的结构结点，现在又变成了链表 a，明明都是 Node；节点数组，就是静态链表；单链表里的指针变成了数组下标？；数组的插入删除操作要移动元素，静态链表不用移动，所以**主要目的**就在这里
#### 方式二（课本）
```cpp
typedef struct Node{//静态链表结构类型的定义
	ElemType data;//存放数据元素
	int next;//下一个结点的游标
}SLinklist[MaxSize];
```
等价于先定义一个结构体 struct Node，再用 typedef 重命名 struct Node，`SLinklist[MaxSize]` 看起来像数组一样的东西，这么些如果后续写 `SLinklist a` 相当于声明一个数组，元素个数是MaxSize，且每个数组元素都是 `struct Node`

## 基本操作
### 初始化
```cpp
bool InitSLinklist(SLinklist *a){
	if(a==nullptr) return false;
	a[0].next = -1;
	return true;
}
```
### 查找
静态链表当中要找到某一个位序的结点**时间复杂度**是 ` O (n)`
某一个数组下标的结点**时间复杂度**是 ` O (1)`
位序指的是各个结点再**逻辑**上的顺序，结点中存储的数组下标只是反应各个结点再**物理**上的顺序
🧚‍♀️你在后续二叉树的数组存储里会用到；有点寄存器寻址的感觉，不知道是不是；所以这种存储结构的优点是什么，感觉结合了顺序表和链表的所有缺点啊；FAT 表，但是需要有文件目录；寄存器间接寻址
