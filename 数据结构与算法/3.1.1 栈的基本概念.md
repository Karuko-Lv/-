## 栈的定义
栈和线性表之间有很紧密的联系-->栈是一种特殊的线性表，逻辑结构和普通的线性表相同，
但在数据的运算上插入和删除操作是有**区别**的
区别：普通的线性表可以在任意一个地方插入删除元素，但是栈这种数据结构限制插入删除操作，只能在线性表的一端进行（表尾）

## 栈的基本操作
![[截屏2026-04-10 22.29.34.png]]
### 创、销
```cpp
InitStack(&S):初始化栈
DestroyStack(&L):销毁栈
```
### 增、删
```cpp
Push(&S,x):进栈
Pop(&S,&x):出栈
```

🧚‍♀️带&符号表明要修改数据，不带&符号表示只读不写；增删都会让 S 变化，所以要带&，而读取不改变 S，所以不带；方法调用就有函数栈这个概念，完全符合函数调用的规则

## 判断合法出栈顺序
### 人眼秒杀法（规律判断法 —— 适合做选择题）

记住一句口诀：**在任何一个已经出栈的数字后面，所有“比它小（即比它先入栈）”的数字，必须是降序排列的。**

因为栈的特性是“后进先出”（LIFO）。如果一个大数字先出栈了，说明压在它下面的所有小数字还在栈里，它们被死死压住了，所以将来出栈时，只能是紧挨着它的先出，最底下的最后出（即降序）。

- **✅ 合法例子：`3, 2, 5, 4, 1`**
    
    - 看数字 `3`：在它后面，比它小的是 `2` 和 `1`。顺序是 `2 -> 1`，**降序，合法**。
        
    - 看数字 `5`：在它后面，比它小的是 `4` 和 `1`（注意 `2` 已经出去了不用管）。顺序是 `4 -> 1`，**降序，合法**。
        
- **❌ 非法例子：`3, 1, 2, 4, 5`**
    
    - 看数字 `3`：在它后面，比它小的是 `1` 和 `2`。顺序是 `1 -> 2`，**升序，非法！**（一旦非法，直接排除）。
        
    - _原理：`3` 出栈了，说明栈底肯定是 `1`，上面压着 `2`。接下来如果要出栈，必须先出 `2` 才能出 `1`，绝不可能是 `1` 先出来。_
### 模拟还原法（逻辑推演法 —— 适合写代码或死胡同时）

有时候数字不是 12345，而是字母或者乱序的，用规律法容易看眼花，这时候就在纸上画一个杯子（栈）进行模拟。

**模拟的核心逻辑只有两条：**

1. 如果【栈顶元素】**等于**【当前目标出栈数字】，则**必须出栈**。
    
2. 如果【栈顶元素】**不等于**【当前目标出栈数字】（或者栈是空的），则继续把未入栈的元素**依次入栈**，直到栈顶碰到目标数字为止。
    

如果未入栈的元素都全部入栈了，栈顶依然凑不出目标数字，那就是**非法序列**。
