## 栈的定义
栈和线性表之间有很紧密的联系-->栈是一种特殊的线性表，逻辑结构和普通的线性表相同，
但在数据的运算上插入和删除操作是有**区别**的
区别：普通的线性表可以在任意一个地方插入删除元素，但是栈这种数据结构限制插入删除操作，只能在线性表的一端进行（表尾）

### 创、销
```cpp
InitStack(&S):初始化栈
DestroyStack(&L):销毁栈
```
### 增、删
```cpp
Push(&S,x):进栈
Pop(&S,&x):出栈
```

🧚‍♀️带&符号表明要修改数据，不带&符号表示只读不写；增删都会让 S 变化，所以要带&，而读取不改变 S，所以不带；方法调用就有函数栈这个概念，完全符合函数调用的规则

## 判断合法出栈顺序
### 人眼秒杀法（规律判断法 —— 适合做选择题）

记住一句口诀：**在任何一个已经出栈的数字后面，所有“比它小（即比它先入栈）”的数字，必须是降序排列的。**

因为栈的特性是“后进先出”（LIFO）。如果一个大数字先出栈了，说明压在它下面的所有小数字还在栈里，它们被死死压住了，所以将来出栈时，只能是紧挨着它的先出，最底下的最后出（即降序）。

- **✅ 合法例子：`3, 2, 5, 4, 1`**
    
    - 看数字 `3`：在它后面，比它小的是 `2` 和 `1`。顺序是 `2 -> 1`，**降序，合法**。
        
    - 看数字 `5`：在它后面，比它小的是 `4` 和 `1`（注意 `2` 已经出去了不用管）。顺序是 `4 -> 1`，**降序，合法**。
        
- **❌ 非法例子：`3, 1, 2, 4, 5`**
    
    - 看数字 `3`：在它后面，比它小的是 `1` 和 `2`。顺序是 `1 -> 2`，**升序，非法！**（一旦非法，直接排除）。
        
    - _原理：`3` 出栈了，说明栈底肯定是 `1`，上面压着 `2`。接下来如果要出栈，必须先出 `2` 才能出 `1`，绝不可能是 `1` 先出来。_
### 模拟还原法（逻辑推演法 —— 适合写代码或死胡同时）

有时候数字不是 12345，而是字母或者乱序的，用规律法容易看眼花，这时候就在纸上画一个杯子（栈）进行模拟。

**模拟的核心逻辑只有两条：**

1. 如果【栈顶元素】**等于**【当前目标出栈数字】，则**必须出栈**。
    
2. 如果【栈顶元素】**不等于**【当前目标出栈数字】（或者栈是空的），则继续把未入栈的元素**依次入栈**，直到栈顶碰到目标数字为止。
    

如果未入栈的元素都全部入栈了，栈顶依然凑不出目标数字，那就是**非法序列**。

为了让你直观感受到计算机是如何利用模拟法“一步步验证”出栈序列的，我为你做了一个交互式模拟器。你可以自己输入一个序列，观察它入栈出栈的过程

## 栈和堆
### 数据结构与算法
它们是两种具体的逻辑组织形式
1. **栈 (Stack) 作为数据结构：**
    - **长什么样**：就像一摞盘子，或者一个死胡同。
    - **核心规则**：后进先出（LIFO）。
    - **用途**：用来解决需要“回溯”或“保存历史状态”的问题。比如你之前写的括号匹配、浏览器的后退按钮、以及深度优先搜索（DFS）。
        
2. **堆 (Heap) 作为数据结构：**
    - **长什么样**：它本质上是一棵**完全二叉树**（用数组来存储的树）。
    - **核心规则**：
        - **最大堆**：每个父节点的值都大于或等于它的孩子节点（根节点最大）。
        - **最小堆**：每个父节点的值都小于或等于它的孩子节点（根节点最小）。
    - **用途**：用来实现“优先队列”（Priority Queue），或者快速找出一堆数据里的最大/最小值（比如排行榜、堆排序）。
### 操作系统与内存管理
真正在跑一段 C++ 程序时，操作**系统会给程序分配一块内存**。这块内存被强行划分成了两个不同的区域，名字恰好也叫“栈”和“堆”
1. **栈区 (Stack Memory)：**
        - **干什么用**：用来存放在函数里定义的**局部变量**（比如 `int i = 0`，或者 `SqStack S;`）以及函数调用的信息。
    - **特点**：由编译器**自动**分配和释放，速度极快。
    - **为什么叫栈**：因为这块内存的管理方式，确实严格遵循了数据结构中“后进先出”的原则。函数调用完，里面的局部变量就自动出栈销毁了。
2. **堆区 (Heap Memory)：**
    - **干什么用**：用来存放**动态申请**的内存。在 C++ 中，只要用了 **`new`** 关键字（比如 `new LinkNode()`），或者在 C 中用了 `malloc`，这块内存就是在堆区分配的。
    - **特点**：空间极大，由程序员**手动**申请，并且必须手动释放（用 `delete`），否则就会造成内存泄漏。
    - **⚠️ 终极避坑警告**：**内存中的“堆区”和数据结构里的“树状堆”没有任何必然联系！** 内存堆之所以叫堆，更像是在形容“一大堆杂乱无章的空闲内存”，想在哪切一块用就在哪切，它的底层往往是用链表（Free List）来管理的，而不是二叉树。
- 栈（LIFO）只关心“时间”（谁最后进来的，谁就先出去），而堆区的空闲链表关心的是“空间大小”（谁的尺寸最合适）。C++ 程序调用 `new` 或者 `malloc(size)` 申请内存时，操作系统（确切地说是 C 标准库的内存分配器，如 ptmalloc）会拿着要求的 `size`，去遍历这条空闲链表，寻找合适的空地
