## 初始化队列
确定存储结构后讨论给予某种物理存储结构如何实现某种数据结构（队列）的相关重要基本操作
![[截屏2026-04-14 21.16.04.png]]
C 语言中用静态数组存储队列中的数据元素
队列操作受限——只从队头删除元素、队尾插入元素

```cpp
typedef struct {
	int data[MaxSize];//用静态数组存放队列元素
	int front,rear;//队头指针和队尾指针
}SqQueue;
void testQueue(){
	SqQueue Q;//声明一个队列（顺序存储）
	//...
}
```

采用变量声明的方式声明一个队列，执行 `SqQueue Q;//声明一个队列（顺序存储）` 这句代码会导致系统在背后分配一整片连续空间，可以存放十个数据元素，让队头指针指向队头，队尾指向最后位置的后一个位置，也就是接下来可以插入数据元素的位置
![[截屏2026-04-14 21.09.44.png]]

```cpp
//初始化队列
void InitQueue(SqQueue &Q){
	Q.rear = Q.front = 0;
}
void testQueue(){
	//声明一个队列（顺序存储）
	SqQueue Q;
	InitQueue(Q);
}
```
### 判断队列是否为空
```cpp
bool QueueEmpty(SqQueue Q){
	if(Q.rear == Q.front)
		return true;
	else
		return false;
}
```

## 入队操作

只能从队尾方向让新元素入队，由于队列是用静态数组实现的，其容量有限，当插入之前需要**先判断队列是否已经存满**

```cpp
//入队
bool EnQueue(SqQueue &Q,int x){
	if(队列已满)
		return false;
	Q.data[Q.rear] = x;
	Q.rear++;
	return true;
}
```

### 队满判断
如果让队头元素出队了，即使队尾指针已经是 MaxSize 了，元素还是可以插入前面是空的静态数组里面
需要将 rear 指针重新指向队头的空了的位置-->取余![[截屏2026-04-14 21.24.13.png]]
	在这里判断队满 `(Q.rear+1)%MaxSize == Q.front`
	PS：由于判断表空使用的队尾和队头指针指向同一个位置来判断所以在这里需要牺牲一个空间，所以是判断队尾指针的后一个位置是否等于队头指针指向的位置![[截屏2026-04-14 21.29.25.png]]
### 循环队列
本质是将线状的存储空间在逻辑上变成了环状的存储空间
![[截屏2026-04-14 21.24.23.png]]
`Q.rear = (Q.rear+1)%MaxSize` 会导致队尾指针指向下一个应该插入数据元素的位置
🧚‍♀️其实就是补码的原理

## 出队操作
只能让队头的元素出队

```cpp
bool DeQueue(SqQueue &Q,int &x){
	if(Q.rear == Q.front)
		return false;//队空报错
	x=Q.data[Q.front];
	Q.front = (Q.front+1)%MaxSize;
	return true;
}
```

`Q.front = (Q.front+1)%MaxSize;` 队头指针往后移动一位的同时对 MaxSize 取模——目的是让 **front 指针**转着圈圈移动
🧚‍♀️这实际上根本没有删除这个元素吧；下标变了，并没有真正的删除数据
