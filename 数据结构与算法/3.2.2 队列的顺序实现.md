## 初始化队列：队尾指针指向队尾元素的下一个位置
确定存储结构后讨论给予某种物理存储结构如何实现某种数据结构（队列）的相关重要基本操作
![[截屏2026-04-14 21.16.04.png]]
C 语言中用静态数组存储队列中的数据元素
队列操作受限——只从队头删除元素、队尾插入元素

```cpp
typedef struct {
	int data[MaxSize];//用静态数组存放队列元素
	int front,rear;//队头指针和队尾指针
}SqQueue;
void testQueue(){
	SqQueue Q;//声明一个队列（顺序存储）
	//...
}
```

采用变量声明的方式声明一个队列，执行 `SqQueue Q;//声明一个队列（顺序存储）` 这句代码会导致系统在背后分配一整片连续空间，可以存放十个数据元素，让队头指针指向队头，队尾指向最后位置的后一个位置，也就是接下来可以插入数据元素的位置
![[截屏2026-04-14 21.09.44.png]]

```cpp
//初始化队列
void InitQueue(SqQueue &Q){
	Q.rear = Q.front = 0;
}
void testQueue(){
	//声明一个队列（顺序存储）
	SqQueue Q;
	InitQueue(Q);
}
```
### 判断队列是否为空

```cpp
bool QueueEmpty(SqQueue Q){
	if(Q.rear == Q.front)
		return true;
	else
		return false;
}
```

## 入队操作

只能从队尾方向让新元素入队，由于队列是用静态数组实现的，其容量有限，当插入之前需要**先判断队列是否已经存满**

```cpp
//入队
bool EnQueue(SqQueue &Q,int x){
	if(队列已满)
		return false;
	Q.data[Q.rear] = x;
	Q.rear++;
	return true;
}
```
![[截屏2026-04-14 21.20.19.png]]
### 队满判断
如果让队头元素出队了，即使队尾指针已经是 MaxSize 了，元素还是可以插入前面是空的静态数组里面
需要将 rear 指针重新指向队头的空了的位置-->取余![[截屏2026-04-14 21.24.13.png]]
	在这里判断队满 `(Q.rear+1)%MaxSize == Q.front`
	PS：由于判断表空使用的队尾和队头指针指向同一个位置来判断所以在这里需要牺牲一个空间，所以是判断队尾指针的后一个位置是否等于队头指针指向的位置![[截屏2026-04-14 21.29.25.png]]
### 循环队列
本质是将线状的存储空间在逻辑上变成了环状的存储空间
![[截屏2026-04-14 21.24.23.png]]
`Q.rear = (Q.rear+1)%MaxSize` 会导致队尾指针指向下一个应该插入数据元素的位置
🧚‍♀️其实就是补码的原理

## 出队操作
只能让队头的元素出队

```cpp
bool DeQueue(SqQueue &Q,int &x){
	if(Q.rear == Q.front)
		return false;//队空报错
	x=Q.data[Q.front];
	Q.front = (Q.front+1)%MaxSize;
	return true;
}
```

`Q.front = (Q.front+1)%MaxSize;` 队头指针往后移动一位的同时对 MaxSize 取模——目的是让 **front 指针**转着圈圈移动
🧚‍♀️这实际上根本没有删除这个元素吧；下标变了，并没有真正的删除数据

### 获取队头元素的值
```cpp
bool GetHead(SqQueue Q,int &x){
	if(Q.rear == Q.front)
		return false;
	x = Q.data[Q.front];
	return true;
}
```

## 判断队列已满/空
### 方案一：牺牲一个存储单元
![[截屏2026-04-14 21.38.22.png]]
#### 队列元素个数
`（rear + MaxSize - front）%MaxSize`
数论模运算的性质可以证明该式子，**~={red}常考=~**‼️
🧚‍♀️这里的 rear+MaxSize 可以理解为指队尾在队头前面时（此时 0<=队尾<队头），从 0 开始到队尾指针的中所含元素个数，再减去队头前空了几个元素；1.r>f: 正常计算；2.  R<f: (M-f)+r；所以为什么不直接在结构体里定义一个 size  
如果不牺牲 2 这个地方的空间，队满和队空所表现出来的状态是一样的，则没办法用代码逻辑判断队列是否是满的还是空的
### 方案二：结构体中设置一个队列大小变量
在队列结构当中定义一个记录队列长度的变量 size——记录队列当中此时存放了几个数据元素 
🧚‍♀️自己加个 size 类成员变量就行了，直接判断 size 大小；浪费是指申请的空间基本没用上。这里是在定义队列时添加的 size，没有浪费；大草，那 int 一个 size 不还是浪费了内存吗
入队成功让 size 的值++，出队成功 size--，根据 size 的值判断队满或者队空；就是牺牲一个整型空间换来了队列中可以放满内容
