## 函数调用背后的过程
![[截屏2026-04-17 17.14.07.png]]
当 func 2 执行结束后会返回 func 1，接着执行后面的代码；再等 func 1 执行结束之后又会返回 main 函数执行后续代码
==>**特性**：最后被调用的函数最先执行（LIFO）
实际任何一段代码任何一个程序运行之前系统都会开辟一个函数调用栈，用栈保存**各个函数在调用过程中必须保存的一些信息**：![[截屏2026-04-17 17.18.10.png]]
1️⃣调用返回地址（调用 `func 1` 函数的时候把 ` func 1` 执行结束之后应该执行的代码存储地址压到栈中）
2️⃣实参（函数调用的参数会放到栈中）
3️⃣局部变量
	「PS」由上图的栈可以看出 `main` 函数中的 `a`、`b` 变量数据对应的是 main：中的 ` a `、` b ` 两份数据，func 1 中修改 ab 的值影响不到 main 函数的 ` a ` 、` b ` 值，除了被调用函数的实参之外函数当中定义的局部变量 ` x ` 也是放在栈当中的; 之后 `func 1` 会调用 `func 2`，同样记录 `func 2` 执行之后应该回到哪一句继续往后执行，即将 `x=x+10086` 的存放地址给放到 ` func 2 ` 的栈里面，和刚才一样存放实参和局部变量，`func2` 执行完了之后就可以将与它相关的信息弹出栈，释放了 ` func 2` 这一片内存空间
函数调用栈其实本质上是内存中的某一片区域，某一片存储空间
🧚‍♀️递归内存爆炸！；Jvm 方法存在虚拟机栈；递归导致栈溢出; new 对象是堆内存；string str 是栈内存
**函数调用背后需要一个栈来支持**

程序从 main 函数开始，但是程序在经过编译之后编译器会在 main 函数之前加一些其他的代码，将一些不知道信息压入栈底，不需要普通程序员关心
🧚‍♀️这里指的 main 函数之前所执行的语句是预处理语句中的相关代码；main 函数之前的是宏操作
