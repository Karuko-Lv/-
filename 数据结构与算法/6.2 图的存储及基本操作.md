## 邻接矩阵（顺序）
![[截屏2026-05-14 14.53.06.png]]
### 存储一般图
原理简单，0 表示两个顶点之间互不邻接，1 表示两个顶点之间互相邻接, 不带权的图只需要表示出各个顶点之间有没有连接的关系即可
无向图的每一条边在邻接矩阵中对应两个 1 
有向图的每一条弧（有向边）在邻接矩阵中对应一个 1
#### 代码实现
二维数组实现邻接矩阵 `int Edge[MaxVertexNum][MaxVertexNum];` 即边表就是邻接矩阵
```cpp
#define MaxVertexNum 100
typedef struct{
	char Vex[MaxVertexNum];//顶点表
	int Edge[MaxVertexNum][MaxVertexNum];//边表
	int vexnum,arcnum;//当前图的顶点数和边树
}MGraph;
```
👆上述代码表示顺序存储的图中边最多支持 100 个顶点；用结点在数组中的存放下标和邻接矩阵中的行列进行一个对应；顶点可以存更复杂的信息顶点表的类型改变即可，边表可以用 bool 类型节省空间

### 存储带权图（网）
对应位置写边的权值，两顶点之间不存在边，用∞表示不存在相连的边（代码中定义一个常量代表无穷，int 值的最大值）
🧚‍♀️自己和自己就是主对角线上的元素
#### 代码实现
```cpp
#define MaxVertexNum 100      //顶点数量最大值
#define INFINITY 最大的int值   //宏定义常量“无穷”
typedef char VerTexType;
typedef int EdgeType; //带权图中边上权值
typedef struct{
	VertexType Ver[MaxVertexNum];
	EdgeType Edge[MaxVertexNum][MaxVertexNum];
	int vernum,arcnum;//当前图的顶点数和边树
}MGraph;
```
带权图边上的权值由业务场景决定

#### 性质
邻接矩阵的性质，不带权的图即矩阵元素值为 0 或 1
记图 G 的邻接矩阵为 $A$（指的是整个邻接矩阵不是其中某个顶点）
$A^n$ 的元素 $A^n[i][j]$ 等于由顶点 $i$ 到顶点 $j$ 的长度为 $n$ 的路径的数目
$\mathrm{A}^{2}[1][4]=\mathrm{a}_{1,1}\mathrm{a}_{1,4}+\mathrm{a}_{1,2}\mathrm{a}_{2,4}+\mathrm{a}_{1,3}\mathrm{a}_{3,4}+\mathrm{a}_{1,4}\mathrm{a}_{4,4}=1$
$\mathrm{A}^{2}[2][2]=\mathrm{a}_{2,1}\mathrm{a}_{1,2}+\mathrm{a}_{2,2}\mathrm{a}_{2,2}+\mathrm{a}_{2,3}\mathrm{a}_{3,2}+\mathrm{a}_{2,4}\mathrm{a}_{4,2}=3$


从 1 结点到 4 结点即从 A 结点到 D 结点长度为 2 的路径只能找到一条 A->B->D

从 B 到 B 长度为 2 的路径一共可以找到三条

## 邻接表（顺序+链式）
![[Pasted image 20260514153900.png]]
⚠️解决了空间复杂度高的问题但是同时存在存放有向图有一些操作不方便的情况，所以**邻接表可以进一步优化**
**Adjacency List**

用一个一维数组存储各个顶点的信息，其中包括顶点的数据域和指向顶点的第一条边（或弧）的指针
声明图即声明顶点结点的数组➕记录当前图中具体有多少边和结点
### 代码实现
```cpp
typedef struct ArcNode{
	int adjvex;//边/弧指向哪一个结点
	struct ArcNode *next;//指向下一个弧的指针
	//InfoType info;//边权值
}ArcNode;
typedef struct VNode{
	VertexType data;
	ArcNode *first;
}VNode,AdjList[MaxVertexNum];
typedef struct{
	AdjList vertices;
	int vexnum,arcnum;
}ALGraph;
```
🧚‍♀️
![[截屏2026-05-14 15.20.38.png]]
都是各个结点顺序存储，再用链表指明和结点相连的各个边
⚠️对于图的邻接表，表示方式是不唯一的，例如 A 和 B、C、D 都相连，所以 A 结点的边链表可以用 123 321 132 等顺序，即各个边在列表当中出现的先后顺序是任意的
🆚邻接矩阵只要确定了顶点编号图的邻接矩阵是唯一的【选择爱考】
![[截屏2026-05-14 15.38.10.png]]
### 空间复杂度
![[截屏2026-05-14 15.29.44.png]]
#### 无向图
无向图中每一条边在邻接表都会对应两个节点，故无向图中边的数据有冗余，边结点的数量应该是边的实际数量的两倍
所以邻接表存储无向图空间复杂度数量级 $O(|V|+2|E|)$
#### 有向图
每一条弧只会对应一个边结点
所以邻接表存储有向图空间复杂度数量级 $O(|V|+|E|)$

#### 求度的时间复杂度
Q：求顶点的度/出度/入度？
A：🤨无向图度遍历邻接表中的数组中结点对应的 `ArcNode` 的长度
🧑‍🏫无向图遍历和顶点相关的边链表，有多少结点度就是多少，遍历边链表即可以找到和当前顶点相连的所有的边
🤨有向图的出度很简单但是入度需要遍历每个邻接表的所有顶点的边链表
🧑‍🏫找到有向图指向某个结点的弧是很麻烦的~={red}**很大的缺点**=~
🧚‍♀️孩子表示法，找孩子简单，找父节点难
