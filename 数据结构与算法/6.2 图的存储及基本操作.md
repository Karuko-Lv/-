## 邻接矩阵（顺序）
![[截屏2026-05-14 14.53.06.png]]
### 存储一般图
原理简单，0 表示两个顶点之间互不邻接，1 表示两个顶点之间互相邻接, 不带权的图只需要表示出各个顶点之间有没有连接的关系即可
无向图的每一条边在邻接矩阵中对应两个 1 
有向图的每一条弧（有向边）在邻接矩阵中对应一个 1
#### 代码实现
二维数组实现邻接矩阵 `int Edge[MaxVertexNum][MaxVertexNum];` 即边表就是邻接矩阵
```cpp
#define MaxVertexNum 100
typedef struct{
	char Vex[MaxVertexNum];//顶点表
	int Edge[MaxVertexNum][MaxVertexNum];//边表
	int vexnum,arcnum;//当前图的顶点数和边树
}MGraph;
```
👆上述代码表示顺序存储的图中边最多支持 100 个顶点；用结点在数组中的存放下标和邻接矩阵中的行列进行一个对应；顶点可以存更复杂的信息顶点表的类型改变即可，边表可以用 bool 类型节省空间

#### 性质
邻接矩阵的性质，不带权的图即矩阵元素值为 0 或 1
记图 G 的邻接矩阵为 $A$（指的是整个邻接矩阵不是其中某个顶点）
$A^n$ 的元素 $A^n[i][j]$ 等于由顶点 $i$ 到顶点 $j$ 的长度为 $n$ 的路径的数目
$\mathrm{A}^{2}[1][4]=\mathrm{a}_{1,1}\mathrm{a}_{1,4}+\mathrm{a}_{1,2}\mathrm{a}_{2,4}+\mathrm{a}_{1,3}\mathrm{a}_{3,4}+\mathrm{a}_{1,4}\mathrm{a}_{4,4}=1$
$\mathrm{A}^{2}[2][2]=\mathrm{a}_{2,1}\mathrm{a}_{1,2}+\mathrm{a}_{2,2}\mathrm{a}_{2,2}+\mathrm{a}_{2,3}\mathrm{a}_{3,2}+\mathrm{a}_{2,4}\mathrm{a}_{4,2}=3$


从 1 结点到 4 结点即从 A 结点到 D 结点长度为 2 的路径只能找到一条 A->B->D

## 邻接表（顺序+链式）
![[Pasted image 20260514153900.png]]
⚠️解决了空间复杂度高的问题但是同时存在存放有向图有一些操作不方便的情况，所以**邻接表可以进一步优化**
**Adjacency List**

用一个一维数组存储各个顶点的信息，其中包括顶点的数据域和指向顶点的第一条边（或弧）的指针
声明图即声明顶点结点的数组➕记录当前图中具体有多少边和结点
### 代码实现
```cpp
typedef struct ArcNode{

	//InfoType info;//边权值
}ArcNode;
typedef struct VNode{
	VertexType data;
	ArcNode *first;
}VNode,AdjList[MaxVertexNum];
typedef struct{
	AdjList vertices;
	int vexnum,arcnum;
}ALGraph;
```

### 空间复杂度
![[截屏2026-05-14 15.29.44.png]]
#### 无向图
无向图中每一条边在邻接表都会对应两个节点，故无向图中边的数据有冗余，边结点的数量应该是边的实际数量的两倍
所以邻接表存储无向图空间复杂度数量级 $O(|V|+2|E|)$
#### 有向图
每一条弧只会对应一个边结点
所以邻接表存储有向图空间复杂度数量级 $O(|V|+|E|)$

#### 求度的时间复杂度
Q：求顶点的度/出度/入度？
A：🤨无向图度遍历邻接表中的数组中结点对应的 `ArcNode` 的长度
🧑‍🏫无向图遍历和顶点相关的边链表，有多少结点度就是多少，遍历边链表即可以找到和当前顶点相连的所有的边
🤨有向图的出度很简单但是入度需要遍历每个邻接表的所有顶点的边链表
🧑‍🏫找到有向图指向某个结点的弧是很麻烦的~={red}**很大的缺点**=~

##    十字链表（有向图）
![[截屏2026-05-14 16.36.26.png]]
![[截屏2026-05-14 15.40.38.png]] 
![[截屏2026-05-14 15.58.48.png]]
🧚‍♀️设计成十字链表是为了同时方便找入度和出度，横着遍历一行（绿色指针）为出度，竖着遍历一列（橙色指针）为入度
🧑‍🏫通过绿色的指针线一直往后找各个结点可以找到**从当前顶点往外发射的所有的弧**；通过橙色的指针线一直往后找可以找到所有指向当前顶点的弧
### 代码实现
两种结构体：1️⃣顶点结构体 2️⃣弧结构体
```cpp
#define MAX_VERTEX_NUM 100 // 定义图的最大顶点数

typedef char VertexType;   // 顶点数据类型（如图中的 'A', 'B', 'C', 'D'）
typedef int InfoType;      // 权值/信息域数据类型

// 1. 弧结点结构定义 (Arc Node)
typedef struct ArcBox {
    int tailvex;              // 弧尾顶点编号
    int headvex;              // 弧头顶点编号
    InfoType info;            // 权值/信息域 (如果不需要权值可省略)
    struct ArcBox *hlink;     // 指向【弧头相同】的下一条弧的指针 (同入边)
    struct ArcBox *tlink;     // 指向【弧尾相同】的下一条弧的指针 (同出边)
} ArcBox;

// 2. 顶点结点结构定义 (Vertex Node)
typedef struct VexNode {
    VertexType data;          // 数据域，存储顶点信息
    ArcBox *firstin;          // 该顶点作为弧头的第一条弧 (入边链表头指针)
    ArcBox *firstout;         // 该顶点作为弧尾的第一条弧 (出边链表头指针)
} VexNode;

// 3. 图的总体结构定义
typedef struct {
    VexNode xlist[MAX_VERTEX_NUM]; // 用数组顺序存储顶点结点
    int vexnum;                    // 图的当前顶点数
    int arcnum;                    // 图的当前弧数
} OLGraph;
```

### 性能分析 
空间复杂度和邻接表一样，同时解决了连接表中找入度不方便的问题，只需要顺着橙色的指针线路往后找即可找到所有指向当前结点的入边
⚠️十字链表法只能存储有向图
![[截屏2026-05-14 16.21.01.png]]
🧚‍♀️十字链表相对于邻接表和逆邻接表结合；这点空间消耗不比你遍历消耗小多了；但是有时候空间换时间也是没有办法的办法；十字链表上机用处为零，单独维护一个相同图相反的边就行了
## 邻接多重表（无向图）
邻接矩阵存储无向图问题时空间复杂度太高，
邻接表存储无向图主要的问题时每一条辨对应两份冗余信息，且删除也得删除两份数据
🧚‍♀️压缩不就行了吗；教材上说邻接矩阵的优点就是便于增加和删除。。。；所以说什么，没有完全完美的，要想一样快，就只能牺牲另一样。所以说你考试的时候阅卷看他是希望时间开始空间复杂度低来选择用哪个存储方式
🧑‍🏫解决方案：用邻接多重表来存储无向图
与十字链表很相似
![[截屏2026-05-14 16.28.13.png]]

由于是无向图橙色和绿色逻辑上是等价的没有先后之分
「PS」无向图如果是带权图在边结点可以增加一个字段表示每一条边的权值
优点：找到和某一个顶点相连的边很方便；且每一条边只会对应一个边结点，只有一份数据==>不需要像邻接表一样同时维护两份冗余的数据（删除结点/边会方便很多）
例如 1️⃣删除 AB 的边，修改两个指针和一个 A-B 边结点即可

1️⃣顶点结构体 2️⃣边结点结构体
```cpp
#define MAX_VERTEX_NUM 100 // 定义图的最大顶点数

typedef char VertexType;   // 顶点数据类型（如图中的 'A', 'B', 'C', 'D', 'E'）
typedef int InfoType;      // 权值/信息域数据类型

// 1. 边结点结构定义 (Edge Node)
typedef struct EBox {
    int ivex;             // 边的两个顶点编号之一 (i)
    int jvex;             // 边的两个顶点编号之二 (j)
    InfoType info;        // 权值/信息域 (如果不需要权值可省略)
    struct EBox *ilink;   // 指向【依附于顶点 i 的下一条边】的指针
    struct EBox *jlink;   // 指向【依附于顶点 j 的下一条边】的指针
} EBox;

// 2. 顶点结点结构定义 (Vertex Node)
typedef struct VexNode {
    VertexType data;      // 数据域，存储顶点信息
    EBox *firstedge;      // 指向【与该顶点相连的第一条边】的指针
} VexNode;

// 3. 图的总体结构定义
typedef struct {
    VexNode adjmulist[MAX_VERTEX_NUM]; // 存储顶点结点的数组
    int vexnum;                        // 图的当前顶点数
    int edgenum;                       // 图的当前边数
} AMLGraph;
```

```cpp
#include <vector>
#include <stdbool.h>

// 假设 graph 已经是一个预定义好的二维数组或 vector
// 参数 u 为起点，v 为终点
bool Adjacent(const std::vector<std::vector<int>>& graph, int x, int y) {
    // 直接通过下标访问并判断
    if (graph[x][y] != 0) {
        return true;  // 存在边/弧
    }
    return false;     // 不存在
}

//如果是纯 C 语言的二维数组：
bool AdjacentC(int graph[][MAX_V], int x, int y) {
    return graph[x][y] != 0;
}
```

##### 邻接表
```cpp
#include <vector>

// adj 是邻接表，adj[u] 存储了所有从 u 出发可以到达的顶点
bool Adjacent(const std::vector<std::vector<int>>& adj, int x, int y) {
    // 遍历顶点 u 的所有相邻顶点
    for (int neighbor : adj[x]) {
        if (neighbor == y) {
            return true; // 找到了目标顶点，说明存在边
        }
    }
    return false; // 遍历完都没找到，说明不存在
}
```

```cpp
        }
    }

}
```

### 插入顶点 x
`InsertVerTex(G,x)`
#### 代码
##### 邻接矩阵
C 语言
```cpp
#include <stdbool.h>
#define MAX_V 100

// 邻接矩阵图结构
typedef struct {
    char data[MAX_V];           // 顶点数据数组
    int edge[MAX_V][MAX_V];     // 邻接矩阵
    int vexnum, arcnum;         // 当前顶点数和边数
} MGraph;

// 插入顶点 x
bool InsertVertex(MGraph *G, char x) {
    if (G->vexnum >= MAX_V) {
        return false; // 数组已满，无法插入
    }
    
    int new_index = G->vexnum;
    G->data[new_index] = x; // 1. 将新顶点数据 'X' 放入 data 数组末尾
    
    // 2. 将新增的第 new_index 行和第 new_index 列全部初始化为 0
    for (int i = 0; i <= new_index; i++) {
        G->edge[new_index][i] = 0; // 新增行的元素置 0
        G->edge[i][new_index] = 0; // 新增列的元素置 0
    }
    
    G->vexnum++; // 顶点数加 1
    return true;
}
```
C++（动态 `std::vector`）
如果使用动态数组，操作会稍微复杂一点，因为需要对现有的每一行进行扩容。
```cpp
#include <vector>

void InsertVertexMatrixCpp(std::vector<std::vector<int>>& graph, std::vector<char>& data, char x) {
    data.push_back(x); // 添加数据 'X'
    int new_v = data.size(); // 插入后的总顶点数
    
    // 1. 给原来的每一行末尾追加一个 0 (对应图片中新增的列)
    for (int i = 0; i < new_v - 1; ++i) {
        graph[i].push_back(0); 
    }
    
    // 2. 在矩阵末尾新增一行，长度为 new_v，全部初始化为 0 (对应图片中新增的行)
    graph.push_back(std::vector<int>(new_v, 0));
}
```
「PS」插入 0 是在邻接矩阵被初始化的时候就做好的，所以插入新结点的唯一开销是写入顶点相关的信息
🧚‍♀️这个静态数组不太好扩展吧；初始化的时候给定 maxsize 就行了；用 vector int 存，数组动态数组就是 vector
##### 邻接表

```cpp
#include <stdio.h>
#include <stdbool.h>
#define MAX_V 100

// 边结点
typedef struct ArcNode {
    int adjvex;
    struct ArcNode *nextarc;
} ArcNode;

// 顶点结点
typedef struct VNode {
    char data;
    ArcNode *first; // 指向第一条弧
} VNode;

// 邻接表图结构
typedef struct {
    VNode vertices[MAX_V]; // 顶点数组
    int vexnum, arcnum;
} ALGraph;

// 插入顶点 x
bool InsertVertexList(ALGraph *G, char x) {
    if (G->vexnum >= MAX_V) {
        return false; // 数组已满
    }
    
    int new_index = G->vexnum;
    G->vertices[new_index].data = x;     // 1. 存入新顶点数据 'X'
    G->vertices[new_index].first = NULL; // 2. 将边表头指针初始化为空 (非常重要)
    
    G->vexnum++; // 顶点数加 1
    return true;
}
```

```cpp
#include <stdbool.h>
#define MAX_V 100

typedef struct {
    char data[MAX_V];           // 顶点数据
    bool isValid[MAX_V];        // 【核心】标记该顶点是否存在 (对应图中手写的 bool)
    int edge[MAX_V][MAX_V];     // 邻接矩阵
    int vexnum, arcnum;
} MGraph;

// 删除下标为 x 的顶点 (无向图为例)
bool DeleteVertexMatrix(MGraph *G, int x) {
    // 检查越界或该点已经被删除
    if (x < 0 || x >= MAX_V || !G->isValid[x]) {
        return false; 
    }

    // 1. 将数据域标记为空/已删除
    G->isValid[x] = false; 
    
    // 2. 清除与该顶点相关的所有的边 (将第 x 行和第 x 列置为 0)
    for (int i = 0; i < MAX_V; i++) {
        if (G->isValid[i] && G->edge[x][i] != 0) {
            G->edge[x][i] = 0; // 清空出边
            G->edge[i][x] = 0; // 清空入边
            G->arcnum--;       // 图的总边数减 1
        }
    }
    
    G->vexnum--; // 有效顶点数减 1
    return true;
}
```
**时间复杂度：** $O(|V|)$。只需要遍历一行/一列即可。
#### 邻接表
除了删除 C 还有删除 C 相连的边结点之外还需要在其他和他相连的顶点边表中找到指向 C 的结点的边信息，并且给删除
最好情况：当前删除结点原本没有连接任何的边 $O(1)$
最坏情况：当前删除结点连了尽可能多的边，和其他所有的顶点都有相连的边，且都是接在最后的一个位置
🧚‍♀️反正计算机中算法都有利有弊，空间与时间不可兼得，最后都会折中；看书后面习题，应该是 O (V+E) 吧；这不是 V 加 E 嘛
```cpp
#include <stdio.h>
#include <stdlib.h>
#include <stdbool.h>
#define MAX_V 100

typedef struct ArcNode {
    int adjvex;
    struct ArcNode *nextarc;
} ArcNode;

typedef struct VNode {
    char data;
    bool isValid;    // 同样使用逻辑删除标记
    ArcNode *first;
} VNode;

typedef struct {
    VNode vertices[MAX_V];
    int vexnum, arcnum;
} ALGraph;

// 删除下标为 x 的顶点 (无向图为例)
bool DeleteVertexList(ALGraph *G, int x) {
    if (x < 0 || x >= MAX_V || !G->vertices[x].isValid) return false;

    // 1. 标记顶点被删除
    G->vertices[x].isValid = false;
    G->vexnum--;

    // 2. 释放顶点 x 自己链表上的所有边结点
    ArcNode *p = G->vertices[x].first;
    while (p != NULL) {
        ArcNode *temp = p;
        p = p->nextarc;
        free(temp);
        G->arcnum--; // 总边数减 1
    }
    G->vertices[x].first = NULL;

    // 3. 遍历其他所有有效顶点的链表，删除指向 x 的边结点
    for (int i = 0; i < MAX_V; i++) {
        if (i == x || !G->vertices[i].isValid) continue;

        ArcNode *curr = G->vertices[i].first;
        ArcNode *prev = NULL;

        // 标准的单链表删除节点操作
        while (curr != NULL) {
            if (curr->adjvex == x) {
                // 找到了指向 x 的边，执行删除
                if (prev == NULL) {
                    G->vertices[i].first = curr->nextarc; // 删除的是头结点
                } else {
                    prev->nextarc = curr->nextarc;        // 删除的是中间结点
                }
                ArcNode *temp = curr;
                curr = curr->nextarc;
                free(temp);
                // 注意：由于是无向图，删除 x 自己的链表时边数已经减过一次了，这里不需要再 G->arcnum--
            } else {
                // 没找到，双指针继续后移
                prev = curr;
                curr = curr->nextarc;
            }
        }
    }
    return true;
}
```
1️⃣**清空自己的链表：** 把顶点 `C` (`x=2`) 后面挂着的链表全部删除（释放内存）。
2️⃣**清理别人的链表：** 遍历其他所有顶点的链表，如果在里面发现了指向 `C` (`adjvex == 2`) 的结点，必须将其从单链表中剔除（图中标记为红色的方块）。
**时间复杂度：** $O(V + E)$。因为必须遍历所有其他顶点的链表才能把指向被删顶点的边清理干净。

```cpp
bool AddEdgeMatrix(MGraph *G, int x, int y) {
    // 1. 越界检查与合法性校验（不能添加自环边 x==y）
    if (x < 0 || x >= G->vexnum || y < 0 || y >= G->vexnum || x == y) {

    if (G->edge[x][y] == 0) {

    }
```

```cpp
#include <stdio.h>
#include <stdlib.h>
#include <stdbool.h>

#define MAX_V 100

// 边结点
typedef struct ArcNode {
    int adjvex;              // 邻接点域，存储该顶点对应的下标
    struct ArcNode *nextarc; // 链域，指向下一个邻接点
} ArcNode;

// 顶点结点
typedef struct VNode {
    char data;               // 顶点信息
    ArcNode *first;          // 指向第一条依附该顶点的弧的指针
} VNode;

// 邻接表图结构
typedef struct {
    VNode vertices[MAX_V];
    int vexnum, arcnum;
} ALGraph;

// 在图 G 中添加无向边 (x, y) —— 使用头插法 O(1)
bool AddEdge(ALGraph *G, int x, int y) {
    // 1. 简单的合法性校验（假设不包含重复边的校验）
    if (x < 0 || x >= G->vexnum || y < 0 || y >= G->vexnum || x == y) {
        return false; 
    }

    // ==========================================
    // 第一步：将 y 插入到 x 的链表头部 (对应图中 C 后面插入 5)
    // ==========================================
    ArcNode *newNodeY = (ArcNode *)malloc(sizeof(ArcNode));
    newNodeY->adjvex = y;
    
    // 头插法核心逻辑：
    // 1. 新结点的 next 指向原来的第一个结点
    newNodeY->nextarc = G->vertices[x].first; 
    // 2. 头指针指向这个新结点
    G->vertices[x].first = newNodeY;          

    // ==========================================
    // 第二步：将 x 插入到 y 的链表头部 (对应图中 F 后面插入 2)
    // ==========================================
    ArcNode *newNodeX = (ArcNode *)malloc(sizeof(ArcNode));
    newNodeX->adjvex = x;
    
    // 头插法核心逻辑：
    newNodeX->nextarc = G->vertices[y].first;
    G->vertices[y].first = newNodeX;

    G->arcnum++; // 图的总边数加 1
    return true;
}
```

#### 代码
##### 邻接矩阵
C 语言
```cpp
#include <stdio.h>

#define MAX_V 100

// 邻接矩阵结构定义
typedef struct {
    char data[MAX_V];
    int edge[MAX_V][MAX_V];
    int vexnum, arcnum;
} MGraph;

// 求图 G 中顶点 x 的第一个邻接点
int FirstNeighbor(MGraph G, int x) {
    // 1. 合法性校验：如果图中不存在顶点 x，返回 -1
    if (x < 0 || x >= G.vexnum) {
        return -1; 
    }
    
    // 2. 顺序扫描第 x 行
    for (int i = 0; i < G.vexnum; i++) {
        // 如果找到非 0 元素（即存在边），立即返回该列的下标
        if (G.edge[x][i] != 0) {
            return i;
        }
    }
    
    // 3. 扫描完毕都没找到，说明 x 没有邻接点（孤立点），返回 -1
    return -1; 
}
```

##### 补充说明：有向图的“入边”问题

如前所述，上面的代码通杀无向图和有向图的“找第一个出边邻居”。

但如果题目刁钻一点，要求在**有向图**中，找**指向顶点 $x$ 的第一个邻接点（即第一个“入边”邻居）**，情况就会反转：
- **邻接矩阵**：依然很快，只需要从左到右扫描第 $x$ **列**即可，复杂度仍是 $O(1) \sim O(|V|)$。
- **邻接表**：会变得极其痛苦。因为邻接表只存出边，找入边必须遍历其他所有顶点的单链表，时间复杂度直接退化为 $O(|V| + |E|)$。这也是为什么后来发明了“十字链表”和“逆邻接表”的原因。

`NextNeighbor(G,x,y)`

##### 邻接矩阵

```cpp
#include <stdio.h>

#define MAX_V 100

// 邻接矩阵结构定义
typedef struct {
    char data[MAX_V];
    int edge[MAX_V][MAX_V];
    int vexnum, arcnum;
} MGraph;

// 求图 G 中顶点 x 的下一个邻接点 (在 y 之后)
int NextNeighborMatrix(MGraph G, int x, int y) {

    if (x < 0 || x >= G.vexnum || y < 0 || y >= G.vexnum) {
        return -1; 
    }
    
    // 2. 从 y 的下一列 (y + 1) 开始顺序扫描第 x 行
    for (int i = y + 1; i < G.vexnum; i++) {
        // 找到非 0 元素，说明有边
        if (G.edge[x][i] != 0) {
            return i;
        }

    return -1;
```

##### 邻接表
C 语言
```cpp
#include <stdio.h>

#define MAX_V 100

// 边结点
typedef struct ArcNode {
    int adjvex;              
    struct ArcNode *nextarc; 
} ArcNode;

// 顶点结点
typedef struct VNode {
    char data;
    ArcNode *first;          
} VNode;

// 邻接表结构定义
typedef struct {
    VNode vertices[MAX_V];
    int vexnum, arcnum;
} ALGraph;

// 求图 G 中顶点 x 的下一个邻接点 (在 y 之后)
int NextNeighborList(ALGraph G, int x, int y) {
    // 1. 合法性校验
    if (x < 0 || x >= G.vexnum) {
        return -1; 
    }
    
    // 2. 获取顶点 x 的第一条边
    ArcNode *p = G.vertices[x].first;
    
    // 3. 在单链表中遍历，寻找值为 y 的结点
    while (p != NULL && p->adjvex != y) {
        p = p->nextarc;
    }
    
    // 4. 此时 p 要么指向了 y 结点，要么为空（说明 y 根本不是 x 的邻居）
    // 如果 p 不为空，并且 p 的下一个结点也不为空，则返回下一个结点的值
    if (p != NULL && p->nextarc != NULL) {
        return p->nextarc->adjvex;
    }
    
    // 5. 其他情况（没找到 y，或者 y 已经是最后一个结点）均返回 -1
    return -1; 
}
```
