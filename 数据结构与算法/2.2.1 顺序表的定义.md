```cpp
#include<stdio.h>
#define MaxSize 10
typedef struct{
	int data[MaxSize];//静态数组
	int length;//静态顺序表当前的长度
}SqList;
void InitList(SqList &L){
	for(int i = 0; i < MaxSize ; i++){
		L.data[i] = 0;//数据元素初始值设为0
	}
	L.length = 0;//顺序表初始长度为0
}

int main(){
	SqList L;//声明一个顺序表
	InitList(L);//初始化一个顺序表
	//还有一系列操作
	return 0;
}

```

动态分配顺序表需要定义一个指向顺序表第一个数据元素的地址的指针

`malloc ` 函数所实现的事情会申请一整片连续的内存空间，这一整片的内存空间一定有起始的内存地址，malloc 函数执行之后会 return 返回一个指向这一片存储空间开始地址的指针

![[截屏2026-03-15 10.10.50.png]] 🧚‍♀️实际运用中，一般会省略强制类型转换，编译器会进行隐式转换
C++可以用 `new` 和 `delete` 这两个关键字来实现类似于 `malloc` 和 `free` 的功能
	`malloc` 函数作用是申请一整片内存空间
```cpp
#include<stdio.h>
#define InitSize 10
typedef struct{
	int* data;//动态顺序表的起始地址
	int length;//动态顺序表的长度
	int MaxSize;//最大长度
}SeqList;
void InitList(SeqList &L){
	L.length=0;
	L.MaxSize = InitSize;
	for(int i=0;i<L.MaxSize;i++){
		L.data[i]=0;
	}
}
int main(){
	SeqList L;
	InitList(L);
	return 0;
}
```
上述代码报错，**💀 野指针狂飙（没批地就建房）** 你是不是只写了 `SeqList L;`，但是**忘了写 `L.data = (int*)malloc(...)`**，就直接迫不及待地去给 `L.data[2]` 赋值了？

🧚‍♀️直接可以用 realloc 函数进行扩展-->但是 realloc 这个函数在调用的时候会有些意想不到的坑
```cpp
#define InitSize 10
typedef struct{
	int* data;
	int length;
	int MaxSize;
}SeqList;

void InitList(SeqList &L){
	L.length = 0;
	L.MaxSize = InitSize;
	L.data = new int[L.MaxSize];
}

void IncreaseSize(SeqList &L,int len){
	int *p = L.data;
	L.MaxSize += len;
	L.data = new int[L.MaxSize];
	for(int i = 0;i<L.length;i++){
	//⚠️这里不用L.MaxSize
		L.data = p;
	}
	delete p;
}
int main(){
	SeqList L;
	InitList(L);
	IncreaseSize(L,5);
	return 0;
}

```
![[截屏2026-03-15 10.47.50.png]]
🧚‍♀️因为增加元素的方法应该要修改 length，不然还是会出现上节课脏数据那块的情况；L.length = 0 没错，动态顺序表长度此时为零；此操作的顺序为: 建立临时指针指向原空间，data 指针指向新空间，原空间内容通过 p 访问存入 data 相应位置，长度更改，原空间通过 p 指针删除；* p 这个指针用来暂时指向原线性表的所在的空间，然后用 data 这个指针去申请新的空间，申请完了之后，再把每个元素都挪过去。
「PS」free 函数会将 p 指针指向的一整片的空间释放掉，归还给系统，且由于 p 是局部变量当函数执行结束后存储p 的内存空间会被系统自动回收

由于需要将数据复制到新的区域，因此虽然动态分配可以让顺序表大小可以灵活的改变但是时间开销还是很大的

### 课后习题总结
1. 顺序存储对应的是链式存储
2. 顺序存储不用存放指针域，存储密度大
3. 有序表是有序（元素按值排序）的线性表（不一定是顺序存储的也可以是链式存储的）
4. 数组🆚顺序表
	1. 数组逻辑上可以不连续存放，顺序表逻辑上必须连续存放：顺序表（Sequential List）这个数据结构的定义中，如果表里有 $n$ 个元素，那么它们**必须**严格占据底层数组的第 $0$ 到第 $n-1$ 个位置。中间绝对不允许出现空隙
		- **为什么这么严苛？** 因为顺序表的核心是维护一个 `length`（当前长度）变量。如果你留了空隙（比如占据了 0, 1, 3，位置 2 空着），那么 `length` 应该是 3 还是 4？这会让顺序表的管理系统彻底崩溃
		- 数组完全可以声明一个长度为 10 的数组，只在 `arr[0]`、`arr[5]` 和 `arr[9]` 这三个位置放入有用的数据，其他位置全空着（存放默认的乱码或 null）。对于程序员的逻辑而言，这里的有效数据是不连续的、散落的（这就是我们上次提到的“稀疏数组”的概念）。数组根本不在乎你中间是不是留了空位，它也没有 `length` 变量来监控你，它只提供给你 10 个可以通过索引快速访问的“坑位”
	2. 顺序表在 C 语言中对应实现的数据类型即是数组
5. C 语言中元素为结构体类型则存在元素中各字段类型的概念
6. 顺序表在最后进行插入和删除操作不需要移动任何元素，最后增删 O（1）方便
	-  “最后删除”是 $O(1)$ 且不需要移动元素？：在顺序表中，所谓的“删除尾部元素”，在底层物理层面上**根本不需要去抹除那个位置的数据**。顺序表的**核心**是那个记录有效数据个数的计数器 `length`。当你想要删除最后一个元素时，计算机唯一做的动作就是：
	- **`length = length - 1;`**
7. 顺序表被删结点后的结点需要按需前移
8. 顺序存储的有序表查找指定值元素可以采用“顺序查找法”或“折半查找法”，平均时间复杂度最少为 $O (log_2n)$
9. 顺序表🆚链表 
	1. 交换第 3 和第 4 个元素：顺序表需 3 次交换操作；链表需分别找到两个结点前驱，第 4 个结点断链后再插到第 2个结点后，效率低
	2. 如下

|**编程语言**|**对应【顺序表】的数据类型 (内存必须连续)**|**对应【链表】的数据类型 (内存散落，指针相连)**|
|---|---|---|
|**Java**|`ArrayList` (动态), `int[]` (原生静态数组)|`LinkedList` (双向链表)|
|**Python**|`list` (列表), `tuple` (元组)|`collections.deque` (双端队列底层是链表)|
