单链表的节点需要存放数据元素和指向下一节点的指针

顺序表如果需要扩展容量很不方便，但是单链表很容易
🧚‍♀️顺序表好查改元素，链表好增删元素

单链表由一个一个结点组成，一个结点当中需要有一片空间用于存放数据元素的，需要有另一片空间存放指向下一个结点的指针，所以定义一个 struct 类型的结构体表示一个结点，结点中有一个 data 的变量用于存放数据元素（数据域），还有一个指向下一个结点的指针，变量名字叫next（指针域）

**增加新节点**：用 malloc 函数申请一片存储这个节点的空间，用指针 p 来接受 malloc 函数的返回值，让指针 p 指向新节点的起始地址，定义一个代码逻辑将 p 节点插入到单链表当中 

🧚‍♀️等等，不用 typedef struct{   ……    }Lnode; 吗？；这就是结构体的嵌套；因为指针是指向同时含有数据和指针的结构体；定义了一个 next 指针，这个指针指向的类型也是一个 struct LNode 类型；`LNode *p=new LNode ();` Cpp 写法：`struct LNode *p=new LNode;`；C 语言必须用 struct, C++可以直接省略 struct

C++可以直接去掉 `struct`，但定义结构时不能去
```cpp
// 1. 定义结构体
struct LNode {
    ElemType data;
    LNode *next; // C++里可以直接写 LNode*，不需要写 struct LNode*
};

// 2. 申请新空间
// C 语言做法：LNode *p = (struct LNode *)malloc(sizeof(struct LNode));
LNode *p = new LNode; // C++ 做法：简洁明了，不需要强转，不需要 sizeof
```

![[截屏2026-03-20 17.33.06.png]]![[截屏2026-03-20 17.34.37.png]]用 LinkList 表示指向 struct LNode 的指针，当需要表示一个单链表的时候只需要声明一个头指针 L，这个头指针指向了单链表的第一个节点，由于各个结点用 next 指针一个个连起来的，所以只要找到第一个节点即相当于找到整个单链表
🧚‍♀️我喜欢把指针也重命名一个名字；这个地址是不是应该放在上一个的指针域？；`typedef LNode* LinkList;`：这行代码将 `LNode*` 类型（指向 `LNode` 结构体的指针类型）重命名为 `LinkList`；c++直接用：`struct a {  }*LinkList; ` 就好了；意思就是重新定义了结构体类型和结构体类型指针；linklist 就是 lnode 指针的别名；LNode 就是某一个节点，可能是头节点也可能是其他节点，LinkedList 一般指的是头指针LinkList 是 LNode 指针类型，p 的类型与 LinkList 一样都是 LNode 指针，即返回的是指针; 代码是 `LNode *GetElem ` 相当于，你的这个函数会返回一个指向你想要的 LNode 节点的指针; 这里的 LinkList &L 是不是类似 int* &L 此处是指针的引用了

#### 不带头结点的单链表
##### 初始化单链表![[截屏2026-03-20 18.20.41.png]]
```cpp
//1️⃣申明一个链表的节点：包含数据域和指针域
typedef struct LNode{
	int data;
	struct LNode* next;
}LNode,*LinkList;
//2️⃣不带头节点，所以节点第一个就是有数据域和指针域的
LinkList p = new LNode;
```
初始化一个单链表，是空的且没有头结点所以直接=null 即可
Re:
```cpp
typedef struct LNode{
	int data;
	struct LNode* next;
}LNode,*LinkList;
void InitList(LinkList &L){
	L = NULL;
}
int main(){
	LinkList L;
	InitList(L);
	return 0;
}
```
执行 `LinkList L` 之后内存中会开辟一小部分空间用于存放这个头指针 L，之后执行初始化函数，即将 L 的值设为 `NULL`，防止头指针内存当中有以前遗留的脏数据
C++是可以直接通过 `&L` 改变实参的，C 必须要用指针，但是这里因为改变的是 L，这个原本就是指针，在 C 里面必须使用二级指针，形参也就是 LinkList* L，修改的时候必须解引用 `*L= NULL `, 如果没引用符号在初始化函数修改的是头指针 L 的复制品
##### 判断不带头结点的单链表是否为空
```cpp
#include<iostream>
using namespace std;
typedef struct LNode{
	int data;
	struct LNode* next;
}LNode,*LinkList;
void InitList(LinkList &L){
	L = NULL;
}
//‼️
bool isEmpty(LinkList L){
	if(L == NULL)
		return true;
	else
		return false;
}
int main(){
	LinkList L;
	InitList(L);
	if(isEmpty(L))
		cout<<"L是空链表"<<endl;
	else
		cout<<"L不是空链表"<<endl;
	return 0;
}
```

#### 带头结点的单链表
```cpp
#include <iostream>
using namespace std;
typedef struct LNode{
	int data;
	struct LNode *next;
}LNode,*LinkList;
bool InitList(LinkList &L){
	L = new LNode;
	//看了眼🌰代码 思路是有头结点的单链表的头指针是要指向一个头结点的
	if(L == NULL)//写成LNode == NULL了
		return false;//分配内存失败
	//这一行忘了写‼️
	L->next = NULL;//为了清空遗留脏数据
	return true;
}
int main(){
	LinkList L;
	if(InitList(L))
		cout<<"L初始化成功"<<endl;
	else
		cout<<"L初始化失败"<<endl;
	return 0;
}
```
##### 判断带头结点的单链表是否为空
```cpp
#include <iostream>
using namespace std;
typedef struct LNode{
	int data;
	struct LNode *next;
}LNode,*LinkList;
bool InitList(LinkList &L){
	L = new LNode;
	//看了眼🌰代码 思路是有头结点的单链表的头指针是要指向一个头结点的
	if(L == NULL)//写成LNode == NULL了
		return false;//分配内存失败
	//这一行忘了写‼️
	L->next = NULL;//为了清空遗留脏数据
	return true;
}
bool isEmpty(LinkList &L){
	return L->next==NULL;
}
int main(){
	LinkList L;
	if(InitList(L))
		cout<<"L初始化成功"<<endl;
	else
		cout<<"L初始化失败"<<endl;
	if(isEmpty(L))
		cout<<"L是空链表"<<endl;
	else
		cout<<"L不是空链表"<<endl;
	return 0;
}

```
🧚‍♀️把头指针 L 想象成是指向头结点的路标，是给你说头结点在那里，而头结点就比喻成火车头，不用来装货物，只是单链表的开头，更方便于判断链表是否为空，因为为空的话，头结点就指向 NULL
🤨头结点的定义统一了 L->next 是操作数据的
