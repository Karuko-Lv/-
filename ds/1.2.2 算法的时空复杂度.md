导致算法的空间复杂度变化的**主要原因**是算法当中定义的某些和问题规模相关的变量![[截屏2026-03-14 17.52.58.png]]

虽然变量名字一样，但是内存当中变量都是存在不同区域的
函数调用栈，真实情况除了 n 和 abc 还会存放一些函数返回地址等一些信息，每一级的函数调用都需要 k 个字节，k 是一个常数的空间，递归调用层数和问题规模 n 相等
🧚‍♀️规模太大，会有栈溢出的问题；机组函数调用栈，ebp, esp；递归好像就是用栈实现的，执行完依次出栈；递归回溯姐妹们；递归，我之前也以为只会输出love1，但是后来才想到，每次递归都没完全结束，最后递归到love1，但是外面还嵌套着没有结束的love2,3,4,5，如果还是不理解就展开写一下就明白了；call 和 ret；这就是递归多了会造成栈溢出的原因吗；递归调用需要进栈，找到递归出口后，在一个一个出栈；想象每次 if 调用到 loveyou 就是打开了一个下属的小窗口，n＞1 之前都在不断开窗口没能到 printf 那一步，直到 n＝1 不符合 if 条件了
```cpp
#include <cstdio> 

void loveYou(int n){
	if(n>1){
		loveYou(n-1);
	}
	printf("I Love You %d\n",n);
}

int main(){
	loveYou(5);
	return 0;  // 👉 就是加了这一句！给包工头交差！
}
```
